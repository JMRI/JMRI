Release Notes for JMRI 2.14.1 release

Date: July 12, 2012

From: Bob Jacobsen

Subject: Production release 2.14.1 of JMRI/DecoderPro is available for download.

Notes:

This is an update of JMRI production release 2.14 to include additional features, see below.

JMRI is now only available under the GNU General Public License. For more information, please see our copyright and licensing page.

Warnings

JMRI 2.12 was the last release which is completely supported by Java 1.5. In this release, there are minor interface features, such as drag-and-drop of some items, that require your computer to have Java 1.6 installed. Computers with only Java 1.5 will still be able to use JMRI, including PanelPro and DecoderPro, in this release except for those features. Releases in the next cycle (2.99.* and 3.*) will require Java 1.6.

Known problems with this release

There are currently some connectivity issues for Users running Windows v7 64-bit and a Digitrax PR3 interface. If you are experiencing issues, then please raise a question on the JMRI Users group for assistance.

Changes since production release 2.14

Georgio Terdida fixed a bug that may occur when loading an XnTcp adapter.

Paul Bender fixed the port number display for network connection configurations.

Walt Thompson fixed a problem causing panel files with tooltips to fail validation.

Kevin Dickerson

  • fixed a bug where the PR3 could not operate a throttle in Programming Mode
  • Fix an issue when an NullPointerException could occur when changing between signal mast driver types when the signaling system name does not match the directory that the signal mast are held in
  • Fixed an issue in the "Add/Edit Routes" window that could cause panel files to load slowly.
  • Fixed an issue with the Start-Up Wizard that could cause the system to hang when doing the final connect

Matthew Harris fixed a bug that in certain circumstances caused a NullPointerException when accessing the Context report.

Dan Boudreau fixed a problem with the NCE pushbutton turnout lock. This has been broken since version 2.10.
Also fixed a localization issue where some of the menu items weren't displayed in the selected language.

This release includes all decoder definitions up to the release build date.

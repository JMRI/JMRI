Simple Macintosh install

A Macintosh self-extracting archive is available. When you download and run it, it creates a folder containing the double-clickable application and various support files.
  • Download the archive
  • Find the "JMRI.demo.0.7.sea" file and double-click it to expand the folder.
  • You should now have a "JMRI.demo.0.7" folder. The installation is complete.
To run the program, open the JMRI.demo.0.7 folder and double-click on the "JMRI demo" icon.

Possible problems

If after the download you're left with a "JMRI.demo.0.7A.hqx" file, rather than a "JMRI.demo.0.7.sea" file, your browser is not configured to expand files in "binhex" format. You'll need a program like StuffIt Expander, UnStuff or similar to convert this file to a self-expanding archive. Drag the JMRI.demo.0.7A.hqx file to the icon for one of those programs, which should convert it to a JMRI.demo.0.7.sea file.

Hardware requirements

JMRI is primarily developed on a PowerBook 400MHz. We test that it runs on a PowerMac 7200/90 (90MHz) with 40MB of memory. Performance on that machine is poor, but usable: Some windows take several seconds to open, and certain file operations take too long. We use a USB Macintosh connected to a Keyspan PDA Adapter to provide the serial port; for information please see http://jmri.sourceforge.net/DigiTrax/KeyspanAdapters/KeyspanUSB.html. Its not known if a standard Macintosh RS422 serial port will work directly with an MS100; if you have a pre-USB Macintosh and want to use this demo, please contact us for more information.

Simple Windows install

A .zip format archive is available. When you download and expand it, it creates a folder containing the application script and various support files.
  • Download the archive
  • Find the "JMRI.demo.0.7.zip" file and expand it using WinZIP or a similar program.
  • You should now have a "JMRI.demo.0.7" folder. The installation is complete.

We don't have a startup script for Windows yet, so to start the program type: 

  set CLASSPATH=lib/log4j.jar;lib/collections.jar;lib/xerces.jar;lib/jdom-jdk11.jar
  java -jar jmri.jar
Note this assumes that Swing and the Java communications package are already installed on your computer. If so, you can delete the large lib/swingall.jar file.

Simple OS2 install

A .zip format archive is available. When you download and expand it, it creates a folder containing the application script and various support files.
  • Install the swing and javacomm packages from the IBM web site.
  • Download the archive
  • Find the "JMRI.demo.0.7.zip" file and expand it using 
      PKZIP -extr -dir JMRI.demo.0.7
  • You should now have a "JMRI.demo.0.7" directory. The installation is complete.
The install creates a large lib/swingall.jar file which is not needed on OS/2, so you can delete it without problems.

The provided JmriDemo.CMD script will start the program. Type 

   JmriDemo
in the JMRI.demo.0.7 directory.

Simple Linux install

A .tar.gz format archive is available. When you download and expand it, it creates a folder containing the application script and various support files.
  • Download the archive
  • Find the "JMRI.demo.0.7.tar.gz" file and expand it using 
      gunzip JMRI.demo.0.7.tar.gz
  • You should now have a "JMRI.demo.0.7.tar" file. Expand it using the command 
      tar -xf JMRI.demo.0.7.tar
  • You should now have a "JMRI.demo.0.7" directory. Note that you should run the program from this directory. You can move that directory to another location if needed.
Unfortunately, we don't yet have clear instructions for creating the Java communications package needed for Linux. There is some information available at http://www.interstice.com/~kevinh/linuxcomm.html. You have to follow those instructions to configure your linux machine, and then edit the JMRI.demo.0.7/JmriDemo.tcsh file to include the files you've created.

To start the program, type: 

  ./JmriDemo.tcsh
from the JMRI.demo.0.7 directory.

background information

Note that this structure described here started in the 0.7 release of JMRI. If you have something earlier than that installed, you'd be better off just erasing the old version and starting with a simple install. When we create later releases of JMRI, we'll provide the links to download just the changed parts.

The JMRI-specific code is in a Java archive file called "jmri.jar". This file changes from one JMRI release to the next.

There are also several .jar files in the lib subdirectory that provide libraries needed by the JMRI demo program. When you do a simple install, these are included. They generally do not change from one release to another of the JMRI code. Depending on what's already on your system, you might not need dedicated copies of these:

  • log4j.jar - This provides a configurable logging and error reporting system. It allows us to capture debugging info when somebody has a problem, without creating large log files during normal operation. For more information, see the log4j home page.
  • jdom-jdk11.jar - Lightweight toolkit for reading and writing XML files. In release 0.7, this is only needed by the symbolic programmer tool, but its so useful that other tools will be using it soon. For more information, see the JDOM home page.
  • xerces.jar - An XML parser used for our configuration files. This is a commonly used utility. If you have other Java programs installed, you might already have this.
  • collections.jar - Part of recent versions of Java, this file provides a powerful set of collection classes. You might not need this file if you have Java 1.3 or later installed. Its small, so we include it in our distributions.
  • swingall.jar - Part of recent versions of Java, this file provides the Swing graphical user interface. You might have this (or the equivalent swingnew.jar) file already installed as part of Java, in which case you don't need this one. Macintosh systems before MacOS X don't generally have this, so we include it in the Macintosh (.hqx) distribution. Unfortunately, swingall.jar is a 4MB file.
  • javax.comm.MRJ.jar - A Macintosh-specific implementation of the Java communications classes. Only Macintosh computers use this file. Other distributions don't have it.
  • junit.jar - Part of the development tests, we will send you this file if needed for debugging. Most distributions don't have it.

The "xml" subdirectory is also updated from one JMRI release to the next. To pick up the most recent decoder configurations, etc, you need to update this directory. Be careful to preserve any files you've modified (and that you've not contributed, so that they appear in the distribution).

To summarize, to update to a new JMRI release, you need to:

  • Get a new jmri.jar file and drop it into the distribution directory. This updates the program itself.
  • Update the contents of the xml folder/directory. This updates data files used by the program.

For future releases, we plan to provide "updaters" with just the changed files.

Macintosh details

MacOS 8.1 through 9.2

If you want to make a Java archive (.jar) file available to any program on your Mac, drop it into the MRJClasses folder. Its located inside a set of folders: 
 System Folder
    Extensions
       MRJ Libraries
          MRJClasses
A single swingall.jar file in that folder, for example, will serve all applications that need it. You can then remove it from the lib folder of the JMRI distribution.

MacOS X

There is currently no implementation of the Java communications toolbox for MacOS X. This means that the JMRI demo can run natively in MacOS X or under Classic, but neither can communicate with the actual layout.

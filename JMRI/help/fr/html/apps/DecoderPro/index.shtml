Introduction au Programmateur Symbolique JMRI

Les décodeurs DCC modernes sont des bêtes compliquées à programmer. L'idée simple de "mettre l'adresse dans CV01" ne s'adaptent pas bien lorsque vous programmer des fonctions complexes en utilisant des combinaisons de modèles de bits. Une interface plus conviviale est nécessaire. Il existe des programmes qui offrent de meilleures façons de programmer des décodeurs spécifiques. Malheureusement, elles sont limitées à certains types d'ordinateurs, et que seuls leurs auteurs puissent les personnaliser pour les nouveaux types de décodeurs.

Le programmateur symbolique DecoderPro est destiné à améliorer la situation. Il est configuré en utilisant des fichiers texte, de sorte qu'il peut être adapté aux types de décodeur supplémentaires facilement. Il communique avec les décodeurs en utilisant l'interface de programmation JMRI, de sorte qu'il peut fonctionner sur n'importe quel matériel informatique et Matériel Réseau qui supporte JMRI. Il est disponible gratuitement en téléchargement. Et puisque le code est disponible via l'open source à GitHub, si vous voulez l'améliorer, vous pouvez le faire.

Le Tour Rapide introduit à DecoderPro et comment vous pouvez l'utiliser.

DecoderPro est aussi couvert dans Joe Fugate's TrainMasters.TV video library, part 3.4, car il montre comment il utilise DP sur son réseau Siskiyou Line.

De quoi avez-vous besoin pour utiliser DecoderPro

DecoderPro fonctionne sur Windows, Linux, Macintosh ( MacOs 8.6 à 9.2 OS X ). Il fonctionne probablement aussi avec la plupart des systémes d'exploitations modernes qui ont Java disponible; contactez nous pour de l'aide si vous souhaitez l'essayer sur un autre système. Vous avez besoin d'avoir un port série ou un port USB disponible, soit sur lordinateur lui-même ou soit par un adaptateur USB. Pour plus d'information, svp voir les pages de Configuration.

Vous pouvez aussi avoir besoin d'une connexion à votre système DCC. Pour plus d'information, svp voir les pages matériel DCC.

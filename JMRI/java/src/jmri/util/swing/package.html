Tools and utilities for doing a systematic reworking of the JMRI GUI.

The goal is to allow various GUI types to be available. Subpackages provide key implementation pieces:

{@link jmri.util.swing.sdi}
The traditional JMRI "single-document interface", where there are multiple independent windows
{@link jmri.util.swing.mdi}
A "multi-document interface", where a primary window contains multiple independent sub-windows
{@link jmri.util.swing.multipane}
A "multi-pane" or IDE-style interface", where each window is tiled with inter-related panes
Creating new "windows" through a {@link jmri.util.swing.WindowInterface} reference handles the type-specific aspects.

JMRI Swing Usage Structure

JMRI GUI code starts as {@link jmri.util.swing.JmriPanel} classes, which extends the Swing JPanel. Managing the frames (windows) that enclose them is done automatically when they are opened via the {@link jmri.util.swing.JmriAbstractAction} and {@link jmri.util.swing.JmriNamedPaneAction} action classes. {@link jmri.util.swing.JmriNamedPaneAction} is preferred because it defers loading of the referenced panel class until the action is invoked, which can reduce JMRI startup time signficantly.

Generic creation of menus, toolbars and navigation trees from XML definition files are provided by the {@link jmri.util.swing.JMenuUtil}, {@link jmri.util.swing.JToolBarUtil}, {@link jmri.util.swing.JTreeUtil} classes

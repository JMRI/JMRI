JMRI: Site Map

Motivation

The JMRI web site redesign is being done to address the following problems:

  • Better organization - This requires some real thought. Our existing pages tend to have just grown like Topsy, so they contain too many different things, or aren't linked together in any useful way, or sometimes both.

    It would be great if somebody spread the entire web site across their kitchen table and rearranged the parts until they made some kind of sense.

  • Better explanations - We're really short of step-by-step instructions for doing various things. (It's also hard to find the ones we have, but that's the previous problem). Once you've spend some time confused about using some feature, and then finally figured it out, it would be _very_ useful to write down "Here's how to make it work". That way the next guy perhaps won't have to struggle so much. "What I wish I'd known at the start" is also useful. These shouldn't be detailed technical tomes about all the little bits, but rather a straight-forward pass through one thing. (I'm really poor at writing things like that because I have trouble seeing past the details)
  • A list of Frequently Asked Questions - Having a single page of short questions and answers would help some people get started. The JMRIusers list has a _lot_ of questions on it, so if somebody were to take a read through looking for the ones that come up more than once, that would be a great place to start.

Even small pieces would really help.

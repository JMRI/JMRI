JMRI: Préférences utilisateur ECoS

Les préférences ECoS, vous permet de contrôler la façon dont les Ecos interagissent avec JMRI pour différentes fonctionnalités.

Liste

Ajouter Locos à l'Ecos
Quand une nouvelle Fiche est créée dans JMRI, cette option vous permet de définir si la loco doit être ajoutée à l'Ecos.

Retirer Locos de l'Ecos
Quand une fiche est supprimée dans JMRI, cette option vous permet de définir si la loco doit être supprimée de l'Ecos ou pas.

Ajouter des Locos à la Liste JMRI
Lorsque JMRI détecte qu'une nouvelle loco a été ajoutée dans la base de données Ecos, cette option vous permet de définir si la loco doit être ajoutée dans la liste ou non.

Retirer des Locos de la Liste JMRI
Lorsque JMRI détecter qu'un loco a été retirée de la base de données Ecos, cette option vous permet de définir si la loco doit être supprimée de la liste ou non. Remarque: Actuellement, si une locomotive a été retirée de l'Ecos quand il n'est pas connecté à JMRI, cette élimination n'est pas détectée.

Régulateur

Locos temporaires
L'Ecos nécessite d'entrée dans sa base de données avant qu'il ne soit possible de commander une locomotive. Lorsque vous sélectionnez une fiche ou entrer une adresse de loco dans la zone Adresse d'un régulateur et que cette adresse n'existe pas dans la base Ecos, JMRI créera automatiquement une fiche correspondante avec une description "Créé par JMRI". Lorsque JMRI est arrêté, ces locos temporaires peuvent soit être laissées sur l'Ecos ou supprimées. Ce paramètre vous permet de définir comment cela fonctionne.

JMRI fera trois tentatives pour essayer de gagner cette loco, si la troisième tentative échoue, l'utilisateur sera présenté avec une invite sur l'opportunité de forcer la commande de loco. Ici vous pouvez choisir soit toujours forcer une commande pour prendre le relais, soit ne jamais forcer une commande ou à prendre toujours quand invité à le faire.

Aiguillages

Retirer Aiguillages de l'Ecos
Si un aiguillage est retiré du tableau aiguillage dans JMRI, ce paramètre détermine si l'aiguillage doit être supprimé de l'Ecos.

Ajouter Aiguillages dans JMRI
Si JMRI reçoit une notification indiquant que l'aiguillage ou que l'accessoire a été ajouté à l'Ecos pendant que JMRI est en marche, ce paramètre détermine comment JMRI devrait réagir.
Remarque: C'est ce qui n'empêche pas JMRI de remplir le tableau d'aiguillage lorsque l'application est ouverte.

Retirer Aiguillages dans JMRI
Si JMRI reçoit une notification indiquant que l'aiguillage ou que l'accessoire a été retiré de l'Ecos, ce paramètre détermine comment JMRI devrait réagir.

L'élément Palette

L'élément Palette est utilisé pour placer des icônes et du texte sur l'éditeur de TCO. La palette est une fenêtre à onglets avec des onglets pour chaque type d'élément. La méthode commune en usage pour tous les éléments pour en ajouter un par un TCO est "Glisser et Déposer". Chaque onglet dispose d'un Panneau avec bordure étiqueté Glisser vers le Panneau. Faites glisser l'élément à l'intérieur de cette frontière vers le panneau. Dans le cas où l'icône représente un dispositif de configuration (par exemple un aiguillage, un capteur, etc), vous devez sélectionner une ligne d'une table à associer à ce dispositif particulier avec l'icône - et ensuite faites glisser l'icône sur le TCO.

Articles Iconiques

Pour ces articles utilisant des icônes, de multiples ensembles d'icônes sont offerts par l'élément Palette. Dans chacun de ces cas plusieurs jeux d'icônes différents sont fournis par défaut. Les boutons radio sont utilisés pour sélectionner un jeu d'icônes.

Par exemple, sous l'onglet du capteur vous pouvez avoir des ensembles de grands voyants, petits voyants, chaque taille avec plusieurs jeux avec des couleurs différentes pour les actifs et inactfs - par exemple rouge-vert, jaune-noir, etc

Le bouton Montrer Icônes affiche toutes les icônes dans le jeu sélectionné. Le bouton Modifier Icônes ouvre une boîte de dialogue où les icônes peuvent être modifiées en faisant glisser de nouvelles icônes depuis le catalogue icône.

Nouveaux Jeux Icônes

Des jeux d'icônes peuvent être ajoutés ou supprimés à partir de la boite de dialogue Modifier Icônes. Il n'y a aucune limite sur le nombre de jeux d'icônes qui peuvent être ajoutés à un onglet. Lorsque l'élément Palette est personnalisé avec d'autres jeux d'icônes, la fermeture de la Palette avec le bouton de fermeture dans la barre de titre va ouvrir une boîte de dialogue vous demandant si vous souhaitez enregistrer vos modifications. Lorsque des modifications sont sauvegardées l'élément Palette est stocké dans le fichier CatalogTrees.xml dans le répertoire des préférences.

La boite de dialogue Modifier Icônes est utilisée pour supprimer un jeu d'icônes ou d'ajouter un jeu d'icônes supplémentaires. Le bouton Supprimer Jeu d'Icônes supprime le jeu d'icônes sélectionné. Lorsque le bouton Ajouter Jeu d'Icones est pressé, le jeu d'icônes sélectionné est sauvegardé et un jeu d'icônes avec des icônes "inconnues" est affiché (Dark X). Changer ces icônes pour celles que vous voulez en faisant glisser des icônes du catalogue d'icônes. Le nouveau jeu d'icônes doit être nommé dans le champ de texte au sommet de la boite de dialogue. Enfin, appuyez sur le bouton Créer Jeu d'Icones et la boîte de dialogue est fermée avec le nouveau jeu d'icônes ajouté à la liste des boutons radio.

Table des éléments

Les éléments associés à une table ont un bouton Ajouter un Nouvel élément au Tableau. Ce bouton ouvre un dialogue où un nouvel élément d'un dispositif peut être ajouté à cette table. Les onglets avec des tableaux nécessitent la sélection d'un dispositif (une ligne dans le tableau) avant que l'icône puisse être glissée sur un TCO pour le représenter.

Onglets élément Palette

  • Aiguillage Les jeux d'icônes d'aiguillage à la fois droite et gauche sont contenues dans cet onglet. Sélectionnez un aiguillage depuis la table et un jeu d'icônes, puis faites glisser sur le TCO.
  • Capteur Sélectionner un capteur de la table et un jeu d'icônes, puis faites glisser sur le TCO.
  • Tête de Signal Quand une tête de signal est sélectionnée dans le tableau, le Bouton [Montrer Icônes] n'affiche que les icônes pour les apparences valides du signal. Lorsque aucune ligne n'est sélectionnée toutes les icônes avec les apparence possibles sont indiquées. Lorsque vous ajoutez un nouveau jeu d'icônes, toutes les icônes avec les apparence possibles sont indiquées.
  • Mât de Signal Un mât de signal doit être sélectionné depuis la table pour faire glisser son icône sur le TCO et d'afficher l'icône qu'il utilise pour ses aspects.
  • Mémoire Sélectionnez un emplacement mémoire dans la table. Les trois types de mémoire ont chacun une zone "Glisser sur le TCO". Faites glisser l'élément à l'intérieur de la frontière "Glisser sur le TCO" sur le TCO. Notez que les types Entrée et Spinner ne peuvent être traînés par les bords de leurs zones de texte.
  • Rapport Sélectionner un rapport dans le tableau et faites glisser l'élément à l'intérieur de la frontière "Glisser sur le TCO" sur le TCO.
  • Lumière Sélectionnez une lumière (comme un objet CMRI) de la table et faites glisser sur le TCO.
  • Multicapteurs La table affiche le Tableau Capteur et le jeu d'icônes par défaut à trois positions de levier. Pour le cas par défaut, trois rangées doivent être sélectionnées depuis la table pour placer l'icône de multicapteurs dans un panneau. Les sélections de tableau sont étiquetés "première", "Seconde", etc, et ces étiquettes correspondant aux étiquettes s'affichent lorsque le jeu d'icônes complet est affiché. Quand vous éditez un jeu d'icônes des niveau de position, peuvent être ajoutés ou supprimés. Autrement dit, vous pouvez faire un capteur à deux multicapteurs ou un jeu d'icônes de quatre ou cinq multicapteur.
  • Icône Les icônes affichées à la frontière (étiquetées à leur nom) peuvent être glissées directement sur un TCO. Les icônes peuvent aussi être glissées directement à partir du Catalogue d'icônes.
  • Arrière Plan Les icônes d'arrière Plan sont placées dans un panneau comme le sont les icônes normales. Cependant, elle sont affectés du plus bas niveau d'affichage et ne peut être sélectionnés avec un simple clic de souris. Utilisez "Ctrl-Clic" pour sélectionner une icône de fond. Plutôt qu'une icône, une couleur de fond personnalisée peut être choisie avec le bouton Couleur de Fond. Ce bouton affiche un sélecteur de couleur avec des onglets pour les différentes façons de définir une couleur.
  • Texte L'onglet Texte affiche un champ de texte pour entrer le texte et les options d'attribut de rendu du texte. Styles et taille de Police, la justification et les attributs de la frontière peut être choisi. Un sélecteur de couleur fournit des couleurs personnalisées pour les polices, les arrière plans et les bordures. Remarque La "touche Entrée" doit être pressée pour afficher le texte avec ses attributs dans le Panneau de Prévisualisation. Faites glisser l' élément du panneau de prévisualisation vers le TCO.
  • RPSReporter Faites glisser l'icône "Glisser vers le TCO" sur le TCO et installer le RPSReporter à partir de son menu déroulant.
  • Horloge Rapide Glisser l'horloge sur le TCO.
  • Indicateur de Voie L'indicateur de voie est conçu pour être utilisé quand les circuits de voie sont configurés avec les OBlocks. La plupart des éléments dans cet onglet appartient à leur utilisation comme présentations des OBlocks et leur utilisation avec des itinéraires mandatés. Cependant, si les OBlocks ne sont pas définis, mais que les capteurs d'occupation sont installés sur le TCO comme des circuits de voie, le Circuit Occupation acceptera un nom de capteur. Lorsque c'est le cas seuls les couleurs "Occupés" et "Libres" peuvent être affichées.
  • Indicateur d'Aiguillage Le commentaire ci-dessus s'applique aussi aux Indicateurs d'Aiguillage.

Plus d'Infos sur les Indicateurs de Voies

Les Indicateurs de Voies (à savoir les segments de voie et aiguillages) ont des couleurs pour afficher l'état du circuit de voie qu'ils représentent. Lorsque le circuit de voie est configuré par un OBlock, et les Portails et Chemins qu'il contient, les Statuts d'état suivants sont affichés en couleur:
  • Libre Indique prêt à l'emploi. Plus précisément, le circuit est dans un état connu et aucun des autre status ne s'applique.
  • Occupés (Jaune) La voie est occupée par un train inconnu ou voiture.
  • Alloué (Vert) La voie est réservée pour une utilisation par un mandat.
  • Position du Train (Bleu) La voie est occupé par un train sous mandat et c'est sa position actuelle.
  • Non En cours d'utilisation (Gris) L'état de la voie n'est pas connu ou qu'elle a été fixée à hors de service. Cet état est également utilisé lorsque les cantons sont supervisés par un panneau local.
  • Erreur Alimentation (Rouge) C'est l'état d'un autre capteur - pas le capteur d'occupation. S'il est utilisé, le plus probable, il serait câblé pour indiquer un court-circuit ou un échec d'alimentation.

Remarque: La configuration des Icônes Indicateur Voie peut être fait en groupe automatiquement - y compris la conversion automatique des icônes de voie régulièrement avec le Générateur de Circuit. Le seul élément où vous pouvez vouloir éditer un indicateur Icône de Voie individuellement est de choisir l'icône qui montre le nom du train quand le bloc représenté est occupé en vertu d'un mandat. (Souvent, plusieurs icônes affichent un OBlock et vous voulez seulement l'un d'eux pour montrer le marqueur.)

Onglet Indicateur de Voie

Entrez le nom du circuit de voie (OBlock), cette icône voie représente dans le champs Circuit d'Occupation. La bouton Ouvrir Liste de Sélection de Détecteur ouvre une liste de choix montrant les circuits voie OBlock défini. Faites glisser ou copier le nom du champ du Circuit d'Occupation. Ne pas utiliser l'onglet "Table Capteur" de la liste de sélection à moins qu'il n'y ait pas de OBlocks configurés et que vous souhaitiez utiliser uniquement le statut "Occupé/Inoccupé".

(Obsolète) Quand un nom de OBlock valide est entré dans le champ Circuit d'Occupation, un panneau CircuitPath est ajouté à l'onglet. Si les chemins ont été définis pour le bloc, ils sont représentés dans ce panneau. Vérifiez les chemins du circuit de voie qui passent sur cette icône. Lorsque ces chemins font partie d'un itinéraire alloué ou sont occupés par un train mandaté, elles montrent ce statut par la couleur.

Si les chemins ont été créés ou édités par le Générateur de Circuit, la fonction ci-dessus se fait automatiquement. Dans l'avenir ci-dessus un "sous" panneau sera mis en lecture seule.

En dessous du champ de circuit d'occupation est une case libélé Affichage du Nom du Train quand occupé par un train mandaté. Quand un train mandaté occupe le circuit de voie, les icônes circuit ayant cette case cochée, montrent un marqueur avec l'Id du train occupant. Très probablement, vous ne voulez seulement l'une des icônes dans le circuits de voie pour afficher l'ID du train.

Les boutons radio Jeux d'Icônes , la zone Glisser sur le TCO et les boutons de fonction Afficher les Icônes et Modifier Icônes comme décrit avant.

Onglet Indicateur Aiguillage

Cet onglet contient les mêmes fonctions que l'onglet Indicateur de Voie avec l'ajout d'un tableau Aiguillage. Un aiguillage doit être sélectionné dans la Table Aiguillage pour être en mesure de faire glisser une icône sur un TCO.

Une légère variation sont les boutons Ajouter Jeu d'Icônes et Supprimer Jeu d'Icônes au bas de l'onglet. En outre, le boutonAfficher les Icônes affiche le jeu d'icône dans une matrice où les lignes montrent l'état de l'icône pour chaque état de l'aiguillage - direct, Dévié, Incohérent et Inconnu. à la droite de chaque ligne de la matrice est un bouton Modifier des Icônes. Ces boutons ouvrent une boîte de dialogue Modifier des Icônes pour chaque ligne de la matrice. Le bouton Masquer les Icônes ferme la matrice icône et retourne sur l'onglet à son point d'origine.

Quand un des boutons Modifier des Icône est pressé sur une rangée de l'affichage de l'icône, une boîte de dialogue d'édition de la ligne est ouverte. Les icônes peuvent être modifiés en faisant glisser des icônes à partir du Catalogue d'Icônes. Aussi le statut entier peut être retiré par le bouton Supprimer Statut - ou restauré par le bouton Ajouter état Manquant.

Aide de la table de transit

Tous les transits dont JMRI a connaissance peuvent être consultés en utilisant le Tableau de transit.

Pour ouvrir cet outil, sélectionnez Transits dans le sous-menu Tables du menu Outils de la fenêtre du menu principal du programme JMRI.

Utilisation de la table de transit

Les transits sont mis en place dans la table de transit. Lorsqu'ils sont configurés, Les transits sont indépendants des trains qui les utilisent.

Tous les transits qui existent actuellement peuvent être référencés en utilisant l'outil Transits sous Tables dans le menu Outils de la plupart des programmes JMRI. Il y a sept colonnes dans la Table de transit:

  • Nom Système - Le Nom Système est attribué au Transit lorsqu'il est créé, et ne peut être modifié.
  • Nom Utilisateur - Si vous souhaitez avoir vos transits porter des noms "lisibles par l'homme", vous pouvez cliquer dans le "Nom Utilisateur" et modifiez ces informations en fonction de votre choix.
  • État - L'état d'un transit indique si il est LIBRE (disponible pour l'activation et l'affectation à un train), ou AFFECTÉ (liée à un Train pour former un Train Actif). L'état d'un Transit nouvellement ajouté est toujours LIBRE.
  • Commentaire - Cette colonne est à votre usage pour enregistrer tout commentaire que vous souhaitez utiliser pour garder une trace de vos transits. C'est pour information seulement.
  • Cliquez sur le bouton Effacer dans une ligne spécifique pour supprimer le transit dans cette ligne. Utilisez ce bouton avec attention ! Ne supprimez pas un Transit en cours d'exécution.
  • Cliquez sur le bouton Editer dans une rangée pour afficher ou modifier la saisi des informations pour un transit. Le Nom Système ne peut pas être changé. Le Nom Utilisateur peut être modifié, les Sections peuvent être redéfinies, etc. Un seul Transit peut être édité à la fois. Le transit ne peut être modifié si un autre est en cours de création, ou si le transit n'est pas dans l'état LIBRE.
  • Cliquez sur le bouton Dupliquer dans une rangée pour ajouter un autre Transit identique à celui-ci. Cliquer sur ce bouton est similaire au fait de cliquer sur le bouton Ajouter... (voir ci-dessous) sauf que les informations de ce transit sont copiées dans la fenêtre Ajouter/Modifier de Transit. Cette option offre un moyen facile de saisir plusieurs exemplaires d'un Transit lorsque plusieurs Trains doit suivre le même itinéraire au même moment.

Pour définir un nouveau Transit, cliquez sur le bouton Ajouter.... Une fenêtre Ajouter/Modifier le Transit apparaîtra, vous permettant de désigner les sections du nouveau Transit. Comme un transit ne peut être utilisé que pour un seul train actif à la fois, l'option Dupliquer (voir ci-dessus) peut permettre de gagner du temps lors de la création de plusieurs copies du même transit.

Utilisation de l'Éditeur de Réseau pour configurer les transits

Si vous utilisez un panneau de l'Éditeur de Réseau pour définir et générer la logique des Mâts de Signalisation et des sections, il est possible de créer un transit dans l'Éditeur de Réseau. Pour ce faire, affichez le menu contextuel (clic droit) d'un mât de signalisation sur le panneau. Si les choses ont été correctement mises en place, vous verrez la possibilité de Créer un transit à partir d'ici. Après avoir sélectionné cette option vous pouvez progresser tout au long de votre parcours et sélectionner chaque Mâts de Signalisation, en affichant le menu contextuel sur chaque en sélectionnant Add to Transit. Lorsque vous êtes aux Mât de Signalisation de la destination finale sur votre itinéraire, sélectionnez Ajouter au transit et Terminer dans le menu contextuel de ce dernier Mât de Signalisation.
Veuillez noter que vous devrez sauvegarder manuellement les Transits sur disque (voir ci-dessous).

Préparation pour un Fonctionnement Continu

Un transit peut être mis en place pour un Fonctionnement Continu, permettant à un Train Actif de voyager à travers son Transit plusieurs fois sans être terminé et réactivé. À condition que son transit soit mis en place pour un fonctionnement continu, chaque fois qu'un train actif termine son parcours dans son Transit, Le train peut l'Activer, en option, pour être réinitialisé automatiquement par le Dispatcher ainsi il est prêt à repasser par son Transit. Pour qu'un Transit soit capable de fonctionnement en continu, sa première section et sa dernière section doivent être les mêmes, et les deux doivent être mises en place pour fonctionner dans le même direction. Cela s'applique à la fois aux systèmes manuels et automatiques de fonctionnement des trains.

Pour plus d'informations sur le réglage de l'option de fonctionnement continu, visitez l'Aide Activer de nouveaux Train .

Spécifier les actions pour le fonctionnement automatique

Les Transits soutiennent toute une série d'Actions que se produisent automatiquement lorsqu'un train circule dans le Transit en mode automatique. Une ou plusieurs Actions peuvent être spécifiées avec chaque section en transit. Les Action sont initiées lorsque leur Section associée dans le Transit est pénétrée par le train en automatique.
Chaque Action comporte deux parties:

  • Une partie Quand, qui précise quand l'Action doit être effectuée, et
  • Une partie Quoi, qui spécifie quelle Actiondoit être effectuée.

Une action couramment utilisée, par exemple, est "Lorsque le train démarre, allumez sses feux". Plus d'informations sur les Actions est situé dans les Actions. Lire plus sur Actions.

Sauvegarde des transits sur disque

Utiliser l'élément Sauvegarder dans le menu Fichier pour enregistrer les informations du transit sur le disque. Ces informations sont sauvegardées avec la configuration dans un fichier XML, ainsi que la configuration du panneau de contrôle et autres articles similaires. Pour stocker vos panneaux dans le même fichier (recommandé), sélectionnez Fichier -> ; Sauvegarder > ; Sauvegadrer Configuration et panneaux à classer... ou sélectionnez Stocker les Panneaux... dans la fenêtre principale de JMRI.

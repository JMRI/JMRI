Tableau de Section

Avant de plonger dans le fonctionnement du Tableau de Sections, fournissons quelques informations de base sur les Sections et leur utilisation avec le Répartiteur .

Conditions d'utilisation des Sections

Les sections peuvent être utilisées avec un panneau d'éditeur de panneaux, avec un panneau Éditeur de Réseau , avec un ordinateur CTC ou directement sans panneau. Les sections font partie de Dispatcher et s'appuient sur la structure cantons et Chemins de JMRI. Par conséquent, avant de tenter de créer des sections, vous devez initialiser les Cantons et Chemins pour votre réseau. (Un moyen facile d'initialiser automatiquement, vos cantons et Chemin doivent esquisser un Éditeur de Réseau du panneau de votre Réseau, avec les cantons attribués. En attribuant des cantons à tous les segments de voie, aiguillages, etc. dans l'éditeur de réseau, les chemins seront créés automatiquement par le logiciel.)

Le cantonnement du réseau doit être soigneusement planifié, afin de fournir des Sections "raisonnables" - Sections qu'il est judicieux pour un répartiteur d'allouer à un train. Par exemple, pour les aiguillages droits ou gauches qui changent de voie principale, cela signifie généralement d'inclure l'aiguille dans le même canton que la voie entrant au niveau de l'aiguille et de prévoir des cantons séparés pour les voies continues et divergentes. Certains préfèrent avoir un aiguillage dans un canton séparé, et cela fonctionne très bien aussi. Du point de vue du dispatching, il n'est pas nécessaire de bloquer séparément les voies de passage des lignes principales desservant des industries, mais le cantonnement est acceptable ici. Un schéma de cantonnement du Réseau qui fonctionne bien pour les signaux devrait également fonctionner pour les sections.

Les signaux ne sont pas nécessaires pour utiliser les sections, mais les sections fonctionneront correctement si les signaux sont installés. Si les signaux sont installés, les capteurs de direction inclus dans les sections permettent de configurer la signalisation APB simple (voir ci-dessous).

Utilisation du Tableau de Section

Toutes les sections dont JMRI a connaissance peuvent être visualisées à l'aide du Tableau de Sections. Les Sections peuvent être créées manuellement dans le volet Tableau de Section ou automatiquement si vous envisagez également d'utiliser la Logique du Mât de Signalisation (voir ci-dessous) .

Pour ouvrir cet outil, sélectionnez Sections dans le sous-menu Tableaux du menu Outils de la fenêtre principale du programme JMRI.

Colonnes du Tableau de Section

Vous pouvez modifier l'ordre des colonnes en faisant glisser leurs titres et trier les lignes en cliquant sur l'un des triangles de tri en regard d'un titre de colonne. La table des sections comprend huit colonnes:
  • Nom Système - Le Nom du Système est attribué à la Section lors de sa création et ne peut pas être modifié.
  • Nom Utilisateur - Si vous souhaitez que vos sections portent des noms "lisibles par l'homme", vous pouvez cliquer dans la colonne "Nom Utilisateur" et modifier les informations à votre guise.
  • État - L'état d'une section indique si une section est "attribuée" ou "utilisée" par un ou plusieurs trains. L'état d'une section nouvellement ajoutée est toujours LIBRE. Lorsqu'une Section est affectée à un déplacement dans la direction allant du "Premier Canton" au "Dernier Canton", l'état est AVANT. Lorsqu'il est affecté à un déplacement dans le sens opposé, l'état est INVERSE.
  • Commentaire: cette colonne vous permet d'enregistrer les commentaires que vous souhaitez utiliser pour garder une trace de vos sections. C'est pour information seulement.
  • Cliquez sur le bouton Supprimer dans une ligne spécifique pour supprimer la section dans cette rangée. N'oubliez pas de supprimer tous les Transits utilisant cette Section. Tout Transit utilisant une Section supprimée sera cassé! Utilisez ce bouton avec prudence!
  • Premier Canton: cette colonne affiche le premier canton de la section. Puisqu'une section est un groupe d'un ou plusieurs cantons connectés, il y a toujours une entrée.
  • Dernier Canton - Cette colonne indique le dernier canton de la section. Puisqu'une section ne peut contenir qu'un seul canton, le dernier canton peut être identique au premier.
  • Cliquez sur le bouton Modifier dans une rangée pour afficher ou modifier les informations entrées pour une section. Le Nom Système ne peut pas être changé. Le Nom Utilisateur peut être modifié, les cantons peuvent être redéfinis, les points d'entrée peuvent être reconfigurés, etc. Une seule Section peut être modifiée à la fois. Une Section ne peut pas être modifiée pendant la création d'une autre.

Création d'une Nouvelle Section

Pour créer une Nouvelle Section, cliquez sur le bouton " Ajouter .." situé au bas du volet Tableau de Section. Une fenêtre apparaîtra vous permettant de définir les Cantons inclus dans la nouvelle Section et, le cas échéant, de désigner les points d'entrée de la Section entraînant un déplacement dans les directions AVANT et INVERSE.

Actuellement, l'utilisateur doit sélectionner manuellement AVANT ou ARRIÈRE pour chaque point d'entrée après l'ajout de tous les Cantons à une Section. Le programme définira automatiquement certaines directions, mais d'autres resteront sous la forme INCONNU. Un point d'entrée doit être réglé sur AVANT si un train entrant dans la Section à ce point d'entrée empruntait la direction AVANT. Sinon, il devrait être réglé sur INVERSE. Un éventuel ajout consiste à définir automatiquement les directions des points d'entrée à partir de la connectivité trouvée dans un panneau de l'éditeur Réseau.

Pour les sections comportant plus d'un canton, le déplacement d'un canton numéroté inférieur vers un canton numéroté supérieur est un déplacement AVANT. En d'autres termes, passer d'un canton situé plus haut dans la liste à un canton situé plus bas dans la liste correspond au voyage AVANT. De la même façon, le déplacement INVERSE passe d'un canton numéroté supérieur à un canton numéroté inférieur. Pour les sections avec un seul canton, il n'est pas important de savoir quelle direction dans la section est désignée AVANT, à condition que les directions de déplacement du point d'entrée soient cohérentes. C'est:

  • Les directions de déplacement INVERSE et AVANT doivent être opposées.
  • L'entrée à n'importe quel point d'entrée AVANT doit entraîner le déplacement dans le même sens que la saisie à tous les autres points d'entrée AVANT.
  • L'entrée à tout point d'entrée INVERSE doit entraîner le déplacement dans le même sens que la saisie à tous les autres points d'entrée INVERSE.

AVERTISSEMENT: Si les points d'entrée ne sont pas cohérents, un comportement étrange peut survenir lors de l'utilisation de la section..

Outils du Tableau de Section

Le Tableau des Sections comporte un menu Outils pour faciliter l'utilisation des Sections. Normalement, ces outils seraient utilisés une fois toutes les Sections définies. Les éléments du menu et leur utilisation sont décrits ci-dessous:

  • Valider toutes les Sections ... - Lorsque cet élément est sélectionné, chaque Section est testée pour sa cohérence. Les tests actuels incluent la vérification de la connectivité entre plusieurs Cantons, la vérification de l'absence de points d'entrée ou de points d'entrée supplémentaires et, si un panneau Éditeur de réseau est disponible, la vérification que les Chemins d'accès sont correctement configurés pour les Cantons dans les Sections. Cet outil ne vérifie pas si les directions AVANT et INVERSE sont correctement définies pour chaque point d'entrée, mais cette fonctionnalité est un ajout futur prévu. Une fois les tests terminés, un message indiquant les résultats des tests s'affiche. Les messages d'erreur détaillés, le cas échéant, sont placés dans le journal des erreurs JMRI.
  • Définir les Capteurs de direction dans la Logique ... (non requis si vous utilisez la logique Mât de Signalisation) - Lorsque cet élément est sélectionné, les Capteurs de direction définis pour chaque section sont placés dans la logique de signal des Signaux appropriées - ces Signaux responsables internes de la Section ou en protégeant l'entrée dans la section. Les Capteurs de direction suivent l'état d'attribution de leurs sections. Lorsqu'une section est LIBRE (non attribuée), ses deux capteurs de direction sont activés. En plaçant ces capteurs dans une logique de signal appropriée, tous les signaux internes ou protégeant les points d'entrée de la Section sont forcés au ROUGE lorsque la section est LIBRE. Lorsqu'une section est affectée au déplacement dans le sens AVANT, son Capteur de sens avant est défini sur INACTIF, ce qui permet à la logique de signalisation de fonctionner normalement pour le passage dans la section dans le sens AVANT, tout en maintenant les signaux régissant le déplacement dans le sens INVERSE défini sur ROUGE. De même, lorsque la section est affectée au déplacement dans le sens INVERSE, le capteur de sens INVERSE est défini sur INACTIF et le capteur de sens direct est sur ACTIf, permettant ainsi aux signaux de libérer le trajet dans le sens INVERSE, mais de bloquer le déplacement dans le sens AVANT. Le réglage de l'état des Capteurs de direction est effectué automatiquement par le logiciel dans Répartiteur qui alloue les sections.

    Remarque: Dispatcher peut placer les capteurs de direction dans la logique de signalisation SSL automatiquement lorsqu'un train est activé. Dispatcher placera des capteurs de direction s'il est doté d'un panneau Editeur de réseau et qu'il est autorisé à utiliser sa connectivité (requis pour un fonctionnement automatique). Cela permet d'utiliser la signalisation de direction même si le panneau est enregistré avec ces capteurs retirés de la logique du signal.

    Cet outil ne doit être utilisé qu'une fois que toutes les sections ont été définies et que toute la logique de signalisation a été configurée. Un panneau Éditeur de Réseau est requis, car le logiciel a besoin de la connectivité du panneau pour déterminer quels signaux ont besoin d'un capteur de direction. Si un capteur de direction est déjà dans la logique de signalisation simple (SSL) pour un signal, il n'est pas dupliqué. Cependant, si un capteur de direction incorrect est présent dans le protocole SSL, il n'est pas supprimé. (Utilisez l'outil décrit ci-dessous pour retirer les capteurs de direction.) Une fois les capteurs de direction placés, un message indiquant les résultats s'affiche. Les messages d'erreur détaillés, le cas échéant, sont placés dans le journal des erreurs JMRI. Lorsqu'une erreur survient, les raisons les plus courantes sont 1) des informations manquantes (signaux manquants ou logique de signal non configurée) ou 2) des erreurs de configuration des directions de point d'entrée de section.

  • Supprimer les capteurs de direction de la logique ... (non requis si vous utilisez la logique Mât de Signalisation) - Cet outil assemble une liste de tous les capteurs de direction définis dans les sections se trouvant actuellement dans le tableau des sections, puis supprime tous les capteurs qui sont contenus dans cette liste à partir de la logique de signalisation simple (SSL) de toutes les signaux. Cet outil annule l'effet de l'outil Définir les capteurs de direction dans Logic ..., supprime les capteurs de direction éventuellement placés par Dispatcher., et supprime en outre tous Les capteurs de direction définies pouvant se trouver dans un SSL incorrect. Si des erreurs ont été constatées lors de la configuration des capteurs de direction à l'aide de l'outil ci-dessus, il est parfois utile de l'exécuter avant de tenter de configurer à nouveau les capteurs de direction.
  • Un panneau Éditeur de Réseau est requis pour exécuter cet outil. Une fois que les capteurs de sens ont été supprimés de la Logique Signal, un message indiquant les résultats de l'exécution de l'outil s'affiche. Les messages d'erreur détaillés, le cas échéant, sont placés dans le journal des erreurs JMRI.

Capteurs de Direction et d'Arrêt

Facultativement, chaque section peut avoir deux capteurs de direction, un pour le sens direct et un pour le sens inverse. Ce sont normalement des capteurs internes, portant des Noms Systèmes commençant par IS. Ils suivent automatiquement l'état de leur section. Une section a trois états: LIBRE (non attribué à un train), AVANT (attribué au déplacement aller-retour) et INVERSE ( Lorsque le statut de la section est LIBRE, les deux capteurs de sens sont définis sur ACTIF. Lorsque l'état de la section est AVANT , le capteur de sens avant est INACTIF et le capteur de sens inverse est b> ACTIF. De même, lorsque l'état de la section est REVERSE, la le capteur de sens avant est ACTIF et le capteur de sens inverse est INACTIF. Ces capteurs peuvent être utilisés dans la logique de signal pour forcer les signaux à se déplacer dans le sens opposé au sens attribué à < b> ROUGE (Arrêt). Des outils sont fournis dans le menu Tableau des Sections pour ajouter automatiquement des capteurs de direction à la logique du signal ou pour supprimer automatiquement tous les capteurs de la Logique du Mât de Signalisation ( voir ci-dessus ). Si vous utilisez Dispatcher , vous pouvez ajouter des capteurs de direction pour indiquer la logique, le cas échéant.

De plus, chaque section peut également disposer de deux capteurs d'arrêt qui indiquent quand un train de la section a atteint la fin de la section en se déplaçant dans le sens aller ou dans le sens inverse. Les capteurs d'arrêt doivent être des capteurs physiques de la structure, par exemple des détecteurs de infrarouges. Pour la répartition manuelle, ces capteurs peuvent être affichés sur un panneau pour indiquer au répartiteur quand un train a atteint la fin de son affectation. En mode automatique, ces capteurs peuvent être utilisés pour arrêter un train avant qu'il n'atteigne la fin de la section qui lui est attribuée et avant qu'il dépasse un signal rouge, si des signaux sont présents.

Utilisation de la logique du Mât de Signalisation avec des sections

Si votre signalisation est basée sur la logique du Mât de Signalisation et que la logique associée a été générée à l'aide de l'Éditeur de Réseau, il est également possible d'avoir des sections automatiquement générées sur la base de cette même logique.

L'outil pour accomplir se trouve dans la table logique du Mât de Signalisation | Outils | Générer des sections. L'exécuter créera immédiatement des sections contenant chacune tous les cantons entre toutes les paires de Mât de Signalisation déjà définies. Il configurera également chaque section "générée automatiquement" de manière à inclure ces cantons dans le bon ordre, avec la direction définie comme trajet AVANT, allant de la source au Mât de Signalisation de destination. Veuillez noter que les détails du cantonnement ne sont pas enregistrés lors de l'enregistrement du Panneau, mais sont recréés lors du chargement du fichier.

L'outil créera également automatiquement des capteurs dans le sens aller et le sens inverse et les placera dans la section. Toutefois, la logique du Mât de Signalisation n'utilise pas ces capteurs et leur création n'est là que pour permettre la validation de la section (ou utiliser par l'utilisateur dans Logix non apparenté si désiré)).

Les sections générées avec l'outil auront uniquement une paire de points d'entrée et suivront simplement le chemin entre les deux signaux.

Les capteurs de direction (avant et arrière) sont créés par l'outil Générer des Sections mais, si vous le souhaitez, l'utilisateur doit ajouter manuellement les capteurs d'arrêt (avant et arrière) via le volet Modifier les Sections qui s'ouvre lorsque vous cliquez sur le bouton Modifier pour une section spécifique dans le Tableau des Sections.

Les détails des capteurs facultatifs de direction et d'arrêt sont enregistrés lors de l'enregistrement du fichier du panneau.

Pour utiliser les sections ainsi construites, vous devez ensuite configurer les Transits . Ensuite, vous devrez également configurer le Dispatcher pour qu'il utilise la logique Mât de Signalisation via le Dispatcher Menu Options .

Enregistrement de Sections sur le Disque

Utilisez l'élément Enregistrer dans le menu Fichier du tableau des Sections (ou l'élément Sauvegarder les panneaux ... dans le . Panneaux de la fenêtre principale JMRI) pour enregistrer les informations de la section sur le disque. Ces informations (ainsi que toutes les informations modifiées sur la logique du Mât de Signalisation) sont enregistrées avec la configuration et les panneaux dans un fichier XML, avec la configuration du panneau de configuration, etc. Pour stocker vos panneaux dans le même fichier (recommandé), sélectionnez Fichier> Enregistrer> Enregistrer la configuration et les panneaux dans le fichier ... ou sélectionnez Panneaux> Enregistrer les panneaux ... dans la fenêtre principale JMRI.

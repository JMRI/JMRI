Ajout de Mâts de Signalisation

Les Mâts de Signalisation sont configurés à l´aide de l´un des systèmes de signalisation disponibles et les types de Mâts de Signalisation pris en charge dans ce système.

Dans le panneau Ajouter/Modifier un Mâts de Signalisation , après avoir entré un Nom utilisateur pour le nouveau Mâts de Signalisation, dans le menu contextuel, sélectionnez le système de signal JMRI sur lequel le Mâts de Signalisation doit être basé.
Sélectionnez ensuite l´un des modèles de Mâts de Signalisation disponibles dans le système de signalisation sélectionné, par ex. "Signal à 3 Aspects avec Appel Activé".
Le troisième menu du panneau Ajouter un Mâts de Signalisation vous permet de sélectionner l´un des six types de pilotes de Mâts de Signalisation différents en fonction du matériel utilisé. Plus d´informations à ce sujet dans la section suivante.

Les options communes à (presque) tous les types de Pilotes incluent:

  • Ce Mât peut être É teint : cocher cette case permet à JMRI d´éteindre le mât, en utilisant une instruction supplémentaire comme un état de sortie. Lors de la configuration d´un mât pouvant être éteint, le même identifiant d´aspect doit être entré dans le champ en hautdans la ligne Aspect ci-dessous pour que l´apparence correcte soit affichée dans tous les scénarios possibles.
  • Désactivé : cocher cette case pour un Aspect va griser les options et faire sauter JMRI lors du changement d´état de ce mât
  • Copier les aspects du mât : les pilotes DCC, LNCP et Matrix vous permettent de copier les paramètres d´un nouveau mât pour correspondre à un mât déjà configuré. Bien sûr, le Nom Utilisateur et les sorties doivent être uniques, ils ne sont donc pas copiés.

Lorsque vous êtes prêt, cliquez sur Créer un nouveau (ou Appliquer lors de la modification d´une configuration de mât existante).

Types de pilotes de Mâts de Signalisation

  • Feu de Signalisation contrôlée
  • Aiguillage contrôlé
  • Décodeur DCC
  • Décodeur LNCP
  • Matrice de sortie
  • Virtuel
Feu de Signalisation contrôlé

Les objets Feu de Signalisation individuels sont utilisés pour représenter chaque Feu de Signalisation sur le Mât de Signalisation.
Chaque objet Feu de Signalisation s´occupera ensuite de gérer le changement de son apparence en fonction de l´état du Mâts de Signalisation et de la position de ce Feu de Signalisation par rapport aux autres Feux.
Pour chaque Feu qui a été défini dans le système de signalisation pour le type de Mâts de Signalisation sélectionné, une liste déroulante apparaîtra pour vous permettre de sélectionner le Feu de Signalisation JMRI qui contrôlera ce Feu sur ce Mât de Signalisation.
Aiguillage Contrôlé

Ce type de pilote est destiné à un matériel spécifique qui peut contrôler l´apparence complète d´un Mât de Signalisation en fonction du réglage d´un ou de plusieurs Aiguillage .
Chaque aspect peut être configuré pour envoyer l´état Dévié ou Direct de l´Aiguillage.
Laissez la case Réinitialiser l´Aspect Précédent décochée si l´événement Aiguillage efface également les aspects précédents sur un module distant.
Virtuel

Ce mât ne contrôle directement aucun matériel lié à un mât de signalisation.
Il est idéal pour les endroits où un mât physique ne serait pas présent sur la voie ferrée, comme un butoir, ou pour marquer la fin d´une section de signalisation de voie et le début de Territoire avec la signalisation éteinte
Aucune commande n´est envoyée au réseau, il n´y a donc pas de connexion à choisir.
Décodeur de Mât de Signalisation DCC

Ce type de pilote est destiné à un décodeur accessoire qui peut piloter nativement les mâts de signal et prend en charge le NMRA Extended Accessory Protocol S-9.2.1 .
Entrez les nombres d´aspect configurés dans votre décodeur pour chaque aspect. Les valeurs par défaut de l´ID de chaque aspect tel que défini dans la définition du système de signalisation peuvent être écrasées.
Les Décodeurs comme le Signalist SC1 , ont les types de Mâts de Signalisation préconfigurés et il vous suffit de sélectionner le bon tableau avec un CV et d´obtenir les numéros d´aspect du manuel.
Décodeur de Mâts de Signalisation LNCP
Comme l´option précédente, le type de pilote de point de contrôle LocoNet (LNCP) n´a besoin que d´un ID de décodeur et éventuellement d´ID d´aspect pour piloter un Module RR_Cirkits LNCP .
Matrice de sortie
depuis 4.6
Ce pilote crée une table de vérité pour définir de 1 à 6 sorties par aspect. Pour définir 8 modèles différents, vous aurez besoin d´au moins 3 champs on/off.
Le pilote Output Matrix a été développé pour connecter des décodeurs configurés pour connecter et piloter des signaux, comme un Digirails DR4018 avec mappage d´aspect intégré défini pour correspondre à votre système de signaux JMRI et à votre mappage.
Lorsqu´il est sélectionné dans la liste déroulante Pilote, le panneau vous permet:
  1. Définir le nombre de sorties logiques (3 dans la capture d´écran), sélectionnez dans la liste déroulante le Nombre de sorties logiques ).
    Le panneau est redessiné lorsque vous modifiez ce nombre, affichant autant de ...
  2. Boîtes de sélection de sortie. Sélectionnez ou créez des aiguillages en tant que sorties binaires.
  3. Pour chaque aspect, les cases à cocher vous permettent de définir la position correspondante des sorties, d´où le nom de la matrice
  4. depuis 4.15.7 Définit un délai spécifique au mât qui est appliqué entre chaque commande de sortie
Pour l´instant, seuls les aiguillages peuvent être configurés comme sorties. La documentation de votre décodeur devra expliquer quels aspects sont disponibles et quels états de sortie le décodeur attend pour chaque aspect.
Les noms de sortie et la ligne Non éclairé peuvent être modifiés ultérieurement en cliquant sur le bouton Modifier dans le tableau Mâts de Signalisation. Il n´est pas possible de modifier le nombre de sorties dans la matrice une fois qu´un mât a été enregistré.

Cliquez sur Créer un nouveau pour ajouter le nouveau mât et revenir à la table des Mâts de Signalisation.

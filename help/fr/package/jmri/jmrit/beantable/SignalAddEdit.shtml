Ajout/Edition des Signaux

Le panneau "Édition Signal" permet de visualiser et de modifier les informations facultatives des Signaux existants. Utilisez le bouton "Modifier" de la Table des Signaux pour ouvrir une boîte de dialogue pour ce Signal. Un seul Signal peut être modifiée à la fois. Le type de Signal et le Système de Signalisation ne peuvent plus être modifié s une fois qu'ils ont été créés. Une fois les modifications apportées dans la boîte de dialogue, cliquez sur le bouton "Mettre à jour" en bas du panneau pour modifier les informations du Signal. Cliquez sur "Annuler" pour quitter sans apporter de modifications. Fermer le panneau Modifier Signal revient à cliquer sur "Annuler".

La boîte de dialogue "Ajouter un Nouveau Signal" s'ouvre en cliquant sur le bouton "Ajouter ..." en bas du tableau des Signaux. Elle vous permet de choisir un type de connexion de Signaux dans la liste suivante.

Types de Connexion de Signal Disponibles

Virtuel
Ce type ne se connecte pas du tout au réseau. Utilisez-le lorsque vous souhaitez par exemple afficher un signal sur un panneau, mais n'en avez pas un réel sur la réseau.
Acela
Ce type contrôle les signaux reliés à un CTI Electronics Module Acela SignalMan.

Dans le champ de texte, entrez un Nom Système pour un signal Acela, par ex. AH01. Pour plus d'informations sur le matériel Acela, consultez la page d'aide sur le matériel Acela.

Décodeur de Signal DCC
Ce type contrôle les Signaux attachés à un décodeur qui utilise les paquets de signaux DCC définis par le groupe de travail NMRA DCC. pour le contrôle. (Vous pourrez peut-être trouver des informations sur ces paquets DCC à la page 9 à l'adresse Il est difficile de se connecter au site Web de NMRA Si cela ne fonctionne pas, recherchez "Format de Paquet de Commande de Décodeur d'Accessoire Étendu NMRA 9.2.1"). Il s'agit d'un protocole spécial défini pour piloter les Signaux,et il diffère de la manière dont les "décodeurs d'accessoires" (par exemple, les sorties d'aiguillage) sont contrôlés.

Entrez l'adresse DCC sous forme de nombre. La case à cocher Adresse de décalage utilise une autre interprétation du schéma d'adressage éventuellement utilisé.
Chaque numéro possible du Signal se voit attribuer un numéro différent; à partir de là, vous pouvez attribuer des numéros différents pour chaque aspect en fonction du matériel utilisé.
Le Décodeur de Signal Team Digital SHD2 est un décodeur de signal utilisant une adresse différente pour chaque signal. Un mât de signaux connecté au SHD2 doit être configuré comme un ou plusieurs mâts contrôlés par Signal Head utilisant le type de connexion de signal DCC. Les instructions sont disponibles sur le site Web de Team Digital sous l' Applications > Menu SHD2.
Remarque: le Signalist SC1 peut également être configuré en tant que signalisation individuelles (projecteur à LED bicolores, tricolores ou RVB ou multi-lampes à LED individuelles), mais son utilisation est beaucoup plus simple lorsqu'elle est configurée en tant que mât de signal complet avec tous les signaux sur une adresse DCC unique à l'aide du pilote DCC Signal Mast. Dans ce cas, vous n'avez pas besoin d'ajouter de signal pour votre signalisation. La principale différence entre un signal et un mât de signal est le nombre d'états diffé rents que chacun pourrait avoir. Un signal a seulement (jusqu'à) 8 entrées d'apparence, mais un mât de signal peut avoir jusqu'à 32 états possibles (0 à 31), appelés Aspects dans ce cas. La commande qui sort est la même mais le nombre de valeurs possibles diffère.

Sortie unique
Ce type de connexion commande un Signal avec une sortie. Une connexion Signal qui utilise une seule ligne de sortie pour indiquer jusqu'à 2 apparences plus le clignotant. Vous pouvez sélectionner 2 apparences dans la liste suivante: Éteint, Rouge, Jaune, Vert et Lunaire. Le codage de sortie est tel qu'un état d'aiguillage Dévié contrôle l'apparence et Droit régule l'autre. Le clignotement permet une troisième apparence qui clignotera entre les deux apparences configurées. La connexion peut être établie avec une seule lampe/LED pour afficher un seul aspect. Une autre méthode consiste à connecter une paire de voyants à la sortie unique câblée, de sorte que lorsque la sortie du décodeur est activée, vous obtenez une couleur et lorsque la sortie est désactivée, vous obtenez l'autre. Ce type pourrait également être utilisé pour contrôler des signaux à commande mécanique qui n'ont que deux apparences et ont donc toujours une apparence. Il est possible de gagner une troisième apparence "clignotante". Si vous avez sélectionné Éteint et une couleur, le résultat final est une option de couleur Flash supplémentaire. Si deux couleurs ont été sélectionnées, une option Flash vous est proposée, mais les couleurs clignotent alternativement. Ces connexions sont contrôlées via la logique d'aiguillage JMRI et apparaissent donc dans le Tableau Aiguillage. Ils ne sont pas connectés pour commuter des moteurs, cependant, ce sont juste des sorties électriques.

Pour chaque connexion, entrez l'Aiguillage JMRI approprié pour la connexion électrique sur le réseau, soit dans la liste déroulante des Aiguillages déjà définis, soit en créant un nouvel Aiguillage pour une connexion particulière.

Sélectionnez l'une des apparences répertoriées pour les deux:
Apparence à l'état Dévié - Apparence affichée lorsque la connexion est à l'état Dévié.
Apparence à l'état Droit: apparence affichée lorsque la connexion est à l'état Droit.

Double sortie
Ce type commande un signal via deux connexions électriques. L'une va à un voyant/led Vert, et l'autre à un rouge. Le jaune est créé en les allumant tous les deux en même temps.

Pour chaque connexion, entrez l'aiguillage JMRI approprié pour la connexion électrique sur le réseau, soit dans la liste déroulante des Aiguillages déjà définis, soit en créant un nouveal Aiguillage pour une connexion particulière. Ils ne doivent pas nécessairement être consécutifs ou dans un ordre particulier.

Double tête de sortie
Une connexion Signal qui utilise seulement deux lignes de sortie pour indiquer 4 aspects de signal.
Généralement, ces aspects sont les suivants: Libre, Approche, Arrêt et Éteint. Le codage de sortie est tel qu'un aiguillage Déviée commande l'arrêt et l'autre Dévié commande Libre. Les deux sorties Déviée contrôlent l'approche et les deux Droits sont Éteints. Chaque changement d'aspect nécessite l'envoi de deux commandes d'aiguillage.
Sortie triple
Ce type de contrôle d'un signal via trois connexions électriques. Une va à une lampe/LED verte, une autre à une jaune, et la troisième à une rouge.
Ces connexions sont commandées via la logique d'aiguillage JMRI et donc apparaître dans le Tableau Aiguillage. Elles ne sont pas vraiment liées à la commutation des moteurs, même si, elles sont simplement des sorties électriques.

Pour chaque connexion, entrez le branchement JMRI approprié pour la connexion électrique sur le réseau, soit dans la liste déroulante des aiguillages déjà définis, soit en créant un nouvel aiguillage pour une connexion particulière. Ils n'ont pas besoin d'être consécutifs ni dans un ordre particulier.
Si vous avez besoin de déboguer l'un de ces signaux, commencez par vérifier si les connexions Aiguillage fonctionnent.

Triple Sortie RVB
Ce type commande un Signal via trois sorties pour indiquer 5 aspects de signal. Généralement, ces aspects sont Libre, Approche, Arrêt, Restriction et Éteint.
Le codage de sortie est tel qu'un Aiguillage Dévié contrôle l'Arrêt et un autre Dévié (On) contrôle Libre. Deux de ces sorties Déviés (On) contrôlent l'Approche, les trois Déviés contrôlent Éteint. Chaque changement d'aspect nécessite l'envoi de trois commandes.
La première connexion alimente une lampe/LED verte, la seconde une lampe rouge et la troisiè me une lampe bleue. Ils sont contrôlés via la logique d'Aiguillage JMRI et apparaissent donc dans le Tableau Aiguillage. Ils ne sont pas liées à la commutation des moteurs, cependant, ce sont juste des sorties électriques.

Pour chaque connexion, entrez l'Aiguillage JMRI approprié pour la connexion électrique sur le réseau, soit dans la liste déroulante des Aiguillages déjà définis, soit en créant un nouvel Aiguillage pour une connexion particulière. Ils ne doivent pas nécessairement être consécutifs ou dans un ordre particulier.

sortie Quadruple
Ce type commande d'un signal via quatre connexions électriques. On va à une lampe /LED verte, à une jaune, à une rouge, et le quatrième à une "lunaire" (blanc).
Ces connexions sont commandées via la logique aiguillage JMRI, et vont donc apparaître dans le Tableau Aiguillage. Elles ne sont pas vraiment liées à la commutation des moteurs, même si, elles sont simplement des sorties électriques.

Pour chaque connexion, entrez l'adresse de l'Aiguillage JMRI appropriée pour la connexion électrique sur le réseau, soit dans la liste déroulante des aiguillages déjà définis, soit en créant un nouvel aiguillage pour une connexion particulière. Ils ne doivent pas nécessairement être consécutifs ou dans un ordre particulier.

SE8c 4 aspects
La sortie est contrôlée par une seule commande d'aiguillage par changement d'aspect. Une commande d'aiguillage change entre "Arrêt" et "Libre", et la deuxième commande d'aiguille change entre "Approche" et "Éteint".

Ce type contrôle les signaux attachés à une Digitrax Carte de Pilote de Signal SE8C.

Etant donné que chaque signal Aspect SE8C 4 fonctionne avec une adresse DCC spécifique, son nom de système est déterminé automatiquement à partir de cette adresse; vous n'êtes pas obligé d'y entrer. Par exemple, le signal SE8C 4 Aspect pour l'adresse matérielle 257 se verra attribuer un nom de système basé sur celui-ci. Voir pages 5 à 7 du Manuel Digitrax SE8C.

Vous pouvez éventuellement entrer un Nom Utilisateur plus lisible si vous le souhaitez.

Dans le champ "Numéro d'Aiguillage", entrez l'adresse DCC appropriée de la sortie SE8C sous forme de nombre. Par exemple, le 1ère signal de la 1ère SE8C, telle que configurée à l'usine Digitrax, est "257". Sélectionnez "Créer nouveau", puis sélectionnez votre connexion de réseau (généralement "LocoNet") dans la liste déroulante, puis entrez "257" dans le champ à côté. Faites la même chose dans le second champ Aiguillage, en entrant "258" comme deuxième adresse de Signal.

Pour des utilisations plus avancées, par exemple si vous utilisez plusieurs types de matériel de réseau, vous pouvez soit sélectionner un Aiguillage dans la liste déroulante des interventions déjà définies, soit en créant un nouvel Aiguillage pour une connexion particulière.

LDT LS-DEC
Ce type est particuliè rement utile pour la signalisation européenne. Il contrôle les signaux reliés à un Littfinski DatenTechnik (LDT) et Décodeur Signal LS-DEC , ou un décodeur DP2N modèle Marathon.

Ces contrôleurs utilisent sept adresses DCC d'Aiguillage (accessoire décodeur) pour spécifier les adresses des sept Apparences possibles du Signal. Entrez celles-ci, soit à partir de la liste déroulante des Aiguillages déjà définis ou en créant un nouvel aiguillage pour une connexion particulière. Ils n'ont pas besoin d'être consécutives ou dans un ordre particulier

Pour plus d'informations sur la configuration de ces signaux, voir la page exemples.

Pilote de Signal MERG
Ce type de pilote contrôle un Signal utilisant un Pilote de Signal MERG (SD2). Un SD2 est connecté au DCC à l'aide d'un décodeur d'accessoires à état stable. Le nombre d'aspects à afficher sur le Mât de Signalisation déterminera le nombre de sorties Aiguillage requises, par exemple.
2 aspects: nécessite 1 Aiguillage (photo de droite)
3 aspects: nécessite 2 Aiguillages
4 aspects: nécessite 3 Aiguillages

Le Pilote de Signal MERG dans JMRI a été configuré pour contrôler uniquement la sortie Aspect SD2. Il ne contrôle pas les autres fonctionnalités du SD2 telles que "Règles Vapeur" ou "Feather", mais elles peuvent facilement être configurées à l'aide de Logix ou de Itinéraires.
Remarques - Le pilote MERG SD2 ne prend pas en charge le format jaune clignotant. Pour les aspects jaune clignotant, un circuit supplé mentaire est requis, comme indiqué dans les Bulletins techniques MERG.
Comme le pilote MERG SD2 dépasse les aspects Vert/Jaune/Rouge pris en charge dans JMRI avec un aspect Double jaune (jaune sur jaune), l'état Double jaune est représenté dans l'é tat par "Lunar".
Pour plus d'informations sur le pilote de signal Merg, consultez la page MERG

Grapevine
Ce type contrôle les Signaux attachées à un ProTrak nœud Grapevine.
Dans le champ de texte, entrez un Nom Système pour un signal Grapevine, par ex. GH1204 pour le 4ème connecteur sur le 2ème connecteur de banque du 1er noeud. Pour plus d'informations sur la manière dont ces noms sont générés, voir la page sur les noms de Gravepine.

Notes

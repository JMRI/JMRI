JMRI: Fenêtre Préférences

La fenêtre de préférences vous configure le fonctionnement de base de votre application JMRI.

L'écran est divisé en deux sections, sur le côté gauche il y a une liste des groupes de préférence qui peut être réglé. Cliquer sur chaque préférence va ensuite remplir le côté droit, avec les différentes options qui peuvent être configurées.

Sans aucun doute la commande la plus importante se trouve au bas de cette section: Le bouton Enregistrer!

Soyez sûr de cliquer sur "Enregistrer" lorsque vous quittez cette fenêtre. Vos modifications des préférences ne prennent pas effet avant que le programme soit arrêté et redémarré, de sorte que le programme vous invite à le faire après avoir sauvé les Préférences.

Connexions

Vous pouvez sélectionner l'une des diverses connexions réseau JMRI ici.

Chaque connexion a son propre onglet unique. Pour commencer, choisissez d'abord le fabricant de votre système DCC, suivi par le type de connexion que vous utilisez. Il vous sera alors présenté avec une série de paramètres les plus souvent requis qui doivent être fixés. Si vous devez modifier certains des paramètres les plus avancés, sélectionnez connexion, puis cochez la case "Connexion Paramètres supplémentaires" et cela va vous fournir de plus amples détails.

Pour ajouter une connexion réseau supplémentaire cliquez sur l'onglet avec un "+", ce sera alors un onglet connexion raffraichi. Pour supprimer une connexion, dans la liste déroulante des frabricants du système, sélectionnez (aucun sélectionnés) pour effacer simplement vos choix actuels et en entrer de nouveaux, ou cliquez sur le petit "x" dans le nom de l'onglet pour supprimer la connexion.

Pour plus d'informations sur comment configurer cela pour votre matériel, consultez les pages d'Aide Matériels .

Rappelez-vous de presser Enregistrer!

Defaults

Si vous utilisez plusieurs connexions, par exemple une connexion LocoNet et une connexion MERG, le volet Par défaut vous permet de sélectionner la connexion que vous souhaitez utiliser à une fin particulière.

La connexion interne est la connexion par défaut. Cependant, comme cela est presque toujours indésirable, les applications JMRI vous avertiront si la connexion interne est la connexion par défaut pour chaque objectif. Pour désactiver cette notification, modifiez les propriétés du profil en définissant la propriété jmri-managers.allInternalDefaults sur true .

Démarrage

Les préférences Démarrer permettent d'ouvrir des fenêtres et des fichiers, d'exécuter des scripts et de placer des boutons supplémentaires dans la fenêtre principale au démarrage de l'application.

Cliquez sur le bouton Ajouter ▾, puis sélectionnez le type d'action de démarrage à ajouter. La nouvelle action sera ajoutée en dernier.

L'ordre d'action au démarrage est important. Dans la plupart des cas, les scripts doivent être les derniers éléments de l'ordre de démarrage. L'ordre de démarrage peut être modifié en sélectionnant un élément de démarrage et en le déplaçant vers le haut ou vers le bas dans la liste.

Il existe cinq types d'actions de démarrage:

Exécuter l'action
Demandez au programme d'exécuter automatiquement divers menus sélections au démarrage. Par exemple, vous pouvez avoir le Tableau Aiguillages qui s'ouvre automatiquement ou ouvre automatiquement une fenêtre pour programmer sur le réseau (P.O.M).
Bouton Ajouter à la fenêtre principale
Ajoutez des boutons supplémentaires à la fenêtre principale JMRI qui sont connectés à des actions spécifiques dans les menus pour un accès facile et rapide à cette action.
Ouvrir le fichier
Charger automatiquement les fichiers du panneau au démarrage du programme. Vous pouvez charger autant de fichiers de panneau que vous le souhaitez; chacun sera complètement chargé avant que le suivant ne soit lancé.
Exécuter le script
Exécuter automatiquement un script au démarrage du programme. Vous pouvez exécuter autant de scripts que vous le souhaitez; chacun sera exécuté avant le prochain démarrage. Notez que tous les fichiers dont dépend le script doivent être ouverts avant de l'exécuter.
Définir l'itinéraire
Définissez un itinéraire au démarrage du programme. Vous pouvez définir autant d'itinéraires que vous le souhaitez; chacun sera mis en ordre. Notez que tous les fichiers dont l'itinéraire dépend doivent être ouverts avant de définir l'itinéraire.

Affichage

Interface Graphique Utilisateur (GUI)
Une rangée de boîtes radio vous permettent de sélectionner la façon dont le programme va être afficher. Les choix disponibles dépendent du type d'ordinateur que vous avez, et tous les éléments suivants peuvent être disponibles:
  • Métal - le style par défaut de Java, utilisable sur tous les ordinateurs
  • Theme/CDE - Une forme Linux/Unix qui fonctionne bien avec les gestionnaires écran x Windows. Il est également disponible sur (presque) tous les types d'ordinateurs.
  • Windows - uniquement disponible sur les systèmes Microsoft Windows, ce qui rend l'apparence du programme, autant que possible comme une application normale de Windows.
  • Mac OS X - disponible uniquement sur MacOS, ce qui rend l'apparence du programme comme une application Macintosh

De plus, cet onglet vous permet de définir:
    La taille de la police de l'interface graphique. Indiquez si vous souhaitez utiliser des graphiques pour afficher l'état des aiguillages, des capteurs et des cantons dans la colonne d'état de leurs tableaux depuis la version 4.7.4
  • La durée d'affichage des info-bulles (le cas échéant).
Locale
Beaucoup de gens ont contribué à l'"Internationalisation" de JMRI. Ceux-ci ont convertit (beaucoup) de texte à l'écran dans une langue autres que l'Anglais. Si vous sélectionnez votre pays ou votre région et votre langue ici, puis enregistrez et redémarrez les préférences, JMRI va utiliser une internationalisation pour ce pays et la langue si elle est disponible. Les éléments qui n'ont pas été traduits continueront à paraître en Anglais, de sorte que vous ne manquerez rien. Pour plus d'informations sur la façon de le mettre en place, et en particulier la façon de contribuer à des traductions nouvelles et les améliorées, s'il vous plaît voir notre Page d'Internationalisation.
Console
Ceci vous permet de personnaliser l'apparence de la console système JMRI. Il est possible de choisir parmi plusieurs schémas de couleurs prédéfinis et de choisir la taille et le style de police. Vous pouvez également choisir d'enregistrer la taille et l'emplacement de la fenêtre de la Console Système afin qu'elle apparaisse toujours au même endroit et à la même taille.

Messages

Cela vous permet de changer la façon dont les messages d'information et d'alerte sont affichés dans JMRI.
Il vous permettra également de configurer une des réponses par défaut et la question affichée.

Inventaire

Programmeur
Le programmeur DecoderPro vous permet de choisir (beaucoup ou peu) quelle part du contenu sera afficher en utilisant un "programmeur" pour un décodeur "configurable". Vous pouvez choisir l'interface Basique ou Complète quand vous sélectionnez votre décodeur de locomotive mais ceci vous offre un endroit pratique pour en définir un par défaut que vous n'aurez pas à sélectionner à chaque fois que vous ouvrez un décodeur.

Cette section contient également des cases à cocher pour les options:

    "Afficher les onglets de programmeur vides": Ceci vous permet de décider si le programmeur DecoderPro affichera des volets (onglets) vides. Certaines personnes aiment les voir, même s'ils sont vides; certaines personnes préfèrent ne pas les voir, alors elles ne continuent pas à essayer de les ouvrir et de les regarder. Nous fournissons cette préférence afin que vous puissiez choisir. "Afficher les numéros de CV dans les info-bulles": si vous autorisez le curseur de la souris à "survoler" une valeur dans DecoderPro, une info-bulle apparaît avec plus d'informations. Cochez cette case pour inclure le numéro de CV dans cette info-bulle. Certaines personnes veulent connaître le numéro de CV, d'autres veulent que DecoderPro traite les CV de manière totalement invisible, nous vous proposons donc une option vous permettant de choisir. "Autoriser la mise en cache lors de l'écriture de CV d'index pour des opérations de lecture ou d'écriture": certains décodeurs exigent que des valeurs spécifiques soient écrites sur des CV "d'index" avant que certains autres emplacements puissent être lus ou écrits. Les décodeurs QSI et ESU l'utilisent beaucoup. Lorsque vous effectuez beaucoup de lectures ou d'écritures, par exemple un décodeur complet, les CV d'index peuvent finir par être écrits avec la même valeur plusieurs fois de suite. Lorsque cette option est sélectionnée, DecoderPro permet d'ignorer (gagner du temps sur) les écritures de la même valeur d'index qui sont par ailleurs redondantes. N'utilisez pas cette option (ou utilisez-la uniquement en combinaison avec l'opération suivante) si vous rencontrez des problèmes de fiabilité lors de l'écriture des décodeurs, car le bon fonctionnement de cette option dépend d'une écriture fiable.

    Plus de détails: une seule mauvaise écriture avec cette option désactive un CV; écrire un mauvais index avec cette option désactive le CV qui était censé être écrit et le mauvais adressé par la mauvaise écriture, mais l'écriture sur le prochain CV de la séquence est OK en raison de la redondance suivante de l'index; écrire un mauvais index avec cette option sur laisse en place cette valeur d'index mal écrite, écrit donc une séquence de mauvaises valeurs de CV. Conclusion: si vous travaillez avec un décodeur compliqué avec beaucoup de CV, vous avez besoin d'une connexion de programmation fiable, voir option suivante. "Si possible, confirme que le CV écrit en suivant ceci avec une lecture": Lorsque cette option est sélectionnée, DecoderPro lira chaque CV après l'avoir écrit. Si la valeur ne correspond pas, Il note que l'écriture a échoué.

    Si vous rencontrez des problèmes intermitents pour écrire dans un décodeur, cela peut améliorer la fiabilité. Si DecoderPro n'est jamais capable d'écrire correctement, cela ne vous aidera pas. Cela ne sert que si le problème est occasionnel, et réessayer va résoudre le problème plus souvent qu'autrement.

    Certains systèmes DCC vérifient la présence d'une " impulsion d'accusé de réception d'écriture" auprès du décodeur avant de déclarer que l'écriture est réussie. Celles-ci sont presque aussi fiables que cela et beaucoup plus rapides. Donc, si vous utilisez l'un de ces systèmes DCC, cela ne sera peut-être pas nécessaire. D'autre part, si vous utilisez un système DCC qui "écrit à l'aveugle", sans contrôle, cela pourrait être vraiment utile. Pour savoir si vous en avez un, essayez de rédiger un CV avec la locomotive en dehors de la voie. Si DecoderPro pense que l'écriture a réussi, le système DCC écrit en aveugle, sans vérification.

    Cela n'aide pas (mais ne fait pas mal) si vous ne pouvez pas relire le décodeur. La plupart des systèmes DCC ne peuvent pas lire pendant la programmation en mode OP, par exemple. Certaines Centrales de commande (Digitrax DB150) ne peuvent pas lire le décodeur. Dans ces cas, cette option demande simplement à la Centrale de commande d'écrire sans relire.

Ces options sont stockées avec les préférences pour une utilisation future. Une fenêtre DecoderPro Roster sélectionnera les nouvelles valeurs des options sans avoir à fermer et à redémarrer le programme.
Inventaire
Normalement, la liste est stockée dans un emplacement standard comme une partie du répertoire Préférences. Cette option vous permet de de déplacer cela ailleurs.

Pour le configurer, cliquez sur le bouton "Définir ..." et suivez les instructions.

Vous pouvez également entrer une valeur par défaut pour le champ "propriétaire" dans les fiches de la Liste DecoderPro ici. Ceci remplit la valeur pour les nouvelles entrées, ce qui peut vous faire économiser un peu de frappe. Elle n'a aucun effet sur les entrées existantes, même si elle change.

Profils de Config

Les Profils de Configuration JMRI rendent aisé le démarrage de vos applications JMRI pour qu'elles travaillent avec différentes configurations. Chaque profil spécifie un jeu de Préférences. Vous pouvez alors sélectionner lequel utiliser.

Pour avoir plus d'information, voir la page d'aide Profils de Configuration.

Nom du Réseau

L'onglet Nom du Réseau vous permet de spécifier le nom de votre réseau miniature ( module ), quand il est utilisé par plusieurs protocoles de serveur.

Serveur JSON

Le Serveur JSON JMRI est utilisé par les applications sur les autres ordinateurs, incluant les tablettes et les smartphone, pour see connecter à JMRI pour agir sur votre réseau

Sur ce volet vous pouvez spécifier le port pour la connexion ( habituellement laissé par défaut ) et La fréquence à laquelle le serveur vérifie si la connexion a échoué via un message.

Serveur Web

le Serveur Web JMRI vous permet de vous connecter à JMRI depuis un serveur web sur votre ordinateur, tablette et smartphone et de travailler avec votre réseau.

Cet onglet vous permet de commander ce que le serveur web fournit. Pour de plus amples information, voir la page panneau Préférences Serveur Web

Serveur SRCP

Le Serveur SRCP JMRI est une façon standard de connexion de programmes utilisateur "client" à un serveur.

Sur ce volet vous pouvez spécifier le port pour la connexion ( habituellement laissé par défaut ).

Serveur Simple

Le Serveur Simple JMRI est une partie de JMRIClient, une méthode de connexion à distance d'un programme JMRI à un autre.

Dans ce volet, vous pouvez spécifier le port de connexion ( habituellement laissé par défaut ).

Consignes

Ce volet configure les paramètres utilisés dans le calcul des itinéraires sous Consignes et pour la commande des événements que les trains sous Consignes peut rencontrer.

Carte de Vitesse

Quand un train en automatique rencontre un aspect de Signal qui appelle à un changement de vitesse, ou que la progression est stoppée par une occupation en avant, la carte de vitesse est utilisée pour faire les changement de vitesse nécessaire. D'autre paramètres commandent la rampe de changement de vitesse.

Tableau de noms de vitesse et aspect

La Carte de Vitesse Cartographie les noms de vitesse trouvés dans la définition des aspects d'un système de signal pour une valeur. Il est important que tous les noms de vitesse utilisés dans votre système de signalisation soient listés dans le tableau Carte de Vitesse. Des lignes peuvent être ajoutées ou supprimées pour se conformer aux aspects du système de signalisation.

La valeur associée à un nom de vitesse peut être interprétée de quatre manières différentes. Quatre boutons radio permettent de choisir la signification de la valeur.

    Pourcentage normal: la valeur correspond au pourcentage du paramètre de régulation actuel enregistré ou défini par une E/S d Consigne. C'est un pourcentage de la vitesse "normale". Pourcentage d'accélérateur: la valeur correspond à un pourcentage du paramètre d'accélération maximale. Miles par heure - la valeur est une vitesse d'échelle absolue en miles par heure. Kilomètres par heure - la valeur est une vitesse d'échelle absolue en kilomètres par heure.
Le choix de l'interprétation dépend de ce que vous voulez faire lorsqu'un train sous Consignes atteint un aspect signal ou bloc. Pourcentage normal, le train changera toujours de vitesse et le pourcentage d'accélération le plus probable. Les deux derniers choix sont les plus prototypiques, mais si le train se trouve déjà dans la limite de vitesse appelée ou aucun changement de vitesse ne se produira.

Modifications de la vitesse de montée en puissance

La régularité du ralentissement et de l'accélération s'effectue en montant et en descendant avec deux paramètres de "rampe"; l'incrément du Régulateur par étape et l'intervalle de temps de chaque étape. Ces deux paramètres sont associés pour créer un motif de "marche d'escalier" permettant de modifier la vitesse. Généralement, ils sont jumelés en ce sens qu'une petite augmentation ddu Régulateur doit être effectuée dans un intervalle de temps court et une augmentation importante dans un temps plus long. Lors de leur réglage, l'intervalle de temps doit être assez long pour que l'incrément de vitesse se termine compte tenu de l'inertie que vous avez programmé dans vos décodeurs.
    Incrément d'accélération par cran de la rampe Cette valeur correspond à un rapport d'accélération maximale. Les sept ou huit vitesses d'un Régulateur à 128 vitesses sont une augmentation appropriée pour une locomotive à inertie modeste. Une valeur de 0,008 à 0,04 est recommandée. (0.0079 correspond à 1 pas d'un Régulateur à 128 pas) Temps de l'étape de la rampe En fonction de la configuration de la quantité de mouvement sur vos locomotives, réglez cette valeur de sorte que le changement de vitesse soit terminé dans ce laps de temps. Correspondant généralement à ce qui précède, 300 à 1000 millisecondes seraient appropriées.
Vous devriez expérimenter en observant le début et la fin de E/S de Consignes ou en émettant des commandes "Arrêter" et "Reprendre" pour une Consigne enregistré.

Facteur d'accélération

La valeur de Paramètre d'accélération/Facteur de vitesse avec Échelle de présentation définit un paramètre d'accélération comme étant une vitesse de piste de présentation. Le facteur d'accélération défini ici est un paramètre global par défaut.

Pour plus d'informations sur les paramètres de vitesse des Consignes, consultez la Modification de la vitesse des mandats et suivez ses liens.

Fusion des profils de vitesse

Dans la version 4.9.2 Consigne calcule la vitesse d'une voie lorsque vous parcourez un trajet de longueur et de réglage de Régulateur connus. Ces mesures sont effectuées pour chaque adresse soumise à une Consigne et sont conservées pendant toute la session de mise en page. Lorsque JMRI est quitté, à l'arrêt. Ces profils de vitesse de session peuvent être fusionnés dans leurs profils de vitesse de répertoire. Les choix suivants sont disponibles à l'arrêt.
    Ne pas fusionner - N'invitez pas les profils de session, ne les supprimez pas et laissez les profils de vitesse de liste inchangés. Fusion instantanée: à l'arrêt, une boîte de dialogue s'affiche, vous permettant de choisir les profils de vitesse à fusionner et ceux à éliminer. La boîte de dialogue vous permet de visualiser les profils pour chaque adresse. Les éléments affichés sont les suivants: profil de vitesse d'Inventaire actuel, le profil de vitesse fusionné potentiel et le profil de Vitesse de Session. (par défaut) Fusionner tout - N'invitez pas et fusionnez toujours tous les Profils de Vitesse de Session dans les profils de vitesse d'Inventaire.

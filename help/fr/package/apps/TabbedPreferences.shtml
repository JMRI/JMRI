JMRI: Fenêtre Préférences

La fenêtre de préférences vous configure le fonctionnement de base de votre application JMRI.

L'écran est divisé en deux sections, sur le côté gauche est une liste des groupes de Préférence qui peut être configurés. Cliquer sur chaque préférence va ensuite remplir le côté droit, avec les différentes options qui peuvent être configurées.

Sans aucun doute la commande la plus importante se trouve au bas de cette section: Le bouton "Enregistrer"!

Soyez sûr de cliquer sur "Enregistrer" lorsque vous quittez cette fenêtre. Vos modifications des préférences ne prennent pas effet avant que le programme soit arrêté et redémarré, de sorte que le programme vous invite à le faire après avoir sauvé les préférences.

Connexions

Vous pouvez sélectionner l'une des diverses connexions JMRI ici.

Chaque connexion a son propre onglet unique, pour commencer, choisissez d'abord le fabricant de votre système DCC, suivi par le type de connexion que vous utilisez. Il vous sera alors présenté avec une série de paramètres les plus souvent requis qui doivent être fixés. Si vous devez modifier certains des paramètres les plus avancés, puis connexion, puis cochez la case "Connexion Paramètres supplémentaires" et cela va vous fournir de plus amples détails.

Pour ajouter une connexion supplémentaire cliquez sur l'onglet avec un "+", ce sera alors un onglet connexion raffraichi. Pour supprimer une connexion, dans la liste déroulante des fabricants du système, sélectionnez (aucun sélectionnés et ceci va enlever la configuration de cette connexion.

Pour plus d'informations sur comment configurer cela pour votre matériel, consultez les pages matériels .

Rappelez-vous de presser Enregistrer!

Démarrage

Démarragepermet aux fichiers et aux fenêtres d'être ouverts, aux scripts d'être exécutés, et à des boutons supplémentaires d'être placés sur la fenêtre principale quand l'application démarre.

cliquez le bouton Add ▾ et sélectionnez le type d'action au démarrage à ajouter. Cette nouvelle action sera ajouter en dernier.

L'ordre des actions au démarrage est important. Dans la plupart des cas, les scripts devraient être les derniers éléments dans l'ordre de démarrage.L'ordre de démarrage peut être modifié en sélectionnant un élément de démarrage et en le déplaÝant vers le haut ou le bas de la liste

Il y a cinq types d'actions au démarrage:

Faire une action
Avoir le programme qui exécute automatiquement diverses sélections de menus. Par exemple vous pouvez ouvrir automatiquement la Table Aiguillage, ou ouvrir automatiquement une fenêtre pour la programmation en voie principale( POM)
Ajouter un bouton sur la fenêtre principale
Ajouter des boutons supplémentaires à la fenêtre principale JMRI qui sont connectés à des actions spécifiques dans les menus pour accéder facilement et rapidement à cette action
Ouvrir Fichier
Charge automatiquement le panneau des fichiers quand le programme démarra. Vous pouvez charger autant de panneau fichiers que vous voulez; chacun d'eux sera entièrement chargé avant le démarrage du suivant.
Exécuter script
Exécute automatiquement un script quand le programme démarre. Vous pouvez exécuter autant de scripts que vous voulez; chacun d'eux sera entièrement exécuté avant que le suivant ne démarre. Notez que tous les fichiers dont dépend le script doivent être ouverts avant d'exécuter le script.
Définir Itinéraire
Définir un itinéraire quand le programme démarre. Vous pouvez configurer autant d'itinéraires que vous voulez; chacun sear configuré dans l'ordre. Notez que Tous les fichiers dont dépend l'itinéraire doivent être ouverts avant de définir l'itinéraire.

Affichage

Interface Graphique Utilisateur (GUI)
Ces boîtes radio vous permettent de sélectionner la façon dont le programme va être afficher. Les choix disponibles dépendent du type d'ordinateur que vous avez, et tous les éléments suivants peuvent être disponibles:
  • Métal - le style par défaut de Java, utilisable sur tous les ordinateurs
  • / Theme/CDE - Une forme Linux/Unix qui fonctionne bien avec les gestionnaires écran x Windows. Il est également disponible sur (presque) tous les types d'ordinateurs.
  • Windows - uniquement disponible sur les systèmes Microsoft Windows, ce qui rend l'apparence du programme, autant que possible comme une application normale de Windows.
  • Mac OS X - disponible uniquement sur MacOS, ce qui rend l'apparence du programme comme une application Macintosh
Langue
Beaucoup de gens ont contribué à l' Internationalisation de JMRI. Ceux-ci ont convertit (beaucoup) de texte à l'écran dans une langue autres que l'anglais. Si vous sélectionnez votre pays ou votre région et votre langue ici, puis enregistrez et redémarrez les préférences, JMRI va utiliser une internationalisation pour ce pays et la langue si elle est disponible. Les éléments qui n'ont pas été traduits continueront à paraître en anglais, de sorte que vous ne manquerez rien. pour plus d'informations sur la façon de le mettre en place, et en particulier la façon de contribuer à des traductions nouvelles et les améliorées, s'il vous plaît voir notre Page d'internationalisation.

Inventaire
Normalement, l'inventaire est stockée dans un emplacement standard dans le cadre du répertoire préférences. Cette option vous permet de le déplacer ailleurs

Pour le configurer, cliquer le bouton "Définir " et suivres les directives.

Le bouton "Réinitialiser" restaure l'emplacement par défaut. Notez que cela change juste où le programme regarde. Si vous avez stocké l'information de l'inventairela ailleurs et que vous voulez la voir revenir à l'emplacement par défaut, vous devez déplacer le fichier vous-même.

Vous pouvez également entrer une valeur par défaut pour le champ "propriétaire" dans les entrées inventiare DecoderPro.ici. Ceci remplit la valeur pour les nouvelles entrées, ce qui peut vous faire économiser un peu de frappe. Elle n'a aucun effet sur les entrées existantes, même si elles ont changées.

Régulateur

Ce volet des PRéférences vous permet de choisir si les fonctions étendues du Régulateur sont disponibles, et celles qui peuvent être utilisées

Régulateur WI

eStop
S'il est sélectionné, écoute l'envoi d'un paquet qui doit être envoyé à partir du périphérique mobile connecté. Si le serveur RégulateurWI ne reçoit aucune communication dans le le temps choisi le régulateur JMRI affecté ralentira son train.

Le temps choisi est approximatif, la réponse peut être quelques secondes en plus ou en moins.

Réseau
Change le numéro du port à utiliser. Assurez-vous que votre pare-feu autorise l'utilisation de ce port.
Changer seulement le Numéro du Port apès avoir arrété le Serveur RégulateurWI s'il y avait un possibilité que les équipements mobiles étaient encore connectés.
Commandes autorisées
Cochez ces cases pour permettre à des fonctionnalités spécifiques d'être commandées par des équipements mobiles. Cela vous permettra de limiter ce que les utilisateurs ont la possibilité de commander.
  • Puissance Traction - Choisissez si les appareils mobiles connectés via cette interface WiFi permet de contrôler l'alimentation traction
  • Aiguillages - sélectionner si les équipements mobiles peuvent commander les aiguillages.
  • Itinéraires - sélectionner si les équipements mobiles peuvent commander les itinéraires.
  • Couplages - Si sélectionné, les équipements mobiles peuvent réaliser et casser des "Couplages Avancés".
    • Format NMRA - Ces couplages peuvent mieux se comporter pour la construction et la commande avec des périphériques WiFi.
    • Spécifique à DCC - Ces couplages peuvent mieux se comporter quand ils sont construit avec un équipement WiFi, mais commandé par un régulateur du fabricant de votre système DC
Le bouton "Sauvegarder" met les paramètres actuels en vigueur. Pour qu'un régulateur connecté utilise les nouveaux paramètres, vous devez redémarrer JMRI ( et ensuite RégulateurWI ) comme indiqué dans le dialogue.

Config Profils

Les Profils de Configuration JMRI rendent aisé le démarrage de vos applications JMRI pour qu'elles trvaillent avec différentes configurations. Chaque profil spécifie un jeu de Préférences. Vous pouvez alors sélectionner lequel utiliser.

Pour avoir plus d'information, voir la page d'aide Profils de Configuration.

Nom du Réseau

L'onglet Nom du Réseau vous permet de spécifier le nom de votre réseau modèle ( module ), quand il est utilisé par plusieurs de protocoles de serveur.

Serveur JSON

Le Serveur JSON JMRI est utilisé par les applications sur les autres ordinateurs, incluant les tablettes et les smartphone, pour see connecter à JMRI pour agir sur votre réseau

Sur ce volet vous pouvez spécifier le port pour la connexion ( habituellement laissé par défaut ) et La fréquence à laquelle le serveur vérifie si la connexion a échoué via un message.

Serveur Web

Le Serveur Web JMRI vous permet de vous connecter à JMRI depuis un serveur web sur votre ordinateur, tablette et smartphone et de travailler avec votre réseau.

Cet onglet vous permet de commander ce que le serveur web fournit. Pour de plus amples information, voir la page panneau Préférences Serveur Web

Serveur SRCP

Le Serveur SRCP JMRI est une façon standard de connexion de programmes utilisateur "client" à un serveur.

Sur ce volet vous pouvez spécifier le port pour la connexion ( habituellement laissé par défaut ).

Serveur Simple

Le Serveur Simple JMRI est une partie de JMRIClient, une méthode de connexion à distance d'un programme JMRI à un autre.

Dans ce volet, vous pouvez spécifier le port de connexion ( habituellement laissé par défaut ).

Bulletins

Ce volet configur les paramètres utilisés dans le calcul des itinéraires sous Bulletins et pour la commande des événements que le trains sous bulletins peut rencontrer.

Carte de Vitesse

Quand un train en automatique rencontre un Signal qui appelle à un changement de vitesse, ou que la progression est stoppée par une occupation en avant, la crate de vitesse est utilisée pour faire les changement de vitesse necessaire. D'autre paramètres commandent la rampe de changement de vitesse.

La Carte de Vitesse Cartographie les noms de vitesse trouvés dans la définition des aspects d'un système de signal pour une valeur. Il est important que tous les noms de vitesse utilisés dans votre système de signalisation soient listés dans le tableau Carte de Vitesse. La valeur assocoéevà un nom de vites comme enregistré ou configuré par lese peut re interprétée de 4 façons: Comme un pourcentage du réglage actuel du régulateur ou réglé par un NX Bulletin ( "PourcentNormal"), ou comme un pourcentage du réglage à fond du régulateur ( "PercentFullThrottle" ). La dernière des deux façons interprète la valeurcome une eéchelle de vitesse de traction; Miles par Heure ou Kilomètres par heure; Utilisez les boutons radios pour sélectionner comment le code doit interpréter la Carte de Vitesse.

La carte D'apparence des Feux est uilisée quand les feux de signalisations sont utilisés comme signaux. Elle cartographie l'apparence pour un nom de vitesse qui en à son tour est cartographié par le tableau Carte de Vitesse.

Il est important que la bonne échelle soit choisie et que la valeur de "Réglage Régulateur/Facteur de Vitesse" soit raisonnable pour les adresses des machines qui circulent sous bulletins. Le facteur ajuste le réglage du régulateurpour une échelle de vitesse de traction. Le réglage Facteur Régulateur est ici un réglage global, mais les facteurs pour des adresses individuelles de machine peuvent être déterminés avec NXBulletins.

Le "Nombre Maximum de Cantons dans un Itinéraire" est la profondeur à laquelle JMRI cherchera un itinéraire quand il définit un bulletin. C'est une valeur globale qui peut être annulée lors de la création d'un bulletin.

Les autres éléments sur le panneau sont utilisés lorsque JMRI effectue un changement de vitesse à un signal ou une occupation devant. Le changement de vitesse est fait avec une séries de d'incrément des réglages du régulateur. "Ramp Step Time" est la durée de chaque cran et "Ramp Throttle Increment" est le montant du changement du réglage du régulateur

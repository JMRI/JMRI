Fenêtre Moniteur de Slots

Les slots sont utilisés pour contrôler individuellement chaque locomotive et les couplage. L'affichage comprend le format de cran d e vitesse du décodeur, la vitesse actuelle et les paramètres de fonction, les informations de couplage et l'état. Vous pouvez choisir d'afficher tous les slots ou uniquement ceux qui sont utilisés activement.

Les cases à cocher en haut permettent de sélectionner les slots à afficher.

Afficher les emplacements inutilisés

Afficher les emplacements système

Exécute un Arrêt d'Urgence pour toutes les locomotives.

Les colonnes sont:

Colonne Affichages
Emplacement Le numéro de l'emplacement
Adresse Adresse de la locomotive à l'emplacement contrôles
Vitesse Pour les locomotives non conservées et les adresses qui se trouvent en tête d'un couplage, il s'agit du réglage de la vitesse actuelle. C'est une valeur interne pour les slots couplés.
Type de Décodeur Le format des commandes envoyées au décodeur, généralement en 128 crans ou en 28 crans, bien que d'autres valeurs soient possibles.
État L'un des quatre états possibles:
Libre Non réservé à un usage particulier, pas de contenu
Inactif Non réservé à un usage particulier, pas de contenu
Commun Non utilisé pour le moment, mais contient toujours des données valides pour le dernier décodeur qui les utilise.
En cours d'utilisation Contrôle actuellement un décodeur
Couplé Affiche l'état du couplage du slot, qu'il soit ou non couplé, la têttet d'un couplage, inclus dans un couplage, etc.
Identifiant du Régulateur Les régulateurs UT4 ont un identifiant de régulateur fixe. Le DT400 a une valeur HEX principale fixe commune et une seconde valeur HEX programmable par l'utilisateur. Il est affiché sous forme de deux octets hexadécimaux. Pour plus de détails, reportez-vous aux instructions de Digitrax.
Direction Indique si le décodeur est commandé pour aller en avant ou en arrière.
F0-F8 Les paramètres de fonction actuels

Configurer la Fenêtre DS64

L'outil de programmation DS64 vous permet de configurer les options internes d'un DS64 directement à partir de votre ordinateur.

Saisissez l'adresse d'une carte DS64 dans le champ de texte, puis cliquez sur . L'outil relira l'état actuel des différentes options et définira les cases à cocher pour correspondre à leurs valeurs actuelles.

Vous pouvez ensuite, si vous le souhaitez, modifier les cases à cocher et cliquer sur pour rendre vos modifications permanentes.

Limitations

En raison du fonctionnement de la carte DS64, cet outil ne peut pas modifier l'adresse de base de l'unité.

Dialogue Envoyer Paquet LocoNet

Les paquets LocoNet sont envoyés au format hexadécimal brut. Vous pouvez utiliser la fenêtre de surveillance dans Afficher les données brutes pour surveiller les paquets.

La section supérieure vous permet d'envoyer un seul paquet LocoNet. Tapez les nombres hexadécimaux et cliquez sur , vous devez inclure l'octet pour la somme de contrôle à la fin du message.

La section inférieure vous permet de saisir de un à quatre paquets qui seront envoyés de manière répétée en séquence, avec les délais définis par l'utilisateur entre les deux. Appuyez sur pour démarrer la séquence de transmission et appuyez à nouveau pour arrêter la séquence.

Téléchargeur de micrologiciels

Certains produits Digitrax peuvent recevoir le nouveau micrologiciel téléchargé. Les mises à jour du micrologiciel sont distribuées sous forme de fichiers .dml (Digitrax Mangled Firmware). Ils contiennent à la fois le code lui-même et diverses informations de gestion permettant de le télécharger dans le matériel attendu.

Pour utiliser cet outil:

  • Cliquez sur et sélectionnez le fichier .dmf que vous souhaitez télécharger.

  • Cliquez sur pour lire le fichier dans le programme et vérifier son contenu.
  • Modifiez les différentes valeurs dans les champs de contrôle uniquement si cela est demandé par Digitrax (généralement inutile)
  • Cliquez sur pour charger le firmware. Cela prendra un peu de temps.

Télécharger des sons

Les décodeurs sonores Digitrax sont chargés avec des "projets sonores", stockés dans des fichiers "Projet sonore" (.spj). Ceux-ci contiennent un certain nombre de "fragments sonores" au format .wav, ainsi que des informations de contrôle supplémentaires.

L'outil Decoder Sound Downloader vous permet de charger un nouveau fichier .spj dans un décodeur de son Digitrax via une interface Digitrax PR2 ou PR3. Si vous utilisez un PR3, vous devez le mettre en "mode programmeur" avant de télécharger, consultez la section principale page PR3.

Digitrax fournit des exemples de fichiers de projet audio sur leur site Web Sound Depot (http://www.digitrax.com/sounddepot .php) . Vous pouvez également y trouver des liens vers de la documentation et des outils supplémentaires.

JMRI fournit également des outils pour l'édition de fichiers de projet audio pour inclure de nouveaux sons et même pour changer la logique du son . Lorsque vous sélectionnez l'outil Télécharger des sons pour la première fois, il ouvre une boîte de dialogue avec la plupart des commandes désactivées:

(Cliquez sur une image de cette page pour l'agrandir.)

La ligne d'état en bas vous guidera pas à pas.

  1. Tout d'abord, cliquez sur "Sélectionner" et sélectionnez le fichier .spj que vous souhaitez télécharger.
  2. Ensuite, cliquez sur "Lire" pour lire le fichier (il s'agit d'une étape distincte. Vous pouvez simplement cliquer sur "Lire" plusieurs fois si vous modifiez et enregistrez de nouvelles versions du fichier).

    Si tout va bien, le fichier sera lu dans le programme et vérifié, une chaîne de version du fichier sera affichée et le bouton "Télécharger" sera activé.

  3. Cliquez sur "Télécharger" pour lancer le processus de téléchargement. Cela prendra un peu de temps. La progression est décrite dans la ligne d'état:
    "Commencer le téléchargement; effacer le flash"
    La première étape consiste à effacer la mémoire du décodeur afin qu'elle puisse prendre de nouvelles données. Cela prendra environ 30 secondes, selon le décodeur spécifique.
    "Envoi du message d'initialisation"
    Configurez le décodeur pour accepter les données après leur effacement. Cela ne devrait prendre qu'une fraction de seconde.
    "Envoyer des données SDF"
    Chargez les données de définition du son dans le décodeur, ce qui ne devrait prendre que quelques secondes
    "Envoyer des données WAV"
    Les données audio sont en cours de téléchargement. Chaque fois que vous envoyez un bloc, vous obtenez une mise à jour: "Envoyez le bloc de données WAV 1", puis 2, puis 3, etc.
    "Terminé"
    Le téléchargement est terminé.
    En cas de problème, le statut final sera "Téléchargement interrompu". La raison la plus courante est que le PR2 n'a pas pu gérer les données en cours d'envoi, qui sont répertoriées comme "PR2 non prêt".

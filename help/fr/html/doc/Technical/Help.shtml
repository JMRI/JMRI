Organisation des Fichiersd'Aide

Les fichiers d'aide sont les suivants situé dans le répertoire "Aide" du répertoire la distribution JMRI .

La copie originale de ce document est stockée dans notre répertoire en tant que répertoire "help" dans le dépôt de code JMRI. Cela signifie que pourobtenir votre propre copie locale, il suffit de suivre les étapes pour "obtenir le code" page.

Avec cela, tous les fichiers en langue anglaise sont localisés dans le sous-répertoire "en". A terme, cela nous permettra d' internationaliser les pages d'aide.

Il y a plusieurs arborescences de fichiers:

  • package - organisé comme l'arbre des paquets sources, celui-ci contient des fichiers d'aide spécifiques au code, par exemple pour un fenêtre ou composant. Par exemple, une fenêtre qui est créée par le jmri.jmrit.speedometer.SpeedometerFrame (de la classe src/jmri/jmrit/speedometer/SpeedometerFrame.java ) devrait avoir son aide spécifique à la fenêtre dans un fichier à package/jmri/jmrit/speedometer/SpeedometerFrame.shtml.
  • html - descriptions générales, tutoriels, etc.Ceci est organisé en répertoires qui représentent des domaines généraux.
  • manuel - la version la plus récente du DecoderPro et tous les manuels ont été reformatés pour être utilisés dans le système d'aide. (Les anciens sont stockées dans le site web principal )
    • (Les anciens sont stockées dans le réseau principal site)

Même si c'est ainsi que nous aimerions que nos fichiers d'aide soient organisés, les pages d'aide sont créées par les membres de la communauté et parfois des descriptions générales sont mises dans le paquet plutôt que le répertoire html et vice versa. [Si vous voyez quelque chose qui doit être déplacé, veuillez poster une note sur le forum jmri-developers online groups.io car il y a des membres qui essaient parfois de faire un peu de ménage.]

À long terme, nous ne voulons que deux branches à la structure du fichier d'aide : la partie "package" de l'arbre pour les informations d'aide qui sont spécifiques à un morceau de code particulier, et une autre partie de l'arbre pour des informations plus générales. (Il s'agit d'une question religieuse de savoir si cette deuxième partie doit être appelé "html" ou "manuel" ; l'essentiel est que nous nous retrouvions avec juste un). Le web est censé être un web, avec de nombreux chemins d'accès pour accéder au contenu. La deuxième partie de ("html") doit également être organisée comme "une page, un sujet", avec des liens pour les connecter en fonction des besoins.

Accès Internet aux Pages d'Aide

Il n'est pas pratique de devoir maintenir deux pages web séparées pour le site web principal et le système d'aide. Afin de réduire la nécessité de ce système, nous utilisons un formulaire particulier pour les pages web du système d'aide. [Des informations complémentaires sont également disponibles ici.]
  • Utilisez les extensions de fichiers ".shtml" pour que les principaux serveurs web recherchent les fichiers côté serveur comprend.
  • Lorsque vous créez une nouvelle page, commencez par une copie du fichier help/fr/TemplateBar.shtml ou le fichier modèle help/fr/TemplatePlain.shtml, selon que vous souhaitez ou non inclure une "sidebar" (via un fichier nommé Sidebar.shtml). Ce gabarit mettra le haut et le bas appropriés pour que la page s'affiche correctement.
  • En général, tous les fichiers d'aide dans l'arborescence des répertoires html utilisent une barre latérale. En général, les fichiers dans la partie du paquet de l'arbre ne le font pas. Si vous utilisez une barre latérale, vous en trouverez peut-être une qui convient votre objectif dans le sous-répertoire spécifique dans lequel vous travaillez, dans son répertoire parent, ou, dans de rares cas, vous pouvez utiliser celui du répertoire html lui-même (soit /help/fr/html soit /help/fr/html).

    Si aucun de ces éléments ne répond à vos besoins (ou n'existe), vous pouvez créer une barre latérale .shtml dans le sous-répertoire, auquel cas, vous devez suivre le format des autres fichiers Sidebar.shtml dans d'autres sous-répertoires du de même niveau ou dans le répertoire parent. En 2020, un effort important a été fait pour normaliser les fichiers de la barre latérale, en utilisant les includes côté serveur pour les fichiers de parties communes dans le /help/en/parts directory.En général, un fichier Sidebar.shtml dans un sous-répertoire de terminal (sans niveau inférieur) référencera en outre une "partie locale" dans un fichier du même sous-répertoire et nommé Sidebar[type]LocalPart.shtml, oú [type] serait basé sur le nom du répertoire parent, tel que Hardware ou Tool. Cette utilisation des fichiers de la partie locale permet aux fichiers Sidebar.shtml dans le sous-répertoire terminal d'utiliser un code identique, ce qui facilite considérablement la maintenance.

    Si vous n'avez pas besoin d'une partie locale, vous pouvez inclure le Sidebar.shtml du répertoire parent mais UNIQUEMENT si ce fichier n'a pas d'URL relatives (URL avec "../" et similaires). Si c'est le cas, les liens latéraux ne fonctionneront pas lorsqu'ils seront intégrés à votre page. Regardez le contenu de Sidebar.shtml dans le répertoire parent avant d'essayer de le référencer.

  • Tout le formatage des pages de l'aide JMRI et des autres pages web se fait par le biais de CSS inclus créés à l'origine par John Plocher. Ils sont inclus à partir de "/css" via Server-Side Comprend (SSI). Le fichier principal se trouve à /css/default.css et travaille avec quelques autres fichiers dans ce même répertoire. Comme l'aide Java ignore le SSI, le formatage CSS est ignoré lorsque l'aide Java affiche les pages dans JMRI Lui-même. Dans ce cas, JavaHelp fournit le formatage.

Normes et Limites de l'Aide Java HTML

JavaHelp affiche "plainld HTML 4.0", sans fournir une partie du sucre syntaxique fournis par de nombreux navigateurs pour permettre un rendu HTML pauvre. Ne pas inclure d'options de formatage particulières, notamment les commandes CSS intégrées. Il y a quelques points spécifiques à surveiller :
  • Les balises d'ancrage ont besoin d'un format spécifique. Plus précisément, elles doivent être écrites 
    <a name="pull" id="pull"> ;...</a> ;
    autour du contenu du texte uniquement. Il n'y a pas de problème à ce qu'il y ait du vide entre les balises d'ouverture et de fermeture. Il n'est pas acceptable de les combiner en un seul <a name="pull"/>. ne faites pas ça, car cela pourrait embrouiller JavaHelp.

    Par exemple, pour mettre une ancre sur un titre, faites 

    <h2> <a name="foo" id="foo">Foo!</a></h2> ;

    Note : Les balises d'ancrage mal structurées (pas comme - ci-dessus) ne provoquent souvent qu'un < ; le personnage apparaissant par lui-même sur la page d'aide rendue.

  • Les ancres doivent utiliser l'attribut "name" ainsi que l'attribut "id" pour les futures compatibilité. Le nom est reconnu par JavaHelp mais pas par les dernières versions de html alors que id n'est pas reconnu par JavaHelp. Voir l'article ci-dessus pour un exemple.
  • Ne pas utiliser d'espaces ou de %20 dans les noms d'ancre. Si vous devez utiliser une phrase pour un nom, utilisez le caractère de soulignement : "This_Anchor", et non "This Anchor.
  • Vous pouvez utiliser l'élément <code>, ou l'élément <pre>, mais vous ne pouvez pas utiliser les deux ensemble. <code> est bon pour mettre en évidence le texte en ligne comme étant un code (comme nous l'avons utilisé ici) ; <pre> est bon pour donner une bonne apparence à un bloc de HTML ou de Java en ligne. Pour donner une apparence de code à un bloc, enveloppez-le dans <pre style="font-family : monospace ;">
  • Pour décrire une séquence d'actions ou de pulldowns de l'utilisateur, utilisez la double flèche droite pour indiquer la séquence, balise HTML 4 ampersand-rArr-semicolon, identifiez la séquence de trous comme "forte" et mettez la commande terminale entre guillemets :
    File&rArr ; "Store Panels"
  • Vous ne pouvez pas compter sur un emplacement particulier pour vos fichiers sur la machine locale, donc tous les liens aux autres pages d'aide doivent être relatives. Les liens vers des contenus XML doivent également être relatifs, même si ils ne sont pas affichés par JavaHelp.
  • Les liens vers des pages web en dehors du système d'aide fonctionnent et doivent être spécifiés comme pleinement qualifiés, de préférence avec https://. Dans la plupart des cas, elles seront automatiquement ouvertes dans un navigateur web externe. Si vous consultez déjà la page d'aide dans un navigateur externe, ces pages peuvent être ouvertes dans un nouvel onglet ou dans le même onglet (en écrasant la page en cours), selon la façon dont la personne qui a écrit la page d'aide a codé la référence. Au moment de la rédaction du présent document (2020/04/04), un débat est en cours sur la méthode préférée.]

Créer de l'aide/des pages Web pour les Nouveaux Matériels

Lorsque vous intégrez la prise en charge d'un nouveau matériel dans JMRI, vous devez également créer une ou plusieurs pages pour le système d'aide et le web. Plusieurs étapes sont nécessaires pour garantir l'intégration de votre nouvelle aide matérielle dans le système d'aide JMRI et sur son site web. Toutes les références aux fichiers ci-dessous renvoient au JMRI Structure du dépôt à Github. Note d'utilisation de la capitalisation dans les références au produit (fictif) "X" du fabricant "M":

  1. Créer la ou les pages d'aide générale pour le nouveau matériel :
    • Voir autres pages d'aide sur le matériel liées à la page Index du matériel pour des exemples avant de commencer. Il existe différents styles en fonction de la quantité d'informations que vous souhaitez fournir. Vous pouvez également fournir un lien vers des pages de fabricants externes.
    • Créez votre page d'aide et placez-la dans "/help/fr/html/hardware", soit directement sous "X.shtml" s'il s'agit d'un seul fichier, soit dans son propre sous-répertoire "/help/fr/html/hardware/m/" s'il s'agit de plusieurs pages. Vous devez nommer le premier fichier "index.shtml" lorsqu'il se trouve dans son propre sous-répertoire.
    • Si vos pages d'aide comportent des images, créez un sous-répertoire "/help/fr/html/hardware/m/images" pour les stocker.
    • Mettre dans la barre latérale standard par un "include" de "/help/fr/html/hardware/Sidebar.shtml" (en utilisant une adresse relative appropriée) dans votre ou vos pages d'aide. [Quelques créateurs de matériel et de familles de matériel créent leur propre fichier "SidebarHardwareLocalPart.shtml" dans le sous-répertoire "m" pour avoir des informations personnalisées dans le sidebar. Cela nécessite également la création d'un fichier "Sidebar.shtml" au "format standard" dans le sous-répertoire "m". Voir certaines entrées de matériel existantes telles que celui-ci pour les exemples.]
  2. Créer une ou plusieurs pages d'aide "spécifiques au code" pour le nouveau matériel :
    • Comme indiqué dans la première section de cette page, l'aide "spécifique au code" (c'est-à-dire liée à une fenêtre particulière ou) doit se trouver dans les pages d'une autre partie de l'arborescence des répertoires de fichiers. Toutes les pages que vous créez pour expliquer les fenêtres ou le code doit être placé sous la branche "/help/en/package/" en fonction de l'endroit oú le code réel est stocké.
    • Assurez-vous de fournir des liens vers ces pages dans vos pages d'aide générale.
  3. Créer des entrées dans la page d'index du matériel :
    • Ajouter une entrée de liste de matériel et un paragraphe d ans "/help/en/html/hardware/index.shtml" en suivant les formats et la catégorisation d'autres matériels similaires.
    • Ajouter une entrée pour "M X" à la catégorie/index alphabétique à l'endroit approprié près du haut de "hardware/index.shtml".
  4. Créer une entrée dans la "partie latérale" du matériel supporté :
    • Ajouter une entrée par ordre alphabétique pour "M X" à la partie "Hardware" de la barre latérale "/help/en/parts/SidebarSupportedHardware.shtml".
  5. Créer la Table des Contenus (TOC) de l'Aide et les Entrées Index :
    • Ajouter une entrée pour "M X" sous "Hardware Support" au fichier "JMRIHelp_enTOC.xml" qui se trouve dans le répertoire /help/en/.
    • Ajouter une ou plusieurs entrées pour "M X" telles que "X (M)" ou d'autres éléments d'index utiles au fichier "JMRIHelp_enindex.xml". se trouve dans le répertoire /help/fr/.
    • Voir la section ci-dessous sur Aidez la table des matières et l'index pour les étapes supplémentaires de mise à jour le TOC et l'index.

En suivant toutes ces étapes, vous vous assurez que les autres utilisateurs pourront trouver et utiliser le nouveau matériel que vous avez intégré dans JMRI.

Vérifier vos Changements

Le processus Intégration continue vérifie toutes les modifications apportées aux fichiers d'aide deJMRIJ pour s'assurer qu'ils sont corrects. Si vous souhaitez effectuer cette même vérification pendant que vous travaillez, faites:

   
    ant scanhelp

Si vous n'avez pas accès à ant, vous devez utiliser l'un des programmes de validation HTML disponibles en ligne, par exemple à partir de la validation du balisage du W3C Service. Ces services trouveront de nombreuses erreurs courantes (balises non appariées, etc.), mais aussi parfois donner des messages d'avertissement ou même d'erreur qui n'ont pas d'incidence sur l'intégration de JMRI.

Aide Table des Matières et Index

L'aide Java comprend un tableau de contenus et un index. Il sont tous deux disponibles sous forme d'hyperliens sur le web, et fournis via une une interface utilisateur plus agréable lorsque l'on utilise l'aide Java dans le programme.

Les informations sous-jacentes doivent être gérées manuellement dans les fichiers help/fr/JmriHelp_enTOC.xml et help/fr/JmriHelp_enIndex.xml, respectivement. JHelpDev fait ensuite les fichiers dont l'aide Java a besoin, et une étape "fourmi dans le répertoire fait le webtoc.shtml et le webindex.shtml qui sont affichés sur le web.

Note : Cela signifie que vous ne devez pas modifier manuellement le les fichiers webToc.shtml et webindex.shtml, car ces modifications seront écrasées par des étapes ultérieures.

Les fichiers JmriHelp_enTOC.xml et JmriHelp_enIndex.xml sont des fichiers XML directement utilisables, et vous pouvez les modifier manuellement. Assurez-vous d'exécuter l'étape "créer tout" de JHelpDev et "ant" après l'avoir fait.

Vous pouvez également utiliser l'outil JHelpDev pour ajouter des éléments à la table des matières et index si vous le souhaitez. Plus d'informations sur la façon de le faire sur le Site web de JHelpDev. N'oubliez pas de lancer "ant" après cela pour inclure vos mises à jour sur les versions web.

Création des Fichiers de Contrôle avec JHelpDev

JavaHelp utilise divers fichiers XML pour contrôler la façon dont la table des matières et l'index sont affichés. Nous les créons avec JHelpDev. S'il vous plaît, ne pas les éditer manuellement.

JHelpDev est maintenant inclus dans la dist. de JMRIJ Pour utiliser l' outlil:

  1. Vérifiez que vous avez bien mis à jour la version la plus récente dans le dépôt de code avant de commencer.
  2. Vous pouvez exécuter en mode batch en lançant ant depuis la ligne de commande dans le répertoire help/en/. Il ouvrira toujours des fenêtres, mais devrait s'exécuter tout seul. Cela peut prendre environ une minute.
  3. Si vous souhaitez l'utiliser manuellement, veuillez suivre le reste de ces points. (Ou si vous utilisez Eclipse, cliquez avec le bouton droit de la souris sur le help/en/build.xml et sélectionnez Run As Ant Build.)
  4. Démarrez l'outil en cliquant sur le fichier jhelpdev.jar. (Si cela ne fonctionne pas, essayez d'exécuter soit JHelpDev.csh, soit JHelpDev.bat, en fonction du type d'ordinateur que vous avez)
  5. Une fois que la fenêtre apparaît, sélectionnez "Open Project" sous le menu "Fichier".
  6. Naviguez vers le répertoire "help" dans votre copie vérifiée du code, puis vers le répertoire "en" à l'intérieur de celui-ci, puis sélectionnez le fichier "JHelpDev.xml" et cliquez sur "Open".

  7. Vous pourriez recevoir un message sur "[Erreur fatale] index.html:1:3 : Les déclarations de balisage contenues ou pointé par la déclaration de type de document doit être bien formé". Bien qu'il soit dit que c'est fatal, c'est vraiment pas un problème. Il suffit de l'ignorer. Au démarrage, la carte (un fichier contenant l'indice JHelpDev de tous les ancrages du système d'aide JMRI ) est régénéré.

  8. En cliquant sur l'onglet "Éditeur de l'index" ou sur l'éditeur de la table des matières , revoir et mettre à jour les entrées de l'aide. Une ligne rouge marque un hyperlien que JHelpDev ne peut pas localiser sur le disque. Cliquez avec le bouton droit de la souris sur une telle ligne pour ouvrir le menu contextuel Edit.
  9. Cliquez sur le bouton "Créer tout" pour recréer lindex TOC, , etc.
  10. Retour sur la ligne de commande et dans l'aide/fr invoquez "ant index TOC" pour créer les pages de l'index web et de la table des matières.

Les différents fichiers de contrôle utilisés par JavaHelp sont stockés dans notre dépôt de codes, donc une fois que vous aurez fait cela, ils montreront tel que modifié. Veuillez les vérifier lorsque vous consultez la nouvelle aide afin que d'autres personnes n'aient pas à recréer ces contrôles pour eux-mêmes.

Accès à l'Aide du Code

Dans le code JMRI, accédez au système d'aide vient via la classe jmri.util.HelpUtil. (Pour les pour des raisons historiques, il y a un peu de code dans apps.Apps, mais vous devriez l'ignorer).

La façon la plus simple d'ajouter un menu d'aide à un cadre est d'utiliser un JmriJFrame (ce que vous devriez faire de toute façon), et d'appeler addHelpMenu(...) après avoir construit d'autres menus.

Par convention, nous utilisons une arborescence de fichiers similaire pour la source et dles fichiers d'aide. Par exemple, le jmri.jmrit.simpleclock.SimpleClockFrame class hérite de JmriJFrame, et ajoute un menu d'aide avec la ligne 

  addHelpMenu("package.jmri.jmrit.simpleclock.SimpleClockFrame", true) ;
Le fichier d'aide se trouve alors à help/en/package/jmri/jmrit/simpleclock/SimpleClockFrame.shtml.

Vous devez ajouter la nouvelle page à l'index JHelpDev pour que JMRI l'accepte comme cible de l'aide (voir le titre suivant).

Un certain nombre de fichiers d'aide ont été mis en place sans Il est à espérer que certains utilisateurs les modifieront et y contribueront.

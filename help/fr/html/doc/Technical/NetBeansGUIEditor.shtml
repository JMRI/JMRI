Code JMRI : Utilisation du générateur d'interface graphique NetBeans Swing

Des Portions du code de l'interface utilisateur graphique (GUI) de JMRI sont maintenus à l'aide du NetBeans Swing GUI Builder (le "Builder"). Cette page est un guide à l'utilisation de l'outil Builder. Voir Construire avec NetBeans pour une introduction générale à l'utilisation de NetBeans pour la maintenance de JMRI.

Introduction

NetBeans 7.4 ou 8.0 comprend un constructeur d'interface graphique Swing qui facilite la maintenance d'une interface graphique basée sur Java Swing (telle que JMRI) que au lieu d'écrire l'interface utilisateur entièrement à la main.

Cet outil peut créer un code qui n'est pas idéalement séparé en code Java maintenu et généré par l'homme. Si vous utilisez NetBeans, vous êtes protégé de cela, car le constructeur fait un bon travail d'intégration de ses vues avec l'éditeur source. Cependant, si vous utilisez un autre éditeur, vous serez exposé à ceci.

Utilisation du constructeur

La barre d'outils de l'IDE NetBeans contiendra automatiquement un bouton intitulé Design si la classe peuvent également être éditées dans le constructeur de l'interface graphique de Swing.

Cliquer sur le bouton Design présentera la possibilité de faire glisser des boutons, des étiquettes, des panneaux et autres objets sur une fenêtre pour les placer visuellement. Tout objet qui est conforme à la La norme JavaBean peut être placée sur la fenêtre et liée à d'autres objets sur la fenêtre, même si elle n'a pas de représentation visuelle.

Lorsque le constructeur ne peut pas générer automatiquement le code, il fournit des méthodes vides, ou ébauches, qui peuvent être complétées pour mettre en œuvre la logique souhaitée, et passera automatiquement à la vue du code selon les besoins.

Examen du code

Le Builder gère deux fichiers : un fichier .java, contenant le code compilable et un fichier .form, ccontenant une représentation XML de l'interface graphique ainsi que le code utilisé pour créer la méthode initComponents() dans le fichier .java.

.java File

Dans NetBeans, le code généré par le Builder, à n'importe quel autre code Java, bien que certaines parties de celui-ci ne puissent pas être édité (NetBeans ne permet tout simplement pas que le code soit changé). Les méthodes d'ébauche seront modifiables, mais il n'est pas possible de changer les signatures de la méthode (nom, paramètres, valeurs de retour ou clauses de lancement).

.form Fichier

Le fichier .form est complètement caché dans les IDE NetBeans . Ouverture d'un fichier .java avec un fichier .form correspondant (le même nom de fichier dans le même paquet) ajoutera le Design dans la barre d'outils de l'éditeur, permettant l'utilisation du Builder.

Le fichier .form est un fichier XML respectant la DTD dans le code source de NetBeans (lié à la tête du tronc NetBeans ). NetBeans utilise ce fichier pour stocker des informations en dehors de la spécification JavaBeans, nécessaires pour générer correctement la méthode initComponents(), ébauche de méthodes, et des listes de variables. En l'absence d'autres modifications du .java ce fichier pourrait être utilisé pour générer complètement un nouveau Fichier .java, si nécessaire.

Ce fichier ne doit pas être édité à la main, mais doit être généré par le Builder.

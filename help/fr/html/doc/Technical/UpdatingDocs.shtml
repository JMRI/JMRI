Les pages web et les écrans d'aide de JMRI sont rédigés dans un simple sous-ensemble de HTML, le langage qui alimente le web. La partie la plus importante du contenu est le texte simple que les gens liront, et les informations supplémentaires de formatage est maintenue à un niveau très minimal. Pour mettre à jour une page web de JMRI, il faut modifier directement sur GitHub web ou sur votre ordinateur avec un éditeur de texte.

Utiliser une application de traitement de texte comme MS Word ou Front Page pour éditer ces documents .shtml n'est pas une bonne idée car ils ajouteront probablement leurs propres codes de formatage propriétaires qui vont tout gâcher, peut-être même au point d'être inutilisable par JMRI. Aussi, ne pas renommer les fichiers existants, ni apporter de modifications au formatage des informations en haut ou en bas.
Vous pouvez donc vous concentrer sur les modifications rédactionnelles à apporter dans le corps principal d'un fichier, ce qui est en fait ce à quoi les lecteurs sont de toute façon attentifs.

Pour commencer, vous pouvez fouiller dans les fichiers de votre ordinateur Les fichiers du programme JMRI dans le dossier du manuel de DecoderPro. Sur un système Windows, l'emplacement de ce dossier ressemble à quelque chose comme ça... C:\ProgramFiles\JMRI\help\en\manual. Dans ce dossier se trouve un fichier appelé index.shtml. Ouvrez le avec un éditeur de texte et regardez autour de vous. Si vous changez quelque chose, vous pouvez alors ouvrir la page dans le système d'aide JMRI et voir à quoi il ressemble (soit garder une copie de sauvegarde au cas où la page originale, ou vous pouvez réinstaller JMRI pour restaurer la).

Les modifications les plus simples sont celles apportées au texte lui-même : Pour ajouter une phrase ou corriger une formulation, il suffit de le faire avec l'éditeur. Pour ajouter une pause entre les paragraphes, par exemple pour en ajouter un nouveau vous insérez une "balise" qui indique à JMRI ou à un navigateur d'insérer un paragraphe avec une coupure.

Création d'images à partir des captures d'écran

Pour Windows

Pour prendre une capture d'écran dans Windows, commencez par configurer la fenêtre qui contient ce que vous voulez capturer. Faites en sorte qu'il prenne le moins de place possible sur votre écran pendant qu'il affiche ce que vous voulez. . Vous pouvez prendre une photo de la fenêtre active (au lieu de tout votre bureau), en appuyant sur la touche <Alt> et la touche <Print Screen> clé en même temps. (Note : Selon votre clavier, vous devrez peut-être appuyer sur la touche aussi sur <Fn> ou <Function-Lock> ). Cela met l'image sur votre "presse-papiers" et vous pouvezle coller dans un programme d'édition d'images tel que Window's Paint ou Photoshop.

Pour Linux

Si votre distro n'a pas déjà une application par défaut qui prend en charge l'utilisation de l'écran d'impression de <Print Screen>> ; et <Alt + Print Screen>que vous pouvez installer KSnapshot ou votre préféré pour faire votre capture d'écran.

Pour macOS

Après avoir préparé votre capturer, maintenez la touche <Pomme + Shift + 4> Lorsque vous les relâchez en même temps, le curseur devient un signe plus, et vous pouvez le faire glisser autour de la la zone que vous souhaitez. Lorsque vous lâchez prise, vous aurez un fichier image sur votre bureau.

Dimensionnement et sauvegarde des Images

Toute modification de votre image doit être effectuée avant d'utiliser Amaya pour l'insérer dans votre document. N'oubliez pas que si vous le modifiez, il doit être réinséré. Veillez à enregistrer l'image sous l'un des types de fichiers suivants : PNG, JPG, ou JPEG (pas BMP ou TIF), et n'utilisez pas d'espaces ni de lettres majuscules dans le nom du fichier d'image. En général, une bonne taille pour la largeur d'un l'image est de 500 pixels ou moins, à une résolution web de 72 pixels par pouce. Si vous savez comment redimensionner vos images, ce serait très utile.

Soumission à GitHub

Si vous comprenez cela, vous êtes prêts à nous aider à mettre à jour les nombreux documents de JMRI pour le site web JMRI et pour les fichiers d'aide dans un prochaine version du logiciel.

Lorsque vous soumettez vos changements pour être inclus dans les futures JMRI ils font l'objet d'un contrôle rapide et sont ensuite fusionnés dans le contenu précédent. La vérification se fait à l'aide des outils GitHub pour trouver les parties du fichier qui ont été modifiées.

Préparer les fichiers

Le type de fichier à utiliser pour JMRI a l'extension .shtml. Si vous avez fait un (nouveau) .html ou .htm, vous devez changer l'extension en .shtml. Ne pas utilisez des espaces ou des majuscules dans vos noms de fichiers. Si vous êtes sans modifier ni ajouter d'images, il suffit d'enregistrer le fichier avec son nom d'origine, c'est-à-dire "index.shtml", dans un lieu où vous pouvez les trouver facilement lorsque vous êtes prêt à les copier à GitHub.

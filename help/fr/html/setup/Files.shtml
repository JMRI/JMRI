Installation de JMRI : Fichiers de configuration

Cette section tente d'expliquer où sont stockés les fichiers de configuration de JMRI (paramètres) sur votre ordinateur et les informations qu'ils contiennent.

Emplacements des fichiers de configuration
Il existe deux endroits où les fichiers de configuration JMRI peuvent être trouvé :
  1. Les versions originales des fichiers de configuration JMRI se trouvent dans un répertoire "xml" du paquet d'application JMRI et de ses sous-répertoires (L'emplacement exact dépend de la façon dont vous avez installé JMRI sur votre ordinateur)

    Notez que tout ce que vous modifiez ici sera perdu lorsque vous mettrez à jour JMRI dans une nouvelle version. Ce n'est pas le bon endroit pour faire des changements !

  2. Le répertoire des préférences JMRI de l'utilisateur et Les sous-répertoires sont également importants. JMRI regarde ici d'abord pour les fichiers dont il a besoin.
    Vous devez mettre toutes les version des fichiers modifiées localement ici pour ne pas être écrasées par une mise à jour de JMRI. Tous les fichiers dans lesquels le programme écrit et qui contiennent des informations, par exemple les entrées de l'Inventaire et les fichiers de Panneau, sont également écrit ici.
    L'emplacement des préférences de l'utilisateur par défaut varie selon système d'exploitation de l'ordinateur. Normalement, il est localisé :
    • Sur une machine Linux, cherchez un .jmri dans votre répertoire personnel.
    • Sur macOS, les préférences sont stockées dans un dossier "JMRI". dans le dossier /Users/me/Library/Preferences/ folder.
    • Sur Windows, cela peut être à plusieurs endroits en fonction de la version de Windows que vous utilisez et si vous avez configuré plusieurs utilisateurs. Démarrez en cherchant sous "Documents" ou "Documents utilisateur" ou "Utilisateurs et documents". Vous recherchez un dossier appelé "JMRI".
    Le moyen le plus simple de trouver l'emplacement des préférences est de lancez un programme JMRI, par exemple DecoderPro ou PanelPro, sélectionnez "Emplacements..." du menu "Aide" et cliquez sur [Emplacement Fichiers utilisateur n].
    Un cadeau spécial pour les utilisateurs de Windows : si vous allez sur le site Menu Démarrer, dans la section JMRI, vous pouvez sélectionner "Préférences" pour qu'il ouvre ce répertoire pour vous.
  3. Les fichiers d'utilisateurs peuvent être stockés sous l'emplacement des préférences, mais JMRI peut aussi être configuré pour les mettre ailleurs.
    Vous pouvez définir ou modifier les scripts de localisation et fichiers utilisateurs dans le panneau Préférences... -> Emplacements des fichiers . Notez que la modification de l'"emplacement des fichiers utilisateurs" permettra également de modifier l'emplacement par défaut des panneaux, des Inventaires etc.
Types de fichiers de configuration
Dans votre dossier Préférences JMRI, vous trouverez plusieurs types d'informations des fichiers de configuration utilisés par DecoderPro :
  • Définitions des décodeurs - Stocké dans le sous-répertoire décodeurs , un pour tous les décodeurs personnalisés ; ceux-ci définissent comment une famille spécifique de décodeurs est configurée.
  • Configurations des locomotives - Stocké dans le sous-répertoire Inventaire , un pour chaque Locomotive ; celles-ci enregistrent la configuration spécifique de chaque locomotive de votre liste.
  • Définitions du programmeur - Stocké dans le sous-répertoire programmeurs lorsque vous avez utilisé au moins un programmeur ; ceux-ci définissent les formats pour les différentes fiches de programmation qui sont disponibles dans DecoderPro.
  • Le dossier DecoderPro, contenant votre profils.
Fichiers de configuration générale
Il existe de nombreux autres fichiers de configuration JMRI, dont certains sont partagés par les applications, qui vivent dans le répertoire Preferences de l'utilisateur :
  • roster.xml - un index de vos Locomotives
  • decoderIndex.xml - un index des fichiers de définition des décodeurs disponibles. Il contient également les informations sur le modèle de décodeur spécifique qui utilise quel dossier, comment les identifier et les options qu'ils mettent en œuvre.
  • names.xml - non utilisé en fonctionnement normal , il s'agit d'une liste de la variable standard Noms à utiliser dans les définitions de programmeurs.

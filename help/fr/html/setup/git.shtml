Installation de JMRI : Partage de fichiers avec GitHub

GitHub est un service gratuit qui vous permet de partagez des fichiers entre plusieurs ordinateurs sur un réseau en stockant une copie commune dans un dépôt Git. Après vous être inscrit sur GitHub et avoir configuré votre machine(s) locale(s), vous pouvez configurer JMRI pour l'utiliser pour partager des Inventaires, des fichiers de configuration réseau et d'autres informations entre plusieurs ordinateurs.

Note : Si vous n'êtes pas familier avec Git et GitHub, vous feriez peut-être mieux d'utiliser Dropbox pour partager des fichiers entre ordinateurs. Git et GitHub sont un système complet de contrôle de version des logiciels, ce qui les rend plus complexes que la quasi automatisation de Dropbox. Git et Github vous permettent de contrôler les modifications, d'avoir plusieurs versions sur des branches et, en général, de gérer vos configurations JMRI de très près ; mais si vous voulez juste partager des fichiers, Dropbox peut être plus facile.

L'utilisation de Git présente quelques avantages:

  • En utilisation normale, Git conservera l'historique complet de vos fichiers en local afin que vous puissiez toujours revenir à une configuration antérieure.
  • Git fournit tous les outils puissants pour gérer les mises à jour lorsque vous effectuez des modifications sur plusieurs ordinateurs, c'est-à-dire l'ordinateur principal et quelques ordinateurs portables pratiques.
  • Git peut être utilisé avec une connexion réseau de type "sneaker" en poussant/tirant une copie d'un dépôt sur une clé USB.

L'approche suivante devrait également fonctionner avec d'autres services de synchronisation de fichiers basés sur le cloud tels que Google Drive et OneDrive. Cependant, la solution Dropbox est bien testée, ayant été utilisée par un certain nombre de développeurs et d'utilisateurs de JMRI depuis un certain temps.

étape 1

Pour mettre en place un emplacement partagé du fichier utilisateur sur le web:

  • Créer un dossier JMRI sous le contrôle de Git sur le premier ordinateur. Si vous voulez utiliser GitHub, les étapes sont les suivantes :
    • Créez un compte GitHub si vous n'en avez pas.
    • Créez un nouveau dépôt sur GitHub en utilisant le bouton "New" dans le panneau "Repositories".
    • Cliquez sur le bouton nommé "(downarrow) Code" et copiez l'URL de votre dépôt.
    • Sur votre ordinateur, dans le répertoire oú vous voulez contenir vos fichiers JMRI, cochez ce dépôt avec (par exemple) : 
                  clone de git https://github.com/myaccountname/JMRI.git
  • Si vous avez déjà créé des éléments (entrées de la liste, panneaux, etc.), allez dans l'Aide‑>Locations et appuyez sur OpenUser Files Location.

    Cela ouvrira une fenêtre sur votre bureau montrant un répertoire. Rappelez-vous oú il se trouve, parce que vous déplacerez les éléments à partir de là plus tard.
  • Rester sur l'aide‑> fenêtre Locations, vérifier si l'emplacement de votre liste est différent de l'emplacement des fichiers utilisateurs. Si c'est le cas, cliquez sur Ouvrir l'emplacement de la liste pour ouvrir une fenêtre sur cet emplacement. Vous y copierez des fichiers plus tard.
  • Préférences‑> Roster‑ > Roster‑> ; Roster Location‑ > Reset. N'entrez pas d'emplacement de liste. De cette façon, il suivra le réglage de l'emplacement des fichiers utilisateur.
  • Préférences‑> Emplacement des fichiers‑> Emplacement des fichiers utilisateur et le définir dans le dossier JMRI que vous avez créé dans Dropbox.
  • Relancez JMRI et utilisez l'Aide ‑ Localisation pour vous assurer que tout est correctement modifié.
  • Si vous avez déjà créé des fichiers des types listés à ce lien (voir cette page pour obtenir de l'aide), les déplacer de l'ancien endroit à votre nouvel endroit. Par exemple, vos dossiers dans la fenêtre Fichiers utilisateur que nous avons ouverte à la première étape doit être déplacé vers le nouvel emplacement des fichiers utilisateurs que vous venez de créer.
  • Utilisez "git add", "git commit" pour déplacer tous ces fichiers sous le contrôle de Git.
  • Enfin, "git push" pour pousser le contenu vers GitHub.

étape 2

Les seules étapes à effectuer sur les ordinateurs suivants sont:

  • "git clone" avec l'URL de la liste pour faire votre téléchargement initial du dépôt ("git pull" après cela)
  • Preferences ‑> Roster ‑> Roster ‑ Roster Location ‑ Reset. N'entrez pas dans Roster Location. De cette façon, il suivra l'emplacement des fichiers utilisateur.
  • Preferences ‑> Locations ‑> Emplacement des fichiers de l'utilisateur et le définir dans le dossier JMRI que vous avez créé dans Dropbox.

    • Sneaker Net

    Tout d'abord, copiez le répertoire complet d'un de vos ordinateurs sur la clé USB. Ce sera un maître que vous déplacerez. (Vous pouvez aussi directement "cloner" sur la clé USB).

    Ensuite, à partir des préférences JMRI de votre ordinateur, le répertoire Git, vous pouvez mettre la clé USB à jour avec votre répertoire avec 

                commit -m "commentaire vraiment lucide" .
            git push /path/to/USB/directory

    De même, vous pouvez obtenir le contenu de la clé USB avec 

                git pull /path/to/USB/directory

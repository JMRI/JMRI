JMRI : Aspect Signalisation

Commander les signaux du modèle de manière réaliste

Présence de systèmes de signaux réaliste "Aspects" pour le conducteur pour lui dire comment faire fonctionner son train. L'aspect "Stop", par exemple, est assez simple : "Arrêtez le train". L'aspect "Approche Moyenne indique quelque chose de plus compliqué : "Procéder à vitesse moyenne à l'approche du prochain signal ". A chaque aspect est associé une "indication", qui est généralement codifiée dans le règlement d'un chemin de fer.

L'image de droite montre un exemple, la règle 282 du règlement de l'AAR de 1946 (Aspect : Approche moyenne).

JMRI pour l'aspect de la signalisation

Une collection d'instructions simples vous permet de configurer Signalisation en 7 étapes rapides.

Aspect Signalisation est pris en charge dans JMRI en utilisant trois Outils de signalisation:

  • Mât de signalisation les objets représentent un signal entier avec un ou plusieurs feux de signalisation . Un mât de signalisation peut faire fonctionner toutes ses feux, tout comme le font les modèles complets de signalisation basés sur les aspects. Quand son aspect est réglé sur, par exemple, "Approche moyenne", JMRI gère la communication avec le réseau nécessaire pour faire que le signal sur le le réseau affiche les bonnes couleurs.
    Les mâts de signalisation sont configuré par l'intermédiaire de l'option Tableau des mâts de signalisation. Vous en ajoutez de nouveaux avec le bouton "Ajouter...". dans le bas du tableau, ce qui vous amène à la Fenêtre "Ajouter un mât de signalisation". Vous y spécifiez le type de système de signalisation à utiliser (voir la liste ci-dessous), le type de signal spécifique ex. "double projecteur" ou "nain à double tête" et quels sont les feux principaux que ce mât va piloter.
    Vous pouvez ajouter une icône de mât de signalisation dans l'éditeur de panneau sur la droite en cliquant à n'importe quel endroit et en choisissant "Ajouter Élément > Mât de Signalisation".
    Dans l'éditeur de réseau, entrez l'adresse du mât de signalisation et cliquez sur un point d'ancrage ou un aiguillage du panneau où vous veulent placer un signal.. L'icône du mât de signalisation affiche les images spécifiées dans la définition de l'apparence pour ce type et système de mât de signalisation particulier. Les icônes du Signal affichées sur votre panneau font partie d'un système de signalisation et ils ne peuvent pas être modifiés individuellement par l'utilisateur comme les Icônes Aiguillage "https://jmri.org/xml/signals/AAR-1946/aspects.xml">

  • LesGroupes de signaux< sont utilisées pour définir un certain nombre de feux de signaux, où un seul feu de signalisation peut être allumée à tout moment, selon l'état du mât de signalisation associé et un ensemble de conditions.
    Les groupes de signaux seraient utilisés lorsque des indicateurs supplémentaires sont utilisé pour soutenir le mât de signalisation principal, tel que Junction, Affichage de l'itinéraire ou de la vitesse. Ces indicateurs fournissent des informations complémentaires au conducteur (US : Ingénieur), qui ne sont pas donnés par le mât de signalisation principal. Ces indicateurs sont fréquemment vu dans les signaux européens et britanniques.
  • L' Outol Logique Mât de Signalisation permet de construire la logique de signalisation comme des paires de conditions entre les mâts de signalisation sur le réseau. La logique du mât de signalisation utilisera les états des Cantonss, Aiguillages, Capteurs et Mâts de signalisation adjacents pour afficher l'aspect que doit avoir un mât de signalisation .
    Si le réseau a été établie à l'aide de l'éditeur de réseau et que Des mâts de signalisation ont été placés sur le panneau à l'aide des différents outils, il est alors possible pour tous les mâts de signalisation de construire une logique dynamiquement, avec peu dintervention de l'utilisateur . Pour les panneaux créés avec Editeur de Panneau ce Commande certaines de ces informations doivent être saisies manuellement.

<Les ObjetsFeu de Signalisation< dans JMRI ne connaissent pas leur position sur un mât ni le Système de signalisation dont ils font partie, donc en tant qu'objets individuels ils ne peuvent pas montrer l'aspect des signaux. Individuellement, ils peuvent être réglé sur des Apparences spécifiques (couleurs), soit à partir de la Table de Feux de signalisation ou via Logix.

Comment ça marche ?

En bref, un ensemble de fichiers pour le système de signalisation sélectionné contient le code de base permettant de déterminer les combinaisons disponible dans l'outilLogique Mât de Signalisation.
Dans le répertoire xml/signals situé dans le programme JMRI il existe un répertoire pour chaque système de signalisation qui a été défini, avec un fichier "aspects.xml" qui énumère tous les aspects possibles plus un certain nombre de dossiers "d'apparence..." (c'est-à-dire appearance-one-searchlight.xml), dont chacun décrit les aspects possibles en se basant sur un type de mât de signalisation. Il s'agit essentiellement de dire "si le prochain signal a un aspect 'x'", alors l'actuel doit montrer un aspect 'y'" (plus details).
Le tableau logique du mât de signalisation est ensuite utilisé pour vérifier quelles paires de mâts de signalisation sont actives dans la logique du mât de signalisation et si l'aspect ne doit pas être "Stop". Ceci est fait en examinant les positions des aiguillages(le cas échéant), l'occupation des Cantons et - éventuellement - les capteurs. Dans l'onglet Mâts de signalisation vous pouvez même spécifier des mâts de signalisation supplémentaires à surveiller, par exemple pour un interverrouillage. Pour chaque mât de signalisation, il peut y avoir de 1 à n mâts de signalisation de destination, dont l'un est actif sur la base de la positions des aiguillages reliant les cantons protégés.
Si tout va bien, alors l'"apparence" appropriée... pour ce type de mât de signalisation sera utilisé par le fichier Logique Mât pour attribuer un nouvel aspect basé sur l'aspect du prochain mât de signalisation.
Il couvre les fonctions de base dans une vue d'ensemble de très haut niveau. (Merci à Dave Sand)

Code d'accès aux informations sur les aspects

Logix<

Les mâts de signalisation peuvent être à la fois testés (sous conditions) et réglés (dans Actions) via Logix. Lorsque vous modifiez un Logix Conditionnel ou Action, vous devez taper le nom du mât de signalisation et appuyez sur entrée/retour pour que le programme puisse consulter les aspects disponibles pour ce mât de signalisation particulier.

Scripting

Un programme (en Java ou Python) peut avoir accès à l'information Aspect de deux façons. Si le code fait référence à un Mât de signalisation spécifique "m", il peut utiliser

   m.getValidAspects()
pour obtenir la liste des noms d'aspects que ce mât de signalisation peut afficher. Le programme peut alors accéder à la définition du système de signaux avec 
   SignalSystem sys = m.getSignalSystem()
et se renseigner ensuite sur les propriétés de l'aspect : 
   sys.getProperty("Clear", "speed") ;
où le premier argument est le nom de l'aspect (utilisez, pour par exemple, m.getAspect() pour obtenir celui en cours sur le mât) et le second est une propriété spécifique. Les propriétés peuvent être définis par programme, par exemple par le biais de m.setProperty("Clear", "speed", "69"); ou se charger automatiquement à partir d'éléments du aspect.xml fichier qui définit le système de signal spécifique.

En plus des propriétés globales pour un aspect, il peuvent également être des propriétés locales à un type de mât de signalisation spécifique. Un exemple de cela est l'image d'icône par défaut : L'image pour un mât de signalisation à deux feux est différent du Mât de signalisation à un feu, même s'ils représentent tous deux "Libre".

Pour les obtenir:

   m.getAppearanceMap().getProperty("Clear", "imagelink") ;
En d'autres termes, cela signifie "Obtenez les informations d'appel pour ce Mât de Signalisation, puis vérifiez la propriété imagelink de l'"aspect" Libre .

Autrement, si vous connaissez le nom du système de signalisation en cours d'utilisation, un programme peut y accéder directement via l' InstanceManager:

   SignalSystem sys = InstanceManager.getDefault(SignalSystemManager.class).getSystem("basic") ;

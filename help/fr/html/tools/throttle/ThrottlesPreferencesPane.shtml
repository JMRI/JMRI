Volet des préférences Régulateurs



Pour une meilleure expérience, le choix recommandé est cellui illustrée ici.

Ce panneau de préférences est accessible à partir des des sous-menus Régulateur dans l'entrée du menu Outils .

Il est recommandé de fermer toutes les fenêtres Régulateurs avant d'appliquer une nouvel ensemble de préférences, puis rouvrez les une fois les nouveaux paramètres appliquée ou sauvés.
  • Le bouton Annuler ferme la fenêtre et laisse les paramètres actuels des régulateurs comme ils sont.
  • Le bouton Appliquer appliquera l'ensemble actuel des préférences et la fenêtre des préférences restera ouverte.
  • Le bouton Enregistrer enregistrera vos préférences actuelles, et fermerera les fenêtres Préférences, les réglages seront chargés automatiquement au prochain démarrage du JMRI.

Utilisez le Régulateur étendu

Ce réglage activera/désactivera toutes les fonctionnalités étendues des régulateurs.

Réglez les Commandes transparentes

Mettra le fond de la fenêtre interne du régulateur en transparence ou semi- transparence.

Cette fonctionnalité est appelée à bien travailler sur OS X, mais Linux et Windows peuvent aussi fonctionner, cela varie complètement suivant l'interface actuelle Java (modifiable dans préférences principales JMRI )

Utiliser l'image de la Liste comme fond

Quand une liste est sélectionnée, l'image définie dans son Panneau des médias sera chargée comme un fond pour la fenêtre de commande du régulateur.

Lorsque cette option est activée et l'adresse sélectionnée est 0 ou 3, une image par défaut sera chargée.

Redimensionnez la fenêtre à la taille de l'image

Quand une image de fond est chargée pour le régulateur actuel, cela va forcer la taille de la Fenêtre du Régulateur à celle de cette image.

Rechercher les infos liste quand l'adresse est indiquée

Quand une adresse est inscrite dans l'AdressPanel , la base de données JMRI recherchera une entrée de registre correspondante (et sera donc des options de déclenchements décrites ci-dessus et ci-dessous si elles sont activés).

Chargement automatique de la fenêtre de préférences régulateurs lorsqu'un lien est trouvé avec la Liste

Quand une liste est sélectionnée, si un fichier de définition régulateur correspondant existe, JMRI le chargera automatiquement.

Ces fichiers sont stockés dans <le Répertoire Preference JMRI >/Throttle/XYZ.xml où XYZ est dans l'ID de la Liste.
Utiliser Le l'entrée Enregistrer du Menu Fichier dans une fenêtre régulateur va créer ou mettre à jour ces fichiers.

Ignorer position du cadre Régulateur

Pour les vieux fichiers régulateurs qui contiennent des position de Fenêtre Régulateur en eux. Faites comme décrit.
Remarque: les positions des Fenêtre Régulateurs sont maintenant sauvegardées dans le Fichier de Sauvegar Configuration Régulateur (pas dans n'importe quels fichiers de sauvegardes Régulateurs )

Masquage indéfini de la Liste Boutons

Quand une liste est sélectionnée, les boutons des panneaux d'adresse sont initialisés comme définis dans le panneau de Fonctions dans DecoderPro, si cette option est activée, les boutons non définis (sans texte) dans PanelPro seront automatiquement cachés dans le Panneau Fonction .

Remarque: vous pouvez réinitialiser la visibilité des boutons de Fonctions dans une fenêtre régulateur en allant dans le Menu Afficher et cliquer sur "afficher tous les boutons de fonction" .

Activer la barre d'outils papillon

Cette option activera une barre d'outils qui apparaîtra dans toutes les Fenêtres Régulateurs .

JMRI Horloges Rapides - Capacité LocoNet

Un Digitrax LocoNet peut contemir sa propre horloge rapide, pilotée par la centrale DCS100, DCS200, ou DCS50 . Une Horloges Rapide LocoNet séparé est disponible auprès de Logic Rail Technologies. JMRI peut fonctionner avec n'importe lequel de ces éléments, ou avec n'importe quelle horloge qui utilise la méthode Digitrax standard pour contrôler les horloges rapides.

Cette page explique comment JMRI peut soit utiliser soit contrôler l'horloge rapide de LocoNet.

Contrôler l'horloge de LocoNet

L'outil "Moniteur d'horloge" dans le menu "LocoNet" fournit un moyen pratique de :

  • Lire rapidement le contenu actuel de l'horloge LocoNet
  • Régler la date et l'heure
  • Définir le taux. Notez que vous ne pouvez pas explicitement mettre en pause une horloge LocoNet, mais vous pouvez l'arrêter en réglant le taux sur zéro.

Si vous utilisez une centrale DCS100 ou DCS200 pour créer votre horloge rapide LocoNet, vous avez peut-être remarqué qu'elle n'est pas très bonne. Si vous sélectionnez l'option "Correct LocoNet Fast Clock Master", JMRI ajustera l'horloge LocoNet afin qu'elle garde un meilleur temps.

Utilisation de l'horloge LocoNet

Les paramètres recommandés pour fonctionner avec l'horloge rapide de LocoNet sont : set Time Source to Internal Computer Clock, et vérifiez les deux Synchroniser... et Corriger... options. Cependant, vous pouvez essayer d'autres paramètres. Si vous utilisez l'horloge interne de l'ordinateur comme source de temps, vous ne pourrez pas changer la fréquence d'horloge ou le temps d'horloge rapide d'un régulateur.

Vous pouvez également demander à JMRI d'utiliser l'horloge de LocoNet pour sa propre base de temps interne. Cela permet de garder les horloges que JMRI affiche à l'écran, synchronisées avec l'horloge LocoNet, d'où les affichages sur les Régulateurs DT400, etc.

Pour ce faire, sélectionnez l'option "LocoNet Fast Clock sets Internal Option "Horloge" sur la fenêtre du moniteur d'horloge de LocoNet. Il est à noter que vous devez faire ceci avant d'ouvrir les fenêtres de l'horloge de sorte que lorsque vous ouvrez les fenêtres de l'horloge, elles se connectent à la base de temps correcte.

Si vous utilisez une horloge rapide de Logic Rail Technologies, elle doit être configuré comme une horloge esclave, et non comme un maître. Vous pouvez remarquer que l'horloge rapide de Logic Rail Technologies a un retard d'une minute d'horloge rapide. Soyez assuré que les commandes correctes sont envoyées par JMRI. Ce décalage, si présent, n'est qu'un artefact de la programmation interne de l'horlogeLogic Rail Technologies.

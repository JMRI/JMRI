JMRI : Stocker votre travail

Ce que tout le monde devrait savoir

Cette page fournit les informations de base que vous devez connaître sur les commandes utilisées pour stocker les données que vous créez et mettez à jour dans JMRI. Une page d'accompagnement contenant plus d'informations pour les utilisateurs avancés sur quand, comment et où JMRI stocke les données est disponible ici.

Le modèle de JMRI pour la création, la mise à jour et le stockage (ou l'élimination) des informations dont il a besoin pour remplir ses différentes fonctions dépend de la fonction JMRI à laquelle vous avez affaire. Certains types de données sont traités de manière entièrement automatique. D'autres sont créés par vous et mis à jour par JMRI. Pour d'autres encore, vous avez la possibilité de modifier ou de conserver ces données d'un passage à l'autre de JMRI. Par exemple, l'enregistrement des modifications apportées aux entrées de l'Inventaire est distinct de l'enregistrement de votre configuration de mise en page.

Malheureusement, JMRI ne vous informe pas toujours de la présence de données non sauvegardées lorsqu'il quitte (bien que la communauté des développeurs travaille constamment à l'améliorer). Par exemple, lorsque vous créez un capteur dans le tableau des capteurs, il apparaîtra dans la fenêtre des capteurs, mais il disparaîtra si vous fermez JMRI - à moins que vous ne preniez l'une des mesures énumérées ci-dessous pour le sauvegarder.

Pour vous assurer que les données que vous créez (ou qui sont créées et mises à jour pour vous) sont conservées comme vous le souhaitez, vous devez comprendre quand et comment les différents types de données sont enregistrés sur un fichier disque, ou non, et, dans le cas des CV, sur vos locomotives et autres décodeurs DCC. Cette page est destinée à vous donner les informations de base que vous devez connaître.

Ce que vous devez faire pour "stocker" ou "sauvegarder"

JMRI vous avertira parfois (mais pas toujours) lorsque vous voudrez "enregistrer" ou "stocker". Ce sont les endroits où l'action de l'utilisateur peut être requise:

  • Préférences (toutes les apps)
    • Tous les éléments de préférences : Modifier⇒Préférences, Appuyez sur "Sauvegarder"

  • DecoderPro
    • Entrée de l'Inventaire :
      • Sur une fenêtre d'entrée individuelle de l'Inventaire: Sélectionnez Fichier⇒ "Sauvegarder" ou "Sauvegarder dans l'Inventaire"
      • Sur une fenêtre d'entrée individuelle de l'Inventaire: Sélectionnez "Ecrire" dans le décodeur afin de modifier les CV
    • Groupe des Inventaires : Sauvegarde automatique<

  • PanelPro
    • Panneaux : Panneaux⇒nouveaux Panneaux , ou de n'importe quel éditeur de panneau : Sélectionnez Fichier⇒ "Sauvegarder Panneaux..." (commande également disponible sur la fenêtre principale de PanelPro : Panneaux⇒ "Sauvegarder Panneaux..."). Notez que cette commande stocke également toutes les tables, elle remplit donc la même fonction que "Sauvegarder Configuration et Panneaux" qui apparaissent sur la fenêtres de laTable.
    • Tableaux : Outils⇒Tables⇒[Table spécifique], Sélectionner :
      • Fichier⇒ "Sauvegarder Configuration" (stocke TOUTES les tables) ou
      • Fichier⇒ "Sauvegarder Configuration et Panneaux" [qui fait la même chose que "Sauvegarder Panneaux" dans les éditeurs de pannneaux]
    • Horloge Rapide: Outils⇒HorlogesrArr; Définir Horloge Rapide..., puis comme pour les Tables.
    • Images/Icônes : Hit "SauvegarderIndex Image "
    • Scripts : Panneauxl⇒Entrée Script,Touche "Sauvegarder..."
    • Régulateurs : Outils⇒Régulateurs⇒
      • "Sauvegarder la disposition de l'accélérateur"
      • Nouveau Régulateur⇒Fichier⇒ "Sauvegarder Régulateur Actuel"
      • Nouveau Régulateur⇒Fichier⇒ "Sauvegarder Régulateur Actuel Sous..." [vous permet en outre de sauvegarder les boutons de fonction personnalisés]

  • DispatcherPro
    • Tables (sections, transits) : Outils⇒TablesArr;[Table Spécifique ], Sélectionner :
      • FichierArr; "Stocker la configuration" (stocke TOUTES les tables) ou
      • FichierArr; " Stocker la configuration et les panneaux" [qui fait la même chose que "Stocker les Panneaux" dans les éditeurs de panneaux]
      aux
    • Options Dispatcher : Outils⇒Dispatcher⇒OptionsArr; "Sauvegarder Options"

  • OperationsPro
    • Toutes les données : Sauvegarde automatique

  • SoundPro
    • Tables Audio (Auditeur, Buffer, Sources) : Outils⇒Tables⇒Audio, Sélectionner :
      • FichierArr; "Stocker la configuration" (stocke TOUTES les tables) ou
      • FichierArr; "Stocker la configuration et les panneaux" [qui fait la même chose que "Stocker les Panneaux" dans les éditeurs de panneaux]

Assurez-vous doncde la présence de vostouche "Sauvegarder" et "Stocker" pour tous les différents outils que vous utilisez au sein de JMRI et vous pourrez toujours avoir ce sur quoi vous avez travaillé la fois suivante. Et s'il y a un problème, n'oubliez pas que JMRI conserve des fichiers de sauvegarde lorsqu'il écrit vos données vers un stockage permanent, soit automatiquement, soit sur votre ordre, recherchez donc les fichiers avec le suffixe ".bak et vous pourrez vous remettre de la plupart des faux pas.

Panneaux Perdus:Un problème fréquemment signalé sur le forum des utilisateurs JMRI est que les panneaux ont disparu au redémarrage de PanelPro et à l'ouverture d'un fichier de panneaux. Souvent, cela est dû au fait que quelqu'un a sélectionné "Stocker configuration" plutôt que "Stocker la configuration et les panneaux" et a écrasé un fichier de panneau existant. Pas de panique - la copie de sauvegarde (dans le répertoire "backupPanels" dans l'Emplacement Profile) est là. La meilleure pratique pour travailler avec PanelPro est d'utiliser presque toujours "Stocker la configuration et les panneaux" ou "Stocker les Panneaux..." où il apparaît, et de renommer le fichier que vous enregistrez afin d'avoir plusieurs versions nommées par vous pour y revenir si nécessaire.

Choses qui sont stockées automatiquement

  • Positions et tailles des fenêtres (toutes ne sont pas enregistrées)
  • DecoderPro
    • Entrée de l'Inventaire : Enregistré automatiquement lorsqu'il est modifié sur la page principale de l'Inventaire
    • Groupe d'Inventaire : nouveau, y compris les changements apportés à l'association des tableaux de groupes d'Inventaires ; également renommage et suppression de groupes
      • .
  • OperationsPro données : stockées automatiquement lorsque les données sont modifiées.

Ce qui n'est PAS stocké après avoir quitté JMRI

Certaines données que vous créerez pendant votre session JMRI ou qui seront créées par communication avec votre réseau ne sont pas enregistrées après dans JMRI. C'est normal car ces données représentent l'état actuel de vos trains et de votre réseau (par exemple, la position des aiguillages), donc nécessite un retour d'information de votre part ou de votre réseau. Ces données transitoires sont:

  • Etat du capteur
  • Etat de l'Aiguille
  • DispatcherPro Trains
  • [Liste éventuellement incomplète]

Cette page décrit les détails techniques du support de connexion CBUS de JMRI .

Adaptateurs CAN

L'approche générale de JMRI est de soutenir de nombreuses combinaisons de matériel. En soutenant le plus grand nombre possible de gens et leurs désirs matériels, nous pouvons créer une communauté croissante de développeurs, donc plus de code.
Dans ce cadre, le soutien du JMRI CAN est conçu pour utiliser autant de types d'adaptateurs CAN que possible.
Peu de PC disposent aujourd'hui d'un port série RS232, mais il est toujours pris en charge.

CBUS CAN Network ID

Tous les modules CBUS ont besoin d'un CAN ID, et JMRI ne fait pas exception à la règle.

Les modules ne doivent pas avoir un double CAN ID

L'ID CAN est distinct de toute ID NŒUD utilisée dans les événements longs CBUS.

Les numéros de nœuds sont pour les événements longs CBUS, les ID CAN sont uniquement pour la transmission du protocole CAN et sont largement hors de propos dans le traitement des événements CBUS par JMRI.

La plupart des modules MERG peuvent effectuer un "dénombrement" pour CBUS, mais certains autres adaptateurs ne peuvent pas le faire en raison de leur latence intrinsèque. La lenteur de leurs réponses les rend en effet invisibles.
C'est pourquoi Mike Bolton, l'un des développeurs de la CBUS a suggéré que JMRI utilise une ID CAN de 126. C'est un valeur fixe dans certains systèmes.

Si vous avez plusieurs copies de JMRIJ attachées à un réseau MERGCBUS, Veillez à donner ces identifiants CAN uniques.

CAN via MERG CAN-RS ou CAN-USB

Un CANUSB4 ne devrait normalement pas nécessiter l'installation de pilotes spéciaux, il devrait juste fonctionner.

Si vous utilisez un MacBook Pro avec Mojave et que vous avez des difficultés avec les pilotes pour un CANUSB / CANUSB2, le pilote approprié est disponible auprès du FDTI pour l'UM245R, l'option VCP.

JMRI: Configuration Connexion SRCP

Le support SRCP sera disponible dans JMRI 2.1.6.

le protocole SRCP a été créé par le der Moba groupe de modélisme ferroviaire comme un moyen standard de connexion d'un programmes utilisateur "client" à un "serveur", qui à son tour se connecte au réseau ferroviaire. Il y a un certain nombre de serveurs SRCP de trains miniatures , qui à leur tour peuvent actuellement se connecter aux réseau par l'intermédiaire d' Intellibox, des unités de contrôle Marklin 605x et les interfaces de Lenz. Il y a aussi un serveur disponible qui peut créer directement des signaux DCC via une inteface RS232, qui peut ensuite utiliser un booster(s) DCC pour alimenter le réseau. D'autres démons SRCP avec d'autres connexions ont été tout le temps développés .

Le support initial JMRI permet à JMRI de se connecter à un réseau grâce à un Serveur SRCP. JMRI peut alors faire fonctionner les Aiguillages, les Lumières et les Cibles Signal s'ils peuvent être atteints par SRCP. Il y a aussi le support initial pour les Régulateurs JMRI via SRCP.

Configuration JMRI

Pour utiliser une connexion SRCP, sélectionnez "Serveur SRCP" comme protocole dans les préférences.

Ensuite remplissez l'adresse Internet pour le serveur SRCP. Ce peut être des nombres, comme 127.0.0.1 ou un nom comme "srcp.sf.net". Un serveur fonctionnant sur le même ordinateur peut habituellement être appelé "localhost".

Remplissez ensuite le numéro de port. Celui-ci peut être obtenu à partir de la configuration de votre serveur SRCP.

Enfin, enregistrez les préférences et redémarrer le programme.

Limitations

Le soutien SRCP a été introduit en JMRI 2.1.6 au cours du printemps 2008. Cette version présente plusieurs limites:
  • Il ne supporte pas le "mode service" pour la configuration de décodeurs, par exemple les CV.
  • Il a besoins de beaucoup plus de test, surtout avec Marklin et autres systèmes DCC.

Structure Matérielle: Configuration Connexion SRCP

le protocole SRCP a été créé par le groupe de modélisme ferroviaire der Moba comme un moyen standard de connexion d'un programmes utilisateur "client" à un "serveur", qui à son tour se connecte au réseau ferroviaire. Il y a un nombre de serveurs SRCP de trains miniatures disponibles , qui à leur tour peuvent actuellement se connecter aux réseau par l'intermédiaire d' Intellibox, et les unités de commande Marklin 605x et les interfaces de Lenz. Il y a aussi un serveur disponible qui peut créer directement des signaux DCC via une inteface RS232, qui peut ensuite utiliser un booster(s) DCC pour alimenter le réseau. D'autres démons SRCP avec d'autres connexions ont été tout le temps développés .

La structure initiale JMRI permet à JMRI de se connecter à un réseau grâce à un Serveur SRCP. JMRI peut alors faire fonctionner les Aiguillages, les Lumières et les Feux de Signalisation s'ils peuvent être atteints via SRCP. Il y a aussi la structure initiale pour les Régulateurs JMRI via SRCP.

Limitations

La structure SRCP a été introduite dans JMRI v2.1.6durant l'été 2008. La première version avait plusieurs limitations;

  • Il ne supportait le "mode exploitation" pour configurer les décodeurs, ex: configuration des CVs.
  • Il nécessitait plus de tests, spécialement avec Märklin et autres systèmes non-DCC.

Connexion

Pour utiliser une connexion SRCP, sélectionnez "Serveur SRCP" comme protocole dans les Préférences.

Ensuite remplissez l'adresse Internet pour le serveur SRCP. Ce peut être des nombres, comme 127.0.0.1 ou un nom comme "srcp.sf.net". Un serveur fonctionnant sur le même ordinateur peut habituellement être appelé "localhost".

Remplissez ensuite le numéro de port. Celui-ci peut être obtenu à partir de la configuration de votre serveur SRCP.

Enfin, enregistrez les Préférences et redémarrer le programme.

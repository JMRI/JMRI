ID Réseau CBUS CAN MERG<

Tous les modules CBUS MERG ont besoin d'une ID CAN, et l'jmri ne fait pas exception à la règle.

Les modules ne doivent pas avoir un double CAN ID

Les numéros de Nœuds sont pour les événements longs du CBUS MERG , les ID CAN sont uniquement pour la transmission du protocole CAN et sont largement hors de propos dans le traitement par JMRI des événements CBUS MERG .

La plupart des modules MERG peuvent effectuer un "dénombrement" pour le CBUS MERG, mais certains autres adaptateurs ne peuvent pas le faire en raison de leur latence intrinsèque. La lenteur de leurs réponses les rend en effet invisibles.
C'est pourquoi Mike Bolton, l'un des développeurs du CBUS MERG a suggéré que JMRI utilise un CAN ID de 126. C'est un valeur fixe dans certains systèmes.

Si vous avez plusieurs copies de JMRI attachées à un réseau CBUS MERG, Veillez à donner ces identifiants CAN uniques.

CAN via MERG CAN-RS ou CAN-USB<

Un CANUSB4 ne devrait normalement pas nécessiter l'installation de pilotes spéciaux, il devrait juste fonctionner.

Si vous utilisez un MacBook Pro avec Mojave et que vous avez des difficultés avec les pilotes pour un CANUSB / CANUSB2, le pilote approprié est disponible auprès du FDTI pour l'UM245R, l'option VCP.
(Conseil fourni par John (G8GKU) via JMRI Users at Groups.io)

Aperçu

DCC++ est un logiciel libre basé sur Arduino Do-it-Yourself Centrale DCC .

Note : DCC++ est un projet distinct du JMRI. Si vous avez des questions sur la construction d'un système DCC++, son fonctionnement, etc, vous devriez obtenir de l'aide par le fil de discussion DCC++ TrainBoard et le le fil de discussion DCC++ Documentation TrainBoard. Il y a aussi des informations sur le DCC++ Dépôt GitHub et documentation associée. Il y avait aussi un ancien Site web du DCC++ et le DCC++ Trainboard discussion. Il y a aussi des pleins informations et d'exemple sur le site Locoduino pour les francophones Les groupes de discussion du JMRI ne peuvent fournir qu'un nombre limité d'aide sur DCC++ lui-même.

Caractéristiques :

  • Entièrement conforme aux normes NMRA DCC
  • Adressage des locomotives sur 2 et 4 octets
  • Contrôle simultané de plusieurs locomotives
  • 128 crans de la vitesse
  • Contrôle de toutes les fonctions de la cabine F0-F28
  • Activer/désactiver toutes les adresses de fonctions accessoires 0-2048
  • Programmation sur la voie de la programmation
    • Ecriture de la variable de configuration bytes
    • Définir/Effacer variable de configuration spécifique bits
    • Lecture des octets de la variable de configuration
  • Programmation sur la voie principale en exploitaion
    • Ecriture de la variable de configuration bytes
    • Définir/Effacer variable de configuration spécifique bits
  • Contrôler les aiguillages et les capteurs du DCC
  • Accès direct et contrôle des broches Arduino IO pour fonctions accessoires et capteurs

Matériel supporté

Centrale

Actuellement, les stations de base DCC++ peuvent être construites soit à partir des modules Arduino Uno ou Arduino Mega. JMRI devrait soutenir soit via une plate-forme USB ou une connexion réseau.

Interfaces informatiques

Actuellement, quatre interfaces sont soutenu :
  • Série/USB : Connexion directe à Arduino via un câble USB
  • Simulateur : Centrale simulée pour le fonctionnement et essais hors ligne .
  • Réseau : Connexion TCP/IP de JMRI à une centralee avec un shield Ethernet ou WiFi.
  • DCC++ sur TCP : connexion réseau à distance à un réseau local Instance JMRI connectée à une centrale

Simulateur

Pour utiliser le simulateur, il suffit de choisir cette option dans les Préférences JMRI. Notez que certaines fonctionnalités ne fonctionnent pas dans le Simulateur. Par exemple, le simulateur ne prend pas (encore) en charge l'attribution, le stockage et la mémorisation des capteurs et des affectations des aiguillages. L'interface du simulateur est suffisamment fonctionnelle pour garder JMRI "heureux" tout en travaillant sur des projets hors ligne tels que l'Exploitation ou la Conception de Panneaul et Logix, mais n'est pas destiné à être un véritable émulateur de centrale.

DCC++ Sur TCP

Pour le DCC++ Over TCP, un ordinateur hôte doit être connecté à la centrale via une connexion série ou réseau (ou un simulateur). Cet ordinateur hôte exécute ensuite le DCC++ sur le Serveur TCP . Un ordinateur distant (ou plusieurs ordinateurs distants) peut utilisez ensuite l'interface DCC++ Over TCP (Server) pour accès et contrôle de la centrale.

Limitations

JMRI

JMRI supporte actuellement l'interface Centrale V1.1 . Certaines commandes de la V1.0 peuvent ne pas fonctionner.

Pour utiliser un Arduino comme centrale, vous devez d'abord assembler l'appareil et télécharger le firmware de la Centrale . Veuillez suivre les instructions des vidéos d'instruction liées ci-dessous sous Autres informations.

Connexion à une station de base DCC++ via USB

  1. Pour connecter votre ordinateur à une station de base DCC++, d'abord installer les pilotes appropriés.
    • Pour la station de base DCC++ avec JMRI, vous aurez peut-être besoin pour installer un pilote de périphérique sur votre ordinateur. Veuillez consulter le site se référer à la Arduino: Pages de démarrage pour les instructions.
      • Windows (Voir étape 4)
      • Mac : Aucun pilote ne devrait être requis
      • Linux

    Note: Si vous avez déjà installé le logiciel Arduino IDE, les pilotes de périphériques doivent être déjà installé.

    Lorsque les pilotes de périphériques sont installés, connectez la station de base Arduino à votre ordinateur en utilisant un standard Câble USB standard.

  3. Une fois les pilotes installés, vous êtes prêt à configurer JMRI. Lancez un des programmes basés sur JMRI, puis aller dans le panneau des préférences. Celui-ci s'ouvre automatiquement la première fois qu'un programme est exécuté, ou vous pouvez le sélectionner dans la Menu "Modifier".
  4. Sélectionnez "DCC++" dans la boîte de sélection supérieure (" Fabricant système").
  5. Sélectionner "DCC++ Serial Port" dans la deuxième sélection case ("Connexion système")
  6. Sélectionnez le port USB/Série approprié à partir de la troisième boîte de sélection (" Port Série")

Connexion à un système DCC++ à l'aide du réseau

  1. Connectez d'abord la centrale DCC++ à un réseau câblé ou WiFi et enregistrer l'adresse IP et le numéro de port. Il peut être nécessaire de connecter temporairement l'Arduino à un ordinateur hôte et utilisez le Moniteur Série Arduino pour obtenir cette information.
  2. Vous êtes maintenant prêt à configurer JMRI. Démarrez l'un des programmes basés sur JMRI puis allez dans le panneau des préférences. Celui-ci s'ouvre automatiquement la première fois qu'un programme JMRI est exécuté, ou vous pouvez le sélectionner dans le menu "Modifier" (dans le Menu d'application sur OS X).
  3. Sélectionnez "DCC++" dans la boîte de sélection supérieure (" Fabricant système").
  4. Sélectionnez "DCC++ Ethernet" dans la deuxième case de sélection ("Connexion au système").
  5. Entrer l'adresse IP et le numéro de port de la centrale DCC .
  6. Cliquez sur "Sauvegarder". On vous demandera si vous voulez le faire pour le pour arrêter, cliquez sur "Oui".
  7. Redémarrer le programme. Vous devriez être opérationnel.

Connexion à un système DCC++ à l'aide de DCC++ sur TCP

  1. Connectez d'abord la centrale DCC++ à l'ordinateur hôte via l'interface série ou réseau comme décrit ci-dessus.
  2. Sur l'ordinateur hôte, sélectionnez "Load DCC++ Over TCP Server". dans le menu DCC++. Vérifiez les paramètres, puis appuyez sur le Bouton "Démarrer le serveur". Si vous voulez que le serveur démarre automatiquement lors du lancement de JMRI, cliquez sur le case à cocher fournie.
  3. L'ordinateur hôte doit fonctionner avec le serveur DCC actif avant le lancement de l'ordinateur client.
  4. Lancez JMRI sur l'ordinateur client .
  5. Dans les préférences de connexion, sélectionnez DCC++ comme Fabricant du système.
  6. Sélectionnez "DCC++ Server" comme connexion au système Type.
  7. Entrer le nom du réseau ou l'adresse IP de l'hôte et vérifier que le numéro de port est le même que sur l'ordinateur hôte.
  8. Sauvegardez vos préférences et redémarrez.

Une fois que les instances hôte et client de JMRI sont configuré, vous pouvez utiliser le JMRI du client de la même manière que si il était directement connecté à la station de base. Le client L'instance JMRI peut se trouver sur le même ordinateur, dans la même maison, ou à l'autre bout du pays de l'hôte.

    Interfaces Informatiques

    • Série/USB: Connexion directe à un Arduino via un câble USB

    Simulateur

    Pour utiliser le simulateur, choisir simplement cette option dans les Préférences JMRI. Notez que quelques fonctions ne travailleront pas dans ce mode. Par exemple, le Simulateur ne prands pas ( pour le moment) en charge les affectations, le stockage et le rappel des affectations des Capteurs et des Aiguillages. L'interface Simulateur est assez fonctionnelle pour garder JMRI disponible pour travailler sur des projets hors-ligne tels que l'Exploitation ou la désignation des Panneaux ou des Logix, mais n'est pas suffisant pour être un émulateur de centrale à part entière.

    Pour le DCC++ Over TCP, un ordinateur hôte doit être connecté à la centrale à travers une connexion Série ou Réseau ( ou un simulateur ). Cet ordinateur hôte exécute le serveur DCC++ Over TCP. Un ordinateur distant ( ou plusieurs ) peut alors utliser l'interface DCC++ Over TCP pour accéder à distance et contrôler la centrale.

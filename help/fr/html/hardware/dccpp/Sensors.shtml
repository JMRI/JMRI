Capteurs DCC++

N'importe quelle broche de l'Arduino peut être utilisée comme entrée Capteur DCC**

Broches de Commande des Capteur DCC++ indexées

Chaque broche Capteur a une entrée dans le tableau Capteur de la Centrale de Commande. La centrale de commande stocke le numéro de broche E/S de l'Arduino IO actuel intérieurement, et vous affectez une valeur d'ID (comprise entre 0 et 32767) au capteur . JMRI référence le capteur en utilisant la valeur ID, pas le numéro de code Arduino.

Il n'est pas nécessaire que les valeurs d'identifiant de capteur soient séquentielles. Chaque ID de capteur peut être n'importe quelle valeur inutilisée dans la plage autorisée (0-32767).

Si vous utilisez des broches analogiques Arduino comme entrées numériques, utilisez le numéro de broche numérique de la broche. Par exemple, la broche analogique A0 est la broche numérique 14

Pour contrôler une broche Arduino E/S à l'aide du contrôle indexé, suivez ces étapes.

1: Choisir un Nom Système et un Nom Utilisateur

L'Adresse Matérielle d'un capteur indexé est la valeur ID attribuée dans le Tableau interne des Capteurs de la Centrale de Commande Par exemple, si votre capteur est connecté à la broche 7 d'Arduino et que la valeur 13 du capteur est attribuée à ce capteur dans la Centrale de Commande, l'adresse matérielle de ce capteur doit être 13.

Le Nom Utilisateur peut être ce que vous voulez.

2: Ajouter le Capteur au Tableau d'Aiguillage JMRI

REMARQUE: Les configurations de capteur stockées dans la mémoire EEPROM de la Centrale de Commande seront automatiquement ajoutées au Tableau de Capteurs JMRI. au démarrage.

Dans le Tableau de Capteurs JMRI...

  1. Cliquer sur le bouton "Ajoutez"
  2. Etre sûr que "DCC++" est sélectionné dans le sélecteur de Nom de Système
  3. Entrer la valeur de l'ID dans la case Adresse Matérielle
  4. Entrer le Nom Utilisateur du Capteur dans la case Nom Utilisateur
  5. Cliquer "OK"
  6. Cliquez éventuellement sur "Modifier" sur la ligne du nouveau capteur et modifiez la configuration du capteur dans JMRI.
Entrée Signification Donner le Nom Système Masquer Equivalent Minimum Maximum
4:3 (convertit à 50) DCCPP50 node : broche 0

3: Configurez la Centrale de Commande

Utilisez l'outil JMRI/DCC++ "Configure Base Station" (4.6.x: "Configure Sensors & Turnouts") pour stocker les ID Capteur et les numéros des broches de l'Arduino dans la Centrale de Commande.

  1. Dans le Menu DCC++, choisissez "Configure Base Station" (ou "Configure Sensors & Turnouts")
  2. Choisissez l'onglet "Capteur"
  3. Cliquez le bouton "Add Sensor". Ceci ajoutera une nouvelle ligne dans le tableau.
  4. Dans la nouvelle rangée, mettre l'"Index" à la valeur de l'ID de la nouvelle sortie
  5. Dans la nouvelle rangée, mettre dans le "Pin #" le numéro de broche de l'Arduino ppour le nouveau capteur
  6. Si la sortie de la broches nécessite l'activation de la résistance de rappel interne, cochez la case "Pullup"..
  7. Cliquez sur "Enregistrer les Capteurs" ou "Fermer" et répondez aux invites.

Remarque: sur les versions 4.6.x et antérieures de JMRI, si vous choisissez "Enregistrer les capteurs", vous ne pourrez pas écrire les modifications dans la mémoire EEPROM de la station de base. Pour écrire sur EEPROM, vous devez choisir "Fermer" au lieu de "Enregistrer les capteurs".

Notes générales

ID de capteur

Les ID de capteur ne doivent pas nécessairement être séquentielless. Vous pouvez utiliser n'importe quelle valeur non utilisée dans la plage autorisée (0-32767). Les ID de capteur peuvent être identiques aux ID d'Aiguillage ou de Sortie.

Utilisation de l'outil "Configurer la Centrale de Commande" JMRI

L'outil "Configurer la Centrale de Commande" est un moyen pratique de programmer les Tableaux Capteur, Aiguillage et Sortie. Centrale de Commande. Cependant, il n'est pas nécessaire de travailler avec DCC ++ dans JMRI. Vous pouvez, par exemple, configurez les Tableaux de votre Centrale de Commande à l'aide de l'interface de commande série DCC ++ ou du Logiciel DCC ++ Controller. Tant que les ID attribués dans la Centrale de Commande correspondent à l'Adresse Matérielle attribué dans le Tableau des Capteurs de JMRI, tout ira bien.

Les modules XBee deDigi International sont une famille de dispositifs de communication sans fil qui ont leur propre microcontrôleur et des broches d'E/S qui qui les rendent utiles pour de nombreux réseaux de commande distribués, notamment sur les chemins de fer miniatures.

Matériel supporté

Centralede commande

JMRI peut communiquer avec les modules XBee Series 1 et XBee Series 2 .

Interfaces informatiques

Le support JMRI pour les appareils XBee comprend des interfaces série aux réseaux XBee. La connexion à l'ordinateur peut être établie par tout adaptateur USB ou RS232 XBee. Les adaptateurs testés incluent:

  • Le Sparkfun Electronics XBee Explorer USB
  • Le Sparkfun Electronics XBee Explorer Série
  • Le CISECO Tranche de Pi (en conjonction avec un Raspberry Pi)

Connexion

Câblage

JMRI peut communiquer avec les modules XBee Series 1 et XBee Series 2 .

Dans les versions de JMRI antérieures à 4.5.5, le support sous-jacent pour la communication est assurée par Andrew Rapp's Bibliothèque xbee-api pour Java.

À partir de la version 4.5.5 de JMRI, le soutien sous-jacent pour la communication est assurée par La bibliothèque Java Digi XBee.

Les deux bibliothèques exigent que le module XBee connecté au PC soit configuré pour l'API mode avec échappements (mode 2).

Le soutien de JMRI pour les modules Digi XBee comprend actuellement la capacité de lire passivement les entrées des capteurs numériques, comme les capteurs JMRI , et déclenchement d'Aiguillage

Adressage du module XBee

Chaque module XBee comporte 5 adresses dont nous devons être informés :

  • un Numéro de la chaîne
  • une adresse PAN 16 bit
  • une Adresse Utilisateur 16 bit
  • Une ID Global Unique 64 bit
  • une chaîne d'identification de nœud de forme libre

Le numéro de canal IEEE 802.15.4 est défini en configurant le paramètre CH de XBee. La valeur du paramètre CH peut être n'importe quelle (hex) valeur de 0x0B à 0x1A. Cela correspond aux valeurs décimales de 11 à 26. Tous les modules XBee du réseau doivent être définis au même numéro de chaîne.

L'adresse PAN (Personal Area Network) de 16 bits est attribuée en définissant le paramètre d'identification de la XBee. Tous les modules XBee doit être configuré pour utiliser le même PAN.

L'adresse utilisateur 16 bits est définie par le paramètre MY de XBee . Certains microprogrammes XBee définissent automatiquement la valeur de MON adresse unique sur le réseau, mais d'autres ne le font pas. Si le paramètre "Mon" est configurable par l'utilisateur, il doit être une adresse unique.

L'identifiant unique global (GUID) de 64 bits est défini en usine. Cette valeur est la chaîne hexagonale imprimée sous chaque module XBee. Vous pouvez également lire la valeur par la lecture du SH (Serial Number High) et du SL (Serial Number Low) des paramètres de la XBee. L'utilisateur ne peut pas modifier les valeurs de paramètres de SH et SL.

L'identificateur de nœud libre est une valeur de texte de 20 caractères stockées dans le paramètre NI de XBee. L'identificateur de nœud doit être unique, et peut être lisible par l'homme.

Note : Vous devez être informés des autres appareils utilisant l'IEEE 802.15.4 les réseaux radio qui peuvent fonctionner dans la même région. Les régulateurs Digitrax Duplex et la radio Roco MultiMaus Pro utilisent la norme IEEE 802.15.4 comme protocole de couche pour le réseau sous-jacent . Si ces appareils partagent un PAN et un canal avec vos modules XBee, les appareils font tous partie du même réseau et l'adresse de l'utilisateur sur 16 bits doit être unique.

Adressage des broches d'entrée/sortie

Lors de la création d'entrées (capteurs) et de sorties (soit lumières ou Aiguillages) dans JMRI, les broches d'entrée-sortie de XBee peuvent être individuellement adressée. Pour accéder à un seul bit, vous pouvez utiliser le formulaire:

ModuleAdresse:Pin

Lorsque l'adresse du module est l'une des adresses de nœud de 16 bits, l'adresse de noeud 64 bits,, ou la chaîne d'identification du nœud et la broche est un numéro d'identification XBee IO.

Les aiguillages peuvent être créés en utilisant deux bits. Pour spécifier un aiguillage créé avec deux bits, vous utilisez le format qui suit :

ModuleAdresse:Pin1:Pin2

Lorsque l'adresse du module est l'une des adresses de nœud de 16 bits, l'adresse de nœud de 64 bits, ou la chaîne d'identification du nœud et la pin1 et Pin2 sont des numéros de pin XBee IO. Les numéros d'identification n'ont pas besoin d' être consécutifs.

Création automatique de capteurs

Quand un message de statut d'entrée XBee est reçu indiquant qu'un (ou plusieurs) des pins d'entrée XBee est utilisé comme entrée, JMRI créera automatiquement un capteur pour chaque broche d'entrée si elle n'existe pas encore.

Lorsque les capteurs sont créés automatiquement, L'identificateur de nœud sera utilisé pour le ModuleAddress s'il n'est pas vierge. Si l'identificateur de nœud est vide, l'adresse MY sur 16 bits sera utilisé pour l'adresse du module, à moins que le champadresse MY contient une adresse de diffusion. Si ni le champ ou l'adresse MY de 16 bits sont utilisés pour l ModuleAddress, le GUID 64 bits sera utilisé pour le ModuleAdresse.

Paramètres du logiciel

  1. Connectez votre adaptateur XBee en utilisant le cable.
  2. Les adaptateurs XBee connectés à l'USB peuvent avoir besoin d'un pilote installé. Voir la documentation fournie par le fabricant de votre adaptateur XBee.
  3. Assurez-vous d'avoir la dernière version de JMRI, puis allez au panneau de préférences d'un programme basé sur JMRI. Celui-ci s'ouvre automatiquement la première fois qu'un programme est exécuté, ou vous pouvez le sélectionner dans le menu "Edition"
    4. .
  4. Sélectionnez "Réseau IEEE 802.15.4" dans la case de sélection du fabricant du système..
  5. Sélectionnez "XBee Network" dans la boîte de sélection connexion au système .
  6. Vous pouvez ensuite sélectionner le port série approprié dans la Boîte de sélection du port série sous Réglages.
    Notez que les adaptateurs USB XBee apparaîtront sous la forme d'un Port série port.
  7. Le "Préfixe de connexion" est utilisé pour aider JMRI à communiquer séparément avec de multiples "connexions" au matériel du réseau. Chaque "connexion" doit avoir un identifiant unique qui est spécifié comme le "Préfixe de connexion". Par défaut, la première connexion XBee reçoit le préfixe "Z", et les réseaux XBee supplémentaires reçoivent des préfixes comme "Z1", "Z2", ... La plupart des utilisateurs devraient être en mesure d'utiliser la valeur par défaut du "Préfixe de connexion" fournie par les outils JMRI
  8. Quand un outil JMRI est configuré pour plus d'une connexion, chaque connexion reçoit un élément de menu dans la fenêtre principale de l'outil JMRI. Pour aider les utilisateurs à différencier leurs différentes connexions, chaque connexion a un "Nom de Connexion ", qui est utilisé comme nom de l'élément de menu associé dans la fenêtre principale de l'outil JMRI. Les utilisateurs peuvent modifier l'élément "Nom de la connexion" pour toute connexion adaptée à leur besoins.
  9. L'adaptateur XBee "Additional Connection Settings" (paramètres de connexion supplémentaires) donne accès à :
    • La vitesse de transmission pour la connexion.
      Assurez-vous que le débit en bauds sélectionné correspond au réglage de la vitesse du port de votre XBee.
    • Intervalle de sortie
      Défaut : 250 millisecondes, destiné au débogage/aux utilisateurs avertis
  10. Le bouton "Configure XBee Nodes" ouvre l'outil de Configuration du nœud XBee . Si vous enregistrez les préférences après la création des nœuds, les informations relatives aux adresses des nœuds seront stockées dans les Préférences JMRI.
  11. Cliquez sur "Sauvegarder". Il vous sera demandé si vous souhaitez quitter le programme ; cliquez sur "Oui.
  12. Redémarrer le programme. Vous devriez être opérationnel.

Outils JMRI XBee

Configuration Nœud XBee

L'outil de configuration du nœud XBee peut être utilisé pour ajouter des nœuds XBee dans une configuration JMRI. Vous pouvez accéder à l'outil à partir du menu Xbee quand une connexion XBee est active. Le bouton "Configurer XBee Nodes" dans l'onglet Connexions en bas des préférences JMRI ouvre également l' outil Configuration du nœud XBee . Si vous enregistrez les préférences après la création des nœuds, les informations relatives aux adresses des nœuds seront stockées dans les Préférences JMRI.

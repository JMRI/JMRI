JMRI: Support xAP

XAP est un protocole domotique populaire utilisé par les amateurs construisant eux-mêmes à travers par de grandes installations professionnelles.

Pour plus d'informations sur le protocole et les équipements disponibles, s'il vous plaît voir les page d'accueil xAP .

xAP ne doit pas être confondu avec le xPA Lenz, qui est un dispositif pour l'aide d'un téléphone portable comme un régulateur pour un système Lenz DCC.

Le soutien JMRI pour xAP a commencé avec un simple script disponible dans JMRI 2.9.5 et versions ultérieures. Finalement, un ensemble complet d'outils de JMRI xAP sera disponibles directement au sein de l'ensemble des outils JMRI.

Les appareils xAP qui fournissent soit "l'entrée" ou la "sortie" forme le Statut de Base et de Contrôle du Schéma (dispositifs BSC). "Entrée" des appareils sont mis en correspondance avec les capteurs JMRI, et "Sortie" des appareils sont mis en correspondance vers des aiguillages JMRI.

Lorsque JMRI voit un nouveau dispositif BSC, il crée le Capteur ou l'Aiguillage nécessaire. Le nom de la source complète est utilisée pour créer le nom du système, par exemple: 

  EST: xPA: xAPBSC: mi4.homeseer.scafell.91: VIOMBoard.Input14GJDRFXEnable
Le xPA-définis Nom défini par l'utilisateur remplit le nom d'utilisateur JMRI, par exemple: 
  Entrée 14 GJD RFX Activer
Vous pouvez modifier ultérieurement le nom d'utilisateur JMRI pour tout ce que vous souhaitez via le tableau habituel, si désiré.

JMRI: MERG CBUS Preferences

This screen allows you to set options which are specific to MERG CBUS within JMRI.

Unless stated, these options are stored within the user profile.

Some options may require you to restart JMRI before they take effect.

Reporters

Reporters can have an active or inactive status which can be used with sensors.

A tag can only be at one location so when it's heard by aother reporter the location is changed.

If a reporter has tags present at the location it's active, else it's inactive.

A sensor following this status can be used for block tracking, and will use the reporter device number as a short event senor number.

If updating is enabled and there is no existing sensor for the reporter, one will be created.

  • Do Not Update a Sensor - Default
  • Provide and Update Sensor status

Reporters act on the DDES ( and ACDAT ) OPCs.

As this data format is flexible, you can specify how your reporters behave here.

At present, the main distinction is between fixed ID RFID tags, and loco identifier formats.

For loco identifier formats, the 2 byte loco address is the uniqueness of the tag.

To report any extra Tag data, some options can provide a memory variable containing the data.

As some formats mean that RFID writing is enabled, an outbound DDES will not update the tag location when these are heard on the network.

  • Classic 5 byte RFID - for Reporters using fixed RFID tags - Default
  • 2 byte Loco Number only
  • Loco Number with 3 byte Memory Variable

JMRI Virtual Sound Decoder

This is the quick reference page for the JMRI Virtual Sound Decoder Manage VSD Locations tool.

The Manage VSD Locations window provides a consolidated location to manage and set the apparent physical location of sounds for trains using Virtual Sound Decoder.

Locations can be assigned to Reporters, Blocks, Operations Locations and to the Listener (there is only one Listener).

You can set X / Y / Z coordinates for each location.
Reporters, Blocks and Operations Locations can be marked as "Tunnel". This will change the sound when a locomotive is within such a location.

Tabs

  • Reporters Tab: Set locations for JMRI Reporters
  • Blocks Tab: Set locations for JMRI Blocks
  • Operations Locations Tab: Set locations for JMRI Operations Locations
  • Listeners Tab: Set location and orientation of the Listener

Notes

  • Listeners attributes are saved to VSDecoder Preferences (VSDecoderPreferences.xml).
    Operations locations are saved to the folder operations in your current JMRI Profile (OperationsLocationRoster.xml).
    Both are loaded automatically at VSD start respectively JMRI start.
  • Reporters and Blocks are not saved to a file.
    If you want to store Reporters or Blocks to a file, go to Tools ⇒ Tables ⇒ table ⇒ File ⇒ Store...
    In case you have an open Panel with a reference to a location, store that Panel.
    Reporters or Blocks must be loaded manually or via JMRI Preferences Startup Action (Open File).
    See Loading and Storing Your Work for details on loading and storing.
  • If you add new Reporters, Blocks or Locations you must close and re-open the Manage VSD Locations window to have them appear.
  • "Use Location" marks locations which should be saved. It is a temporary filter and is not saved.
  • The Reporters and Blocks tables have an additional column "User Name". If a "User Name" doesn't exist, the "System Name" is displayed.
  • The Listeners attribute "Listener Mode" is not supported and will be removed at a later time.

See the documentation pages for more details on Virtual Sound Decoder.

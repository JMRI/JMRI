Adding OBlocks

To add a new OBlock in the tabbed interface, click the Add OBlock... button at the bottom left edge of the Occupancy Tables pane.

In the pane that opens, enter a valid System Name for the OBlock and if desired enter a User Name. For example, a valid System Name for a LocoNet layout could be OB456.

The User Name is optional, unless you check "Automatically generate System Name", and if you enter one it has to be unique in your JMRI setup. See the "Names" Help page for more information.

Click [Create] to complete this step. Click [Cancel] to close the pane (don't worry, already created items will stay).

Complete the definition of the new OBlock by closing the Add New OBlock pane and clicking the Edit button at the right of the newly created Block entry in the OBlock Table.
Fill out all fields as described on the Block Attributes Help page.

Creating a Number of OBlocks

It is also possible to create a number of Blocks in one go.
Where supported, the "Add a sequential range" check box will be enabled.

Checking the "Add a sequential range" box will enable the "Number of items" field. Set the number of OBlocks that you wish to add. Clicking [Create] will create a number of sequentially numbered Blocks.
If a User Name has been entered, a number will be added to the end of that name (starting at #2) to keep each Block User Name unique.

Adding OBlock Properties

To add the details of a new OBlock, close the Add OBlock pane by clicking Cancel or the Close/X button and click Edit to the right of the new entry in the OBlock Table.

See Names under General Help for more information on JMRI object naming.

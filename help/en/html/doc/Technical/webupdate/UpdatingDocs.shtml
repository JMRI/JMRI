If you don't understand HTML here is a chance to learn a little about it. This will not make you a high paid Web Page designer, but should give you enough knowledge to help update JMRI documents.

Using MSWord or Front Page to edit these .shtml documents is not a good idea because they will probably add in their own proprietary formatting codes that will mess things up, possibly even to the point of being unusable by JMRI.

If you are familiar with writing HTML, you can use whatever editing application works for you, but if you're new to it, I suggest Amaya because it allows you to view the document in two ways-- the way it will display in one pane, and the source code behind it in another. The nice thing is you can make your edits in the “display“ view, and never even look at the code!

If you do use an editor like Amaya, it is very important that you set it to make minimal changes to the files you're editing. If you use an editor that (automatically) reformats the original text, it may not be possible to merge your changes into the common version.

To get started, you can drill down inside your CPU's JMRI program files to the folder for the DecoderPro manual. On a WinXP system, the location for this folder looks something like this -- C:\ProgramFiles\JMRI\help\en\manual. In this folder is a file called index.shtml. Open that file with Amaya and look around, being careful not to change it unless you have saved a re-named version.

Updating or Creating a New Document with Amaya

To UPDATE an existing document, open it in Amaya. First, save a newly-named version of it in the same directory. For instance, if it is called index.shtml, re-name it to index_edit.shtml (that's an underscore line using the Shift-Hyphen key). Open that file with Amaya and edit it in the document view (not the code view).

You can delete, add, or change the text there. (To make it easier for the developers to see what you have done, read about inserting and/or deleting text below.) If you want to change the formatting of a word or phrase (for instance, make a phrase into a Heading), select it first and then use the tools under Elements located by default on the right side of the screen. T1 will make it the largest-sized heading, T2 the second largest, etc. If you are making any changes or additions to the images being used in the page, see Linking to Images below. When finished, follow the How To Submit instructions.

To create a NEW document, open an existing document inside the folder where you will want your new document to be placed. Save a COPY of it in the same directory, but add _new to the end of the name (i.e. index_new.shtml). Open that document, and delete any undesired text and/or images and type in what you want. If there are no images to be used, proceed to How To Submit, below.

Linking to Images

Images seen on your page are not inside your .shtml document, they are actually only linked to. In the HTML code, they will look something like <img alt=“description” src=“images/imagename.gif” width=“400” height=“300”>). What matters here is that, before you use Amaya to insert your image, you must first place the image file inside a folder holding all the other images related to the document. In the example above, this folder is named images and there is a GIF image file inside of it named imagename.gif. For more about making your own images, see Creating Images from Screenshots below.

To insert the link for your image into your .shtml document, use Amaya's display view, and place your cursor where you want the image to appear. From the top menu, use Insert-Image.

Or you can use the Insert/Replace Image tool under Elements.

Either way, a dialog box opens. Browse to your image file, provide a description under Alternate Text, and press Confirm. Amaya will insert the code needed, including the path to where the file is in relation to your document, and the width and height of the image.

Your image will appear. Make sure that the image is in its own paragraph, or the text around it will be beside it instead of under it. Also, because it is a paragraph, you can format it to align left, right, or center.

IMPORTANT NOTE: If your image is the wrong size for the page (for instance, too small to be readable or much larger than needed), you will need to adjust your image file and then re-insert it into the .shtml page. See Sizing and Saving Images below.

Creating Images from Screen Shots

For Windows

To snag a screen shot in Windows, start by setting up the window that holds what you want to capture. Make it take up as little space on your screen as you can while it still shows what you want. You can take a shot of the active window (instead of your whole desktop), by pressing the <Alt> key and the <Print Screen> key at the same time. (Note: Depending on your keyboard, you might need to press the <Fn> or <Function-Lock> key as well.) This puts the image on your “clipboard” and you can paste it into an image-editing program such as Window's Paint or Photoshop.

For Linux

If your distro doesn't already have a default application that supports the use of the <Print Screen> and <Alt + Print Screen> keys you can install KSnapshot or your favorite application to do your screen grabbing.

For Mac OS X

After setting up your shot, hold down <Apple + Shift + 4> key. When you release them at the same time, the cursor becomes a plus sign, and you can drag it around the area you want. When you let go, you will have an image file on your desktop.

Using a Third-Party Application

I love IrFanView for screenshots, and it is free. With it, you can also include your mouse cursor in your snapshot. You can also take a series of shots easily, and even make a slideshow of them. You can tell it what file format you want the image files to be AND where you want them to be saved before you take the shot, and that makes it easier to work with them in a program like Paint or Photoshop.

Sizing and Saving the Images

Any changes you make to your image should be done before you use Amaya to insert it into your document. Remember that if you change it, it needs to be re-inserted. Make sure you save the image as one of the following filetypes: PNG, GIF, JPG, or JPEG (not BMP or TIF), and do not use any spaces or capital letters in the name. Generally, a good size for the width of an image is 500 pixels or less, at a web resolution of 72 pixels per inch. If you know how to resize your images, it would be very helpful.

Submitting to Sourceforge

If you understand this, you are ready to help us update the many JMRI documents that we use for both the JMRI website and for the Help files in the next software build.

When you submit your changes to be included in future JMRI releases, they are given a quick check and then merged into the previous content. The check is done by using a "diff" tool to find the parts of the file that have been changed. To make that easier, please don't reformat pages or use an editor that moves lines around on its own.

Prepare the Files

The proper filetype to use for JMRI has the .shtml extension. If you have made an .html or .htm file, you can change the extension yourself to .shtml (just ignore the warning). Don't use spaces or capital letters in your filenames. If you are not changing or adding images, just save the file with a variation of its original name, i.e. “index_edit.shtml” (that's an underscore line using the <Shift-Hyphen> key), save it where you can find it easily, and click the Submit to Sourceforge link below.

If you are submitting more than one file at a time (i.e. the .shtml and some new or edited image files), then you should first copy these files into a new folder somewhere on your hard drive, and then compress (or Zip) the folder. For Windows XP, right click the folder, then Send To -- Compressed.

Send to Sourceforge

Sourceforge is a community of open-source developers and the hard-working volunteers for the JMRI software project would welcome your help whether you want to register or not. When you click the “Submit to Sourceforge” link below, you will be taken to a page that has a link called “Add new Artifact.” If you do this without being a logged-in user, you can submit your file(s) anonymously. If you plan to do a lot of submissions, we suggest you first use the “Log In” link on the top-right side, and then “Create an Account.” This will allow you to keep track of what you have submitted, and allow others to see who is contributing their time. Once you have logged in with your new username, you can return to the proper page by going back two times in your browser. Either way, when you click the “Add New Artifact” link, you only need to provide a brief description of what you are sending, and then upload your file or compressed folder.

Summary

Using a WYSIWYG (What You See Is What You Get) editor like Amaya will allow even a novice to assist the JMRI developers correct and maintain the many pages used for both the Website and the Help Pages in the JMRI program releases. Keeping it simple is key! You needn't be concerned with how it looks while you are working on it, as there is a separate Cascading Style Sheet (a CSS document) that will control the layout and formatting of everything. It is helpful enough to assist in the re-writing and spell-checking and clarification of the existing text, and even more helpful to add pages (and images) you feel are needed.

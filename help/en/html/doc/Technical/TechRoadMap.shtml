Java and JMRI in the Near Future

The 4.1.* series of test releases in Fall 2015 started the requirement for Java 8. This continued through the following release series. Java 8 continued to be the requirement through the Fall of 2021.

The next long-term-support (LTS) Java release after Java 8 is Java 11 from Fall of 2018. Oracle has aligned their Java and the OpenJDK from that point

In Spring 2022 the Java version required by JMRI moved forward, making JMRI 4.27.1 the last JMRI version running on Java 8.
Newer development releases 4.99.x culminated in the June 2022 production releases requiring Java 11 and therefore being called JMRI 5.0.

Java 11's free support by Oracle ended in September 2023.

Java 17 in 2021 was the LTS release after Java 11. Java 17's free support from Oracle runs through September 2026; after that, there won't be security updates and bug fixes to Java 17.

Starting with the JMRI 5.9.* development series, Java 11 is still supported by JMRI but Java 17 is recommended. We recommend Java 17 to move the user installations forward for eventual future migration, because Java 17 is still getting security updates, and because it's easier to install Java 17 on some OSs.

Java 21 in 2023 was the LTS release after Java 17. We expect that we will have enough confidence in it to start recommending it for use with JMRI 5.13.1 in July 2025. That JMRI release will continue to be compatible with Java 11.

Following Java 21, the next Java LTS release is expected to be Java 25 in September 2025.

For more information on compatibility, see the compatibility discussion.

JMRI: Time providers

jmri.TimeProvider is a new Java interface used to provide a time source, for example a fast clock. There can be multiple time providers at the same time.

TimeProvider uses the type letter U. The German word for clock is Uhr

Migration from jmri.Timebase to jmri.TimeProvider

Timebase will be kept for a long time, but JMRI should only use TimeProvider. Users of JMRI are free to use any of them but TimeProvider offers more features. Timebase uses the current selected TimeProvider.

Differences between Timebase and TimeProvider

Timebase uses Date while TimeProvider uses LocalDateTime.

Timebase property minutes have double values while TimeProvider have int values.

Timebase property time have Date values while TimeProvider have property datetime which has LocalDateTimevalues.

TimeProvider also has the property seconds which has the values seconds since epoc as a long.

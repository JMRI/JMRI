JMRI Setup: Projects

Projects allow a single JMRI application to be launched with one of multiple user-specified Configurations. A Project contains all Preferences as they are set in the JMRI Preferences dialog, and may - optionally - include user files, a Roster, and Throttles.

When opening an older JMRI configuration in one of the newer applications, the way Preferences are stored is converted. The application notifies you with this dialog:

When a JMRI application (version 3.9 and up) is launched for the first time, it checks for a Project. If no Project exists, the application will create a default Project and use it without prompting you. If only a single Project is found, the application will use that Project without prompting you. If there are multiple Projects, the application will prompt you for the Project to use. You may specify that the application should always start with the same Project.

If you do not select a new Project and click OK, the program will continue with the default Project after a short delay. You can check a box in the preferences to bypass this step and go directly to the last-used Project.

Project Management

Projects can be managed in the application preferences, under the Project tab.

The line marked "Current active Project" in the Status column is being used by this run of the JMRI application, and is also where Preferences changes will be saved.

To use or configure a different Project, select it, click "Activate", Save and restart the application.

For more information on how to configure JMRI to share its configuration files between multiple computers, see the help page on using Dropbox with JMRI.

Sharing Projects

Exporting

Projects can be exported from the application preferences Projects section. An exported Project is a compressed archive containing the Project's contents, and will optionally contain the user files and roster if they are stored outside the Project.

Importing

JMRI does not yet directly support importing Projects.

A Project can be imported by uncompressing it using an archive management utility into a new directory and then opening while launching JMRI or from the application's Preferences by clicking the "Add Existing..." button.

Note that using the Project Import/Export capability to copy JMRI data back and forth between two computers is neither supported or recommended.

Upgrading from Earlier Versions of JMRI

When you first run a JMRI application after upgrading from an earlier version of JMRI, the application will copy its existing configuration to a new Project, and use that Project. If you are using a JMRI application you had not used prior to upgrading, the application will behave as described above.

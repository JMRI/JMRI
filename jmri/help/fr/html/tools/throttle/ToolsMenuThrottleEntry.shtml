Fusionner des fenêtre Régulateurs

Les fichiers de configuration de régulateur peuvent consister en une fenêtre de régulateur unique (pour contrôler une seule locomotive ou une adresse de couplage), ou elle peut contenir des multiples fenêtres de régulateur. Dans tous les cas, lors de l'ouverture d'un dossier de configuration de régulateur, le programme va vérifier pour voir si des fenêtres existantes de régulateurs actifs sont déjà ouvertes. Si aucune n'est trouvée, le fichier est ouvert, tout comme une ouverture de configuration simple (expliqué ci-dessus).

Charge aussi la configuration des régulateurs des versions précédentes de JMRI. (<2.7.9).Charge <dossier Préférences JMR >/throttleJMRI_ThrottlePreference.xml si il existe.


Figure 4-5: Boite de Dialogue Fusion

Cependant, quand une ou plusieurs fenêtres de régulateur sont découvertes déjà ouvertes sur l'écran, le programme va ouvrir une boîte de dialogue comme le montre la figure 4-5 sur la droite.

  • Fusionner — quitte les fenêtres régulateur actives actuellement ouvertes en place et ouvre le régulateur (s) contenu dans le dossier de configuration en cours d'ouverture.
  • Remplacer —perd le bas de la fenêtre active du régulateur ouvert(s) et ouvre la fenêtre de régulateur (s) établie dans le fichier de configuration sauvegardé.
  • Annuler —annule la la demande d'"ouverture" et ferme cette fenêtre de dialogue .

Remarque: Ne confondez pas l'action qui se produit ici avec toute tentative d'écraser vos fichiers sauvegardés de configuration de régulateur. Les actions ci-dessus n'ont aucun effet sur la configuratrion réelle des fichiers sauvegardés de régulateur eux-mêmes. La seule action "fusion" et "remplacer" qui se produit est tout simplement sur votre écran.
